# v0.5 / 引入 Agent X 循环迭代文档基线

## 版本号和版本分配依据

- 版本号：v0.5。
- 版本标题：引入 Agent X 循环迭代文档基线。
- 分配依据：当前 `update_log.md` 最新正式版本为 v0.4，本轮人工明确指定 v0.5，且本轮只扩展协作文档，不修改 Swift 玩法、Xcode 工程或 CI workflow。

## 背景

项目已经有 Agent A / Agent B / Agent C 三角色流程：

- Agent A：分析人工目标并写版本化实现提示词。
- Agent B：按提示词实现、跑本地轻量检查、提交并 push 到 `origin/main`。
- Agent C：下载 GitHub Actions artifact，核对 manifest、JUnit、日志、失败摘要和最新 `origin/main` commit。

本轮新增未来角色 Agent X 的文档基线。Agent X 作为主控调度角色，围绕人工总目标 X 循环调用 Agent A -> Agent B -> Agent C，再由 Agent X 判断继续、退回、暂停或完成。

## 目标

- 在 `AGENTS.md` 增加 Agent X 的召唤规则、职责、循环边界、停止条件、禁止项和最终回复格式。
- 在 `md/flow/flow.md` 增加 Agent X 主控循环说明。
- 在 `md/flow/flowchart.md` 更新 Mermaid 图，表达人工总目标、Agent X 拆轮、Agent A 提示词、Agent B 实现 push、GitHub Actions artifact、Agent C 验收和 Agent X 判断。
- 在 `md/test/test.md` 增加 Agent X 循环下的验证规则，明确每轮仍以 Agent B 本地轻量检查、GitHub Actions artifact、Agent C 下载复判为准。
- 在 `md/prompt/README.md` 增加 Agent X 相关提示词管理规则。
- 在 `README.md` 简短补充 `agentx:` 入口和 Agent X 不替代 A/B/C 的边界。
- 在 `update_log.md` 增加 v0.5 版本记录。
- 补充小数据量验证、必要 artifact 下载和下载目录容量检查规则。

## 非目标

- 不启动真实 Agent X 自动循环。
- 不新增 Swift 功能，不修改 App 行为。
- 不修改 Xcode project。
- 不修改 `.github/workflows/ci-results.yml`。
- 不新增测试 target。
- 不创建 PR，不切换分支，不 push。
- 不下载 GitHub Actions artifact 或大体积测试数据。

## 当前架构依据

- `AGENTS.md` 是入口规则和多 Agent 工作流。
- `md/flow/flow.md` 和 `md/flow/flowchart.md` 描述当前真实架构、运行流程和云端协作流。
- `md/test/test.md` 定义默认轻量检查、GitHub Actions 云端重验证和 Agent C artifact 复判。
- `md/prompt/README.md` 定义 Agent A 提示词目录和云端阶段要求。
- `README.md` 面向人工快速了解项目和协作入口。
- `update_log.md` 记录正式版本、关键决策和遗留事项。

## 实现步骤

1. 按顺序读取必读文档，并确认 `git status --short --branch`。
2. 补充 Agent X 召唤词：`agentx`、`x:`、`X:`。
3. 补充 Agent X 职责：主控调度、拆分轮次、每轮调用 A/B/C、根据 Agent C 结论判断下一步。
4. 补充 Agent X 停止条件：总目标完成、同一阻塞连续 3 轮、连续 2 轮无有效 diff、CI 同因连续失败、需要账号/权限/密钥/付费服务/人工决策、工作区冲突、用户要求停止或改向。
5. 补充 Agent X 禁止项：无限循环、跳过 Agent C artifact 验收、冒充最新 run、未完成即宣布完成、扩大无关范围。
6. 更新 flow 和 flowchart，把 Agent X 放在 A/B/C 之外作为主控判断层。
7. 更新 test 和 prompt README，确保每轮验证、artifact 下载、容量限制和 Agent C 复判要求清楚。
8. 更新 README 和 update_log，记录本轮是文档准备，不启动真实循环。

## CI / main push 要求

- 本轮不提交、不 push，除非人工明确要求。
- 后续若由 Agent B 执行功能轮次，仍必须同步 `origin/main`、在 `main` 上提交并 `git push origin main`。
- 后续 Agent C 必须下载最新 run artifact，核对 manifest、JUnit、主日志、失败摘要、run id、run attempt 和最新 `origin/main` commit。
- Agent X 主控循环不得跳过 Agent C 的 artifact 验收。

- 失败或阻塞时，Agent X 不能继续下一轮伪装成功。
