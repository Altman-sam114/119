# v0.16（战术命令建议与补线路径读板）

## 版本号

v0.16

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.15（将领成长与军团编制可读化）。本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”做可验证小步迭代，在 v0.14 敌军意图/战线压力和 v0.15 军团编制读板之上补齐玩家侧“这支部队下一步该打、守、补线还是整备”的只读战术建议，因此版本递增为 v0.16。

## 背景

项目已经具备六边形地图、敌军意图路径、战线压力、战术姿态、将领技能和军团编制读板。玩家能看见敌军要做什么，也能看见本方军团战备状态，但地图和命令面板还缺少接近大战略战棋体验的“推荐命令”和“补线路径”提示：当选中罗马单位时，系统应从核心规则只读派生本单位的建议行动类型、目标位置、路径、理由和风险，并在地图与命令面板中展示。

本轮目标是新增只读战术命令建议报告：核心层根据选中单位、可达格、攻击预览、战线压力、友军支援、城市目标、将领技能和军团编制报告派生建议；ViewModel 转成 UI 摘要与地图叠层；BattleView 在地图和选中单位情报中展示。该能力只帮助玩家读局，不自动执行命令，不改变 AI 行为。

## 目标

1. 在 `RomeLegionsCore` 新增只读战术命令建议模型，按单位生成建议报告。
2. 报告至少包含：单位 id、建议类型、目标位置、目标单位/城市 id、推荐姿态、建议路径、优先级、风险等级、预计伤害/补线距离、简短理由和命令文案。
3. 建议类型至少覆盖：可击杀/可攻击、补线支援、城市推进、原地坚守、整备休整。
4. 报告必须由核心层生成，不在 SwiftUI 中重新评分或复制规则。
5. 不新增 `ArmyUnit` / `GameState` 存档字段，不改变 Codable 结构。
6. 不改变真实移动、攻击、AI 决策、AI 评分、姿态切换、技能使用、招募、城市扩建或胜负结算。
7. `GameViewModel` 新增 UI 友好的战术建议摘要，为选中单位提供 `selectedTacticalRecommendationSummary`，并暴露地图路径/目标位置集合。
8. `BattleView` 在地图上显示选中单位的建议路径与目标标记，在完整/紧凑选中单位情报里展示本单位战术建议卡；紧凑界面仍保持“军令”优先，不把新增读板插到军令前。
9. Swift Testing、Gameplay Smoke 和 RenderBattlePreview 覆盖核心建议字段、只读不变性、补线支援场景、攻击场景和 UI 主链路所需字段。

## 非目标

- 不实现自动执行推荐命令、批量规划、多回合 AI 搜索、真实战略路线系统或玩家自动回合。
- 不修改 `performSimpleAI(for:)`、`aiIntent(for:)`、AI 权重、真实 `moveUnit`、`attack`、`useGeneralSkill`、城市占领或战役胜负结算。
- 不改现有战术姿态数值、将领技能数值、战功公式、城市招募成本或地形移动成本。
- 不引入真实编制槽位、装备、科技树扩展、人口、建筑树或外交界面。
- 不拆分 `BattleView.swift` 作为专项重构；只允许新增本轮必要的小型视图和叠层。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、地图、单位、战斗预览、战线压力和军团编制报告位于 `Sources/RomeLegionsCore/GameState.swift`。
- `reachablePositions(for:)` 已按当前主动阵营、行动状态、地形、占用和有效移动力计算可达格；私有 `reachablePositions(for unit:)` 可供核心内部只读规划复用。
- `attackPreview(attackerID:defenderID:)`、`effectiveAttack/Defense/Movement`、`frontlinePressureReports` 和 `legionFormationReports` 已提供战术建议所需数据。
- `GameViewModel.enemyIntentMapOverlays(for:)` 已有六边形路径派生和地图叠层模式，可作为 UI 摘要和路径展示参考。
- `BattleView` 已有敌军意图路径叠层、目标叠层、选中单位编制卡、姿态预览和完整/紧凑命令面板。

## 实现步骤

1. 在 `GameState.swift` 新增 `TacticalRecommendationKind`、`TacticalRecommendationRisk` 和 `TacticalRecommendationReport` 等只读模型。
2. 新增 `GameState.tacticalRecommendation(unitID:) throws -> TacticalRecommendationReport`：
   - 仅根据现有 `GameState` 派生，不写入任何字段。
   - 只对存在单位生成；若单位不存在抛 `GameRuleError.missingEntity`。
   - 对已行动或非当前主动阵营单位仍返回可读建议，但建议应偏向观察/坚守/整备，不能假装可立即行动。
   - 优先选择可攻击目标：复用 `attackTargets(for:)` 和 `attackPreview`，优先击杀、低反击、高伤害目标。
   - 若无攻击目标，选择补线支援目标：优先支援 `frontlinePressureReports(against: unit.faction)` 中受威胁的本方单位/城市，目标位置应来自当前单位可达格中最接近压力目标的格子。
   - 若无压力目标，选择城市推进目标：面向最近可占领或敌对/中立城市，目标位置来自可达格中最接近城市的格子。
   - 若不可移动或无需移动，则按军团编制报告和生命状态给出坚守或整备建议。
   - 建议路径可以用当前格到目标格的相邻六边形粗路径；若找不到完整路径，至少包含起点与目标点，且不得越界。
   - 推荐姿态可复用军团编制报告建议姿态，并在攻击/补线/整备场景中做保守覆盖。
3. `GameViewModel.swift` 新增 `TacticalRecommendationSummary`，提供：
   - `title`
   - `kindLabel`
   - `riskLabel`
   - `priorityLabel`
   - `targetLabel`
   - `pathLabel`
   - `detail`
   - `accessibilityLabel`
   并新增 `selectedTacticalRecommendationSummary`、`selectedTacticalRecommendationPathPositions` 和 `selectedTacticalRecommendationTargetPosition`。
4. `BattleView.swift`：
   - 地图叠层增加选中单位战术建议路径和目标标记，视觉上与敌军意图叠层区分。
   - 完整/紧凑选中单位情报加入战术建议卡，放在军团编制卡附近。
   - 紧凑命令顺序保持不变：`CompactActionsPanelView` 仍在 `CompactSelectionPanelView` 前。
5. `Tests/RomeLegionsCoreTests/GameStateTests.swift`：
   - 新增攻击建议测试，断言目标单位、预计伤害、推荐姿态、风险和只读不变性。
   - 新增补线支援测试，断言目标位置靠近受压单位/城市、路径非空、建议类型和只读不变性。
   - 新增不可行动或低血量整备/坚守测试，避免已行动单位被提示为可立即移动攻击。
6. `Tools/GameplaySmoke/main.swift`：
   - 增加战术建议主链路断言：默认选中罗马单位能得到建议，建议文案、目标/路径和只读不变性可用。
7. `Tools/RenderBattlePreview/main.swift`：
   - 在单位场景断言中加入 `selectedTacticalRecommendationSummary` 非空、路径/目标位置摘要可用，并确认与既有敌军意图和军团编制摘要共存。
8. 更新文档：
   - `README.md` 已实现能力和预览断言说明。
   - `md/flow/flow.md` 当前数据流、战术建议流程、核心对象和扩展点。
   - `md/flow/flowchart.md` 战斗与地图流程图加入战术建议报告。
   - `md/test/test.md` 测试基线、Gameplay Smoke、RenderBattlePreview 断言和 artifact 版本。
   - `md/prompt/README.md` 提示词目录追加 v0.16。
   - `update_log.md` 追加 v0.16 正式记录。
   - `.github/workflows/ci-results.yml` 的 `CI_VERSION` 更新为 `v0.16`。
