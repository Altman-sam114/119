# v0.11 / 将领详情与战术指挥可读化

## 版本号和版本分配依据

- 版本号：v0.11。
- 版本标题：将领详情与战术指挥可读化。
- 分配依据：`update_log.md` 当前最新正式版本为 v0.10（敌军意图路线与目标地图叠层）。本轮继续响应 Agent X 的“大征服者罗马”长期目标，聚焦将领和战术指挥 UI 体验，因此在 v0.10 后递增为 v0.11。

## 背景

当前项目已经具备将领任命、将领被动修正、主动技能预览、技能冷却、战功状态和战术姿态。核心层也已经提供稳定的只读入口：

- `GameState.effectiveAttack(for:)`
- `GameState.effectiveDefense(for:)`
- `GameState.effectiveMovement(for:)`
- `GameState.generalSkillPreview(unitID:)`
- `GameState.warMeritStatus(for:)`
- `GameViewModel.canSetSelectedTacticalOrder(_:)`

但 `BattleView` 中的将领和姿态信息仍偏分散：紧凑情报、完整情报、将领卡和战术姿态按钮各自拼接字段，玩家不容易一眼看出“这个将领当前具体贡献了什么”“技能为什么能用或不能用”“战功带来什么”“切换到突击/坚守/行军后攻、防、移动会变成多少”。本轮目标是只做 UI 派生和展示，把已有核心规则变得更可读，不改变任何数值、AI、结算或存档结构。

## 目标

1. 在 `GameViewModel` 为选中单位新增 UI 派生数据，集中描述选中单位的指挥信息：
   - 单位基础状态：生命、有效攻击、有效防御、有效机动、射程、当前姿态。
   - 将领状态：将领名、将领特性、被动贡献、技能名称、技能状态、冷却、可执行性、不可用原因。
   - 战功状态：军阶、经验、伤害加成、下一军阶进度。
   - 战术姿态预览：均衡、突击、坚守、行军分别切换后预计攻击、防御、机动、是否当前姿态、是否可切换、不可切换原因。
2. `BattleView` 只消费这些 ViewModel 派生数据，重构当前分散拼接的将领卡、紧凑情报和战术姿态按钮展示。
3. 展示当前将领被动贡献，要求能看懂贡献来源：
   - `eagleStandard`：攻击加成。
   - `siegeEngineer`：攻城伤害加成。
   - `quartermaster`：机动加成。
   - `shieldWall`：防御加成。
   - 没有将领时显示“无将领 / 无被动贡献”，不要伪造加成。
4. 展示主动技能状态，要求至少包含：
   - 技能名和技能说明。
   - 范围、目标数、预计恢复或城防削弱。
   - 冷却剩余。
   - 可用 / 不可用状态。
   - 不可用原因，例如冷却中、无目标、已行动、非当前阵营、战役已结束。
5. 展示战术姿态切换预览：
   - 每个 `TacticalOrder` 显示切换后的攻 / 防 / 移数值。
   - 当前姿态有明确标记。
   - 不可切换时展示原因，不只禁用按钮。
   - 禁用逻辑必须仍来自 `GameViewModel` / `GameState` 既有规则，不在 SwiftUI 中复制核心规则。
6. 保持短横屏、竖屏、宽屏三种战斗页布局可读：不遮挡地图、敌情、技能叠层、攻击按钮和命令入口。
7. 更新渲染预览工具，让三尺寸预览能看到将领详情、技能状态、战功和战术姿态预览，并在渲染前断言 ViewModel 派生数据存在。
8. 同步 README、flow、flowchart、test、update_log 和 CI artifact 版本到 v0.11。

## 非目标

- 不改核心数值：不调整 `TacticalOrder` 的攻防移加成，不调整 `GeneralTrait` 加成，不调整 `WarMeritStatus` 公式，不调整技能恢复量、削城防量或冷却回合。
- 不改 AI 决策：不修改 `performSimpleAI`、`aiIntents(for:)`、AI 评分、目标选择、姿态选择、技能释放判断或敌军意图排序。
- 不新增升级树、将领成长树、将领列表、装备、技能点、天赋点或新资源。
- 不新增、删除或修改 `ArmyUnit` / `GameState` / `AIIntent` 的 Codable 存档字段。
- 不把核心规则写进 SwiftUI view body、按钮闭包或 `onAppear`。
- 不做大规模 `BattleView.swift` 拆分；只允许为本轮 UI 派生展示提取必要的小型值类型和小型 View。
- 不引入第三方框架、外部素材、网络服务或新分支/PR 流程。

## 当前架构依据

- `Sources/RomeLegionsCore/GameState.swift`
  - `TacticalOrder` 定义均衡、突击、坚守、行军及其 `attackBonus`、`defenseBonus`、`movementBonus`、`detail`。
  - `GeneralTrait` 定义将领特性、被动加成、主动技能说明、技能范围、恢复量、削城防量和统一冷却。
  - `GeneralSkillPreview` 已提供技能范围、目标、预计恢复、预计城防削弱、冷却、可执行性、不可用原因、摘要和详情。
  - `WarMeritStatus` 已提供军阶、经验、伤害加成、进度和摘要。
  - `effectiveAttack(for:)`、`effectiveDefense(for:)`、`effectiveMovement(for:)` 是展示攻防移的唯一数值来源。
  - `setTacticalOrder(unitID:order:)` 是真实切换姿态的唯一核心入口。
- `RomeLegionsApp/App/GameViewModel.swift`
  - 已持有 `selectedUnit`、`selectedGeneralSkillPreview`、`selectedWarMeritStatus`、`selectedGeneralSkillButtonDetail`、`selectedGeneralSkillCooldownDetail`、`canSetSelectedTacticalOrder(_:)`。
  - 本轮应在这里新增选中单位的 UI 派生模型，例如 `SelectedCommanderBrief`、`SelectedTacticalOrderPreview`、`GeneralPassiveContribution` 或等价命名。
  - ViewModel 可以通过复制一个临时 `ArmyUnit` 并替换 `tacticalOrder` 来计算“如果切到某姿态后的有效攻防移”；这只是 UI 预览，不得写回 `state.units`。
- `RomeLegionsApp/Views/BattleView.swift`
  - `SelectionPanelView`、`CompactSelectionPanelView`、`GeneralTraitCardView`、`CompactGeneralTraitView`、`TacticalOrderControlView` 是本轮主要展示入口。
  - `ActionsPanelView` / `CompactActionsPanelView` 中的技能按钮 detail 应复用 ViewModel 的技能状态文案。
  - 当前文件已有 `GeneralTrait` 和 `TacticalOrder` 的 SwiftUI extension，可继续复用图标和颜色。
- `Tools/RenderBattlePreview/main.swift`
  - 当前会构造一个凯撒将领单位，并渲染短横屏、竖屏、宽屏 PNG。
  - 本轮应在渲染前加入 ViewModel 派生断言，确保将领详情和战术预览不是只靠肉眼检查。
- `md/test/test.md`
  - 当前 Swift Testing 基线为 41 个用例。
  - 本轮原则上不改核心规则；如确实没有改核心文件，可以不新增 Swift Testing 用例，但必须用 SwiftUI 类型检查 / 预览渲染工具覆盖 ViewModel 和 UI 派生。

## 验收标准

- `GameViewModel` 暴露选中单位的将领详情、被动贡献、技能状态、战功状态和战术姿态预览派生数据。
- `BattleView` 的完整情报、紧凑情报和战术姿态控制只消费 ViewModel 派生数据，不在 view body 里复制核心规则。
- 将领卡能明确展示当前被动贡献、主动技能可用性、冷却、目标/效果和不可用原因。
- 姿态控制能展示每个姿态切换后的攻、防、移和不可切换原因，且当前姿态标记清晰。
- 短横屏、竖屏、宽屏三尺寸预览图中，将领详情、姿态按钮、战功进度、技能状态和 v0.10 敌军意图叠层无明显裁切、重叠或遮挡。
- 不改核心数值、不改 AI 决策、不新增升级树、不新增存档字段。
- `git diff --check`、`node Tools/verify_project.mjs`、YAML 解析、SwiftUI 类型检查和三尺寸预览命令通过。
- `main` push 后 GitHub Actions 最新 run 成功，Agent C 下载最新 v0.11 artifact 复判通过。

## 风险和禁止项

- 禁止改变 `TacticalOrder`、`GeneralTrait`、`WarMeritStatus`、`GeneralSkillPreview` 的核心数值语义。
- 禁止修改 `performSimpleAI`、`aiIntents(for:)`、AI 评分、目标选择或真实执行顺序。
- 禁止新增升级树、天赋树、存档字段或 Codable schema 变化。
- 禁止把姿态切换规则、技能可用规则或战役结束保护复制进 SwiftUI 视图。
- 禁止通过 UI 文案掩盖核心规则错误；真实命令仍必须走 `GameState` mutating 方法。
- 禁止让新增卡片撑爆短横屏或竖屏布局，文本不得溢出按钮或遮挡地图。
- 禁止只靠颜色表达攻防移变化，必须有文字、符号或图标辅助。
- 禁止引入第三方库、外部素材、网络依赖或新分支/PR 流程。
- 禁止提交 DerivedData、PNG 预览图、构建缓存、xcresult 历史包或下载的 artifact。
