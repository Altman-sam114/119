# v0.24（军团成长决策读板）

## 版本号

v0.24

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.23（主动地图叠层图例）。Agent X 总目标要求继续向“大征服者罗马”推进，持续优化战斗界面 UI、地图、AI 和将领。本轮接续 v0.17 军团编制与成长读板、v0.20 本方将领协同、v0.21 机动落点和 v0.23 地图图例，补足单位成长命令的决策解释：训练和任命已有核心命令，但 UI 只给按钮，不展示成本、收益、阻塞原因和推荐依据。因此版本递增为 v0.24。

## 背景

当前 `GameState.trainUnit(id:)` 和 `GameState.appointGeneral(unitID:)` 已能消耗资源、提升经验、恢复生命或任命将领；`BattleView` 完整军令面板有训练/任命按钮，选中单位情报也已有军团编制、战功、将领和协同读板。

短板是训练/任命缺少和城市扩建、招募类似的只读预览。成本、候选将领、预计军阶、预计恢复、阻塞原因和按钮禁用条件散落在命令入口或完全不可见。玩家无法判断“现在该训练谁、为什么任命这个单位、资源差多少”。本轮新增军团成长决策读板，让训练和任命先有核心只读预览，再由 ViewModel 和 SwiftUI 展示。

## 目标

1. 在 `GameState.swift` 新增只读核心预览：
   - `TrainingPreview`
   - `GeneralAppointmentPreview`
   - 公开入口 `trainingPreview(unitID:)` 和 `generalAppointmentPreview(unitID:)`
2. 训练预览至少包含：
   - unitID、cost、canTrain、blockedReason
   - currentExperience、projectedExperience
   - currentRankName、projectedRankName
   - currentDamageBonus、projectedDamageBonus
   - projectedRecoveredHealth、projectedHealth
   - summary/detail
3. 任命预览至少包含：
   - unitID、cost、canAppoint、blockedReason
   - candidateName、candidateTrait
   - currentExperience、projectedExperience
   - currentRankName、projectedRankName
   - projectedDamageBonus
   - summary/detail
4. `trainUnit(id:)` 和 `appointGeneral(unitID:)` 必须复用预览的成本、阻塞判断和候选将领/trait，避免读板和实际结算分叉。
5. 在 `GameViewModel.swift` 新增 UI 摘要：
   - `UnitDevelopmentDecisionSummary`
   - `selectedUnitDevelopmentDecisionSummary`
   - 可选的训练/任命按钮 detail 与 can 执行派生字段
6. `BattleView.swift` 新增“成长”读板：
   - 完整选中单位情报展示训练/任命两行成本、收益、状态。
   - 紧凑情报展示同一读板的压缩版本，避免撑爆视口。
   - 完整军令面板训练/任命按钮 detail 使用预览摘要，并用预览 can 字段禁用。
   - 紧凑军令可以继续不新增训练/任命按钮，除非布局足够稳定；但情报读板必须可见。
7. `Tools/RenderBattlePreview/main.swift` 增加云端断言：
   - 选中单位成长决策摘要存在。
   - 训练和任命预览均存在。
   - title、cost、impact/status、accessibilityLabel 非空。
   - 训练预览预计经验/军阶/恢复可读，任命预览候选将领和 trait 可读。
   - 失败抛出 `missingUnitDevelopmentDecisionSummary`。
8. Swift Testing 和 Gameplay Smoke 增加核心只读预览断言：
   - 预览不改变状态。
   - 训练预览成本与真实训练一致，执行后经验、生命、资源和行动状态匹配预览。
   - 任命预览候选名/trait 与真实任命一致，执行后经验、资源和将领字段匹配预览。
   - 资源不足、非当前阵营、已任命等阻塞可读。
9. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.24`。
10. 更新 README、flow、flowchart、test、prompt README、update_log，并把 v0.23 云端通过证据写入 `update_log.md`。

## 非目标

- 不改变训练或任命的既有资源成本、经验收益、恢复量、候选将领顺序、trait 映射或行动消耗，除非为复用预览做等价抽取。
- 不新增升级树、装备、兵种转职、将领池 UI、将领改名、技能选择或自动训练。
- 不改变军团编制、战术建议、机动落点、将令协同、AI、移动、攻击、技能、城市、外交、任务或胜负结算规则。
- 不新增第三方框架，不拆分大型 SwiftUI 文件，不做 BattleView 全局重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 验收标准

- 核心层存在训练和任命只读预览，并复用到真实命令。
- 预览不改变 `GameState`。
- 训练/任命按钮和 UI 读板使用 ViewModel 派生的预览摘要，而不是 SwiftUI 重算成本或候选。
- 训练预览显示成本、预计经验/军阶/伤害、恢复和阻塞。
- 任命预览显示成本、候选名、trait、预计经验/军阶/伤害和阻塞。
- 完整与紧凑选中单位情报都能看到成长决策读板。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止让训练/任命预览和真实结算使用两套成本、候选或收益逻辑。
- 禁止把核心成长规则写入 SwiftUI。
- 禁止改变既有训练/任命数值、候选顺序或 trait 映射。
- 禁止把本轮扩成升级树、装备系统、将领池管理或自动训练。
- 禁止运行任何本地测试、build、typecheck、RenderBattlePreview、`verify_project`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。
- 禁止提交 DerivedData、xcresult、本地 cache 或历史 artifact。
- 如果 GitHub 权限、网络或 artifact 下载失败，必须明确阻塞，不能伪造云端通过。
