# v0.36（目标线阶段联动高亮）

## 版本号

v0.36

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.35（目标线阶段命令预览），并已通过 `origin/main` 最新云端结果包复判。v0.35 已让目标线阶段说明既有命令入口、下一步、姿态/落点/目标/技能状态，但地图阶段、目标线卡片、姿态按钮、攻击按钮和技能按钮之间的“当前同一阶段”联动仍不够明确。下一步应把同一目标线阶段投射到各命令入口高亮，递增为 v0.36。

## 背景

当前项目已有：

- `BattleObjectiveChainSummary`：组合战场焦点、将令协同、机动落点和本方军议。
- `BattleObjectiveMapOverlay` / `BattleObjectivePositionOverlay`：把目标线映射到地图阶段标记和连线。
- `focusPrimaryBattleObjectiveStage(_:)`：只读定位目标线阶段。
- `BattleObjectiveStageCommandPreview`：把目标线阶段转成只读命令入口、推荐姿态、落点、目标、技能状态、下一步和阻塞原因。
- `BattleObjectiveChainCardView`、`BattleObjectiveStageCommandPreviewView`、`TacticalOrderControlView`、完整/紧凑 `ActionsPanelView`：已经分别展示目标线卡片、阶段命令预览、姿态按钮、攻击按钮和技能按钮。

当前短板是：同一目标线阶段已经能被定位和解释，但玩家仍需要自己把地图阶段徽标、目标线卡、阶段命令预览和具体按钮入口对应起来。v0.36 应增加统一联动高亮，使“当前阶段 -> 应看的地图标记 -> 当前军令入口”成为同一条视觉线索。

## 目标

1. 在 `GameViewModel.swift` 增加目标线阶段联动 cue 的只读派生：
   - 建议在 `BattleObjectiveStageCommandPreview` 增加 `commandEntryCueLabel`、`recommendedOrderStageCueLabel`、`attackStageCueLabel`、`skillStageCueLabel`、`shouldHighlightSkillEntry`、`isCommandUnit(_:)`。
   - 建议新增 `activeBattleObjectiveStageCommandPreview` 和 `activeBattleObjectiveStageRole`，优先级为 `focusedBattleObjectiveStageCommandPreview` > `selectedBattleObjectiveStageCommandPreview` > `primaryBattleObjectiveStageCommandPreview`。
   - 不新增 `@Published` 高亮状态，不保留 stale preview。
2. 在 `BattleView.swift` 建立同一阶段高亮：
   - 目标线阶段按钮应在 `focusedRole == role` 或 `stageCommandPreview?.role == role` 时高亮，并在无障碍文案中读出命令入口 cue。
   - 地图目标线阶段徽标应使用 `activeBattleObjectiveStageRole` 高亮，使未手动 focus 时也能高亮当前主阶段或选中单位阶段。
   - `TacticalOrderControlView` 应同时支持反制推荐和目标线推荐姿态高亮；反制优先，目标线作为次级 cue，不改按钮 action 或 disabled。
   - 攻击按钮应使用 `attackStageCueLabel` 展示目标线目标 cue；反制目标命中时仍反制优先。
   - 技能按钮应在 `.synergy` 阶段且 `shouldHighlightSkillEntry == true`、选中单位为该阶段执行单位时追加 `skillStageCueLabel`，不改变 `canUseSelectedGeneralSkill`。
3. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingBattleObjectiveStageLinkedHighlight`。
   - 断言聚焦阶段、地图 overlay、阶段命令预览、选中单位阶段预览和按钮 cue 同源。
   - 断言推荐姿态 cue 来自 `selectedBattleObjectiveStageCommandPreview.isRecommendedOrder(order)`。
   - 若目标线阶段目标当前可攻击，断言攻击 cue 来自 `selectedBattleObjectiveStageCommandPreview.isAttackTarget(target)` 和 `attackStageCueLabel`。
   - 继续证明只读：联动检查前后单位、城市、资源、回合、主动方和技能冷却不变。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.36`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.35 云端通过证据。

## 非目标

- 不修改 `GameState` 的核心评分、路径、AI、战斗结算、技能结算、姿态规则或报告排序。
- 不改变现有攻击、移动、姿态、技能、结束回合、城市或 AI 的 action / disabled 逻辑。
- 不把高亮状态写入 `GameState` 或新增持久存档字段。
- 不在 SwiftUI `body` 中重新计算目标线阶段规则、路径、攻击目标或技能目标。
- 不新增装备、科技树、外交、存档 UI、建筑树、多回合搜索或新第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

- `BattleObjectiveStageCommandPreview` 已有 role、position、sourceSummaryID、commandUnit、recommendedOrder、targetUnit、canAttackCurrentTarget、canUseGeneralSkill 和 `isRecommendedOrder(_:)` / `isAttackTarget(_:)`，适合继续作为唯一联动 cue 来源。

- 完整/紧凑攻击按钮已经读取 `selectedBattleObjectiveStageCommandPreview`，只需换成带阶段名的 cue。

## 验收标准

- 玩家聚焦或选中目标线阶段后，目标线卡、地图阶段徽标和军令面板显示同一阶段高亮。
- 推荐姿态按钮能显示目标线阶段 cue，且反制 cue 存在时反制优先。
- 可攻击目标按钮能显示带阶段名的目标线攻击 cue，且 action/disabled 不变。
- 将令技能阶段能在技能按钮 detail 中显示目标线技能 cue，且技能可用性仍由 `canUseSelectedGeneralSkill` 决定。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算路径、攻击、技能、姿态或评分规则。
- 联动高亮不改变 `GameState` 核心状态，不移动、不攻击、不发动技能、不切换姿态。
- 云端 RenderBattlePreview 断言覆盖阶段联动高亮、同源性、只读性和关键 UI 文案。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。
