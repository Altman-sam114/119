# v0.28（反制落点地图叠层）

## 版本号

v0.28

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.27（敌情反制建议读板）。v0.27 已把敌方威胁与我方回应串成可读反制建议，但地图上仍缺少“回应单位从哪里介入、建议落到哪里、要反制哪个目标”的空间关系。本轮递增为 v0.28，目标是在战斗地图叠层中展示首要反制建议的路线、推荐落点和威胁目标，让战斗界面继续接近“大征服者罗马”的战场态势读图体验。

## 背景

当前项目已有：

- 敌军意图路线和目标叠层：`EnemyIntentMapOverlay`、`EnemyIntentRouteLayerView`、敌路/敌标图例。
- 本方战术建议路线和目标叠层：`TacticalRecommendationSummary.routeSegments`、`TacticalRecommendationRouteLayerView`、军议图例。
- 机动落点叠层：`maneuverOptionOverlaysByPosition`、`ManeuverOptionTileOverlay`、机动图例。
- v0.27 反制建议读板：`CountermeasureReport`、`CountermeasureSummary`、顶部 chip、敌情卡、战局行和 RenderBattlePreview 摘要断言。

短板是反制建议仍停留在读板文本中，玩家必须自行把回应单位、推荐落点、威胁目标和地图坐标对应起来。本轮要把 v0.27 的反制建议投射到地图上。

## 目标

1. 在 `GameViewModel.swift` 新增只读反制地图叠层摘要：
   - 复用 `CountermeasureSummary.report.responsePosition`、`destination`、`targetPosition`、`responseUnitID`、`targetUnitID`、`targetCityID`。
   - 暴露首要反制路线 segments、推荐落点、威胁目标和按位置索引的 tile overlay。
   - 叠层计算必须在 ViewModel 中完成，不写进 SwiftUI `body` 的复杂逻辑。
2. 在 `MapOverlayLegendKind` 和 `activeMapOverlayLegendItems` 增加反制图例：
   - 图例标题建议为“反制”。
   - 说明需表达蓝绿/青色反制线、推荐落点或目标标记含义。
3. 在 `BattleView.swift` 渲染反制地图叠层：
   - 地图上显示首要反制建议的“回应位置 -> 推荐落点 -> 威胁目标”关系。
   - 推荐落点和威胁目标使用稳定的 hex overlay，不遮挡现有攻击、技能、敌标等更高优先级状态。
   - 叠层样式需与现有敌路、军议、机动、热区、控区统一，使用已有 `CountermeasureKind.systemImage` 和 `CountermeasurePriority.tintColor`。
4. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - `primaryCountermeasureSummary` 仍存在。
   - 新增反制 overlay positions/segments 非空。
   - overlay 包含首要反制建议的 `destination` 和 `targetPosition`。
   - `activeMapOverlayLegendItems` 包含新增反制图例 kind。
   - 失败抛出新的 `missingCountermeasureOverlay`。
5. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.28`。
6. 更新 README、flow、flowchart、test、prompt README、update_log；保留 v0.27 最新云端通过证据。

## 非目标

- 不改变 `GameState` 反制建议生成规则，除非发现必须补齐只读字段的极小缺陷。
- 不自动执行反制，不新增自动下令按钮。
- 不改变 AI 评分、AI 执行、移动、攻击、技能、姿态、城市、外交、胜负结算。
- 不引入第三方框架，不拆分大型 SwiftUI 文件，不做全局 UI 重构。
- 不新增多回合搜索、装备、将领池 UI、科技树、建筑树或存档 UI。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `CountermeasureReport` 已提供本轮所需的 response、destination 和 target 坐标。
- `GameViewModel` 已承载敌军路线、战术建议路线、机动落点、热区、控区和图例派生数据，是新增只读地图叠层的正确位置。
- `BattleView` 的 `WarMapView` 已按 `GeometryReader -> HexMetrics -> route layer -> tile overlay -> legend` 渲染地图，适合加入一个同级反制路线层和 tile overlay。
- `Tools/RenderBattlePreview/main.swift` 已断言地图叠层、图例和关键读板，可扩展反制叠层断言。
- 当前验证固定走 GitHub Actions artifact，不能本地跑测试或渲染。
