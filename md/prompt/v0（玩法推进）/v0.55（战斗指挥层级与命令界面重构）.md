# v0.55 / 战斗指挥层级与命令界面重构

## 版本号与分配依据

- 版本号：`v0.55`。
- 分配依据：当前最新正式版本和玩法推进提示词为 `v0.54`，本轮继续递增一个小版本。
- 本轮属于战斗 UI 结构重构，不新增核心玩法规则，不复制《大征服者：罗马》的品牌、人物、图标或截图素材。

## 背景

v0.54 已完成原创地图材质、战区尺度、城市剪影和军旗标记重构，但战斗页仍有三个明显层级问题：顶部资源带和底部命令坞偏厚；右侧五入口被统一深色容器串成显眼工具栏；非当前侦察视角下仍可能同时出现过多路径、边框和格子标记。公开官方截图只用于提炼“地图主导、边缘命令、局部聚焦”的层级规律，不能下载进仓库或描摹其素材。

## 本轮目标

1. 把顶部改成更薄的战役状态带，保留菜单、势力/回合、战役状态、五类资源和结束回合入口。
2. 把右侧五个抽屉入口改为相互独立的轻量边缘控制，保留全部类别、选中态、44pt 可访问点击区域和原有 action。
3. 把底部命令区改成清楚的“选中对象身份 -> 当前目标/下一步 -> 主要命令 -> 次要命令”层级，减少整块深色遮挡。
4. 进一步降低非当前侦察视角的路线和默认 tile 边界噪声，攻击、可达、技能目标、当前阶段和当前侦察视角始终拥有更高优先级。
5. 增加可由云端 RenderBattlePreview 验证的稳定布局指标和像素门禁，覆盖顶部/底部占屏比例、边缘工具密度、地图主体和单位/城市命令区差异。

## 非目标

- 不改 `GameState`、AI 评分、移动、攻击、技能、姿态、任务、经济、存档或回合规则。
- 不新增 ViewModel 业务计算，不让 SwiftUI 重新计算战术评分、目标选择或禁用规则。
- 不删除抽屉、侦察视角、地图图例、既有情报卡或命令入口。
- 不引入第三方框架，不拆分大型文件，不创建新分支或 PR。
- 不复制、下载、提交或描摹《大征服者：罗马》的截图、人物、UI 图标和品牌资产。

## 当前架构依据

- `RomeLegionsApp/Views/BattleView.swift` 持有战斗壳层、地图、边缘工具、地图情报坞和选择命令坞，只负责展示状态和触发 `GameViewModel` 命令。
- `RomeLegionsApp/App/GameViewModel.swift` 已提供选择态、攻击目标、城市简报、处境读板、战术建议和侦察视角派生数据，本轮应复用这些数据。
- `Tools/RenderBattlePreview/main.swift` 在三个尺寸分别渲染单位与城市场景，并用确定性结构断言和像素采样验证战斗 UI。
- `.github/workflows/ci-results.yml` 负责在 `main` push 后运行全部结构检查、SwiftPM、Gameplay Smoke、RenderBattlePreview 和 Xcode build，并上传未加密结果包。

## 实现步骤

1. 新增纯布局值对象 `BattleInterfaceMetrics`，由容器尺寸稳定派生顶部高度、底部高度、地图间距、边缘按钮尺寸与间距，供 SwiftUI 和 RenderBattlePreview 共用。
2. 在 `BattleView` 使用同一 metrics 布局顶部、地图、命令坞、边缘工具和抽屉，扩大地图有效面积。
3. 收敛 `TopBarView`：紧凑尺寸隐藏次级长文案，突出罗马/回合和资源；结束回合仍使用原命令和禁用条件。
4. 重构 `BattleEdgeToolsView`：去掉包裹五按钮的厚重整块背景，每个入口独立贴边，并保持无障碍文案和激活反馈。
5. 重构 `SelectionCommandDockView`、单位/城市命令按钮：显示选中对象身份与当前目标 cue；攻击/技能/扩建等主要命令优先，休整/跳过/更多等次要命令降权但仍可达。
6. 调整 `MapOverlayPresentation` 和默认六边形边界透明度，只改变显示优先级，不改变任何 overlay 数据来源。
7. 扩展 RenderBattlePreview：断言三种尺寸布局上限、非焦点 overlay 降权策略、边缘工具仍可见、命令坞层级和单位/城市视觉差异。
8. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`md/prompt/README.md`、`update_log.md` 和 workflow artifact 版本。

## 验收标准

1. 三种尺寸中地图仍是第一视觉主体，顶部和底部占屏比例均低于 v0.54 基线。
2. 五个抽屉入口全部存在、可辨、可点击，但不再形成厚重的贯穿式右侧工具条。
3. 选中单位时能按顺序读出身份、目标/下一步、攻击或技能等主要命令；选中城市时能读出城市身份、经营目标和扩建/招募命令。
4. 非当前侦察视角的敌路/战术路线明显降权；默认格边界不压过地貌、城市、单位和当前命令叠层。
5. 所有原有 action、disabled 条件和 accessibility label 保持有效，核心状态和 AI 行为不变。
6. 最新 `origin/main` commit 的 GitHub Actions 与 artifact 经 Agent C 核对通过，六张预览图没有截断、重叠、空白或命令区不可读。

## 风险与禁止项

- 风险：缩小命令坞可能导致 390x844 或 932x430 的命令文字截断；必须使用稳定尺寸、ViewThatFits/紧凑布局和云端截图复判。
- 风险：像素门禁坐标依赖布局常量；必须统一读取 `BattleInterfaceMetrics`，避免测试与 UI 两套魔数。
- 禁止把复杂规则、目标筛选或评分写入 SwiftUI。
- 禁止为了视觉相似复制参考游戏资产或改项目技术栈。
- 禁止运行本地验证、纳入无关 Xcode 改动、创建 PR/候选分支或伪造云端结果。
