# v0.9 / 将领技能冷却与战功状态可读化

## 版本号和版本分配依据

- 版本号：v0.9。
- 版本标题：将领技能冷却与战功状态可读化。
- 分配依据：`update_log.md` 当前最新正式版本为 v0.8，且 v0.8 已把“技能冷却、升级树”列为后续扩展点。本轮先落地可验证的技能冷却和战功可读化，因此递增为 v0.9。

## 背景

v0.8 已新增 `GeneralSkillPreview`，让将领主动技能的范围、目标、预计恢复和削城防在地图、将领卡和军令按钮中可读。当前缺口是：主动技能没有跨回合冷却，玩家和 AI 可以每回合连续使用；`experience` 已参与伤害计算和兵牌星点展示，但没有军阶、战功进度、伤害加成说明，玩家难以理解经验成长价值。

本轮目标是让将领系统更接近“大征服者罗马”的战略读感：技能释放后进入短冷却，冷却在所属阵营新回合开始时递减；经验转成战功状态，明确军阶、伤害加成和下一阶段进度。

## 目标

1. 在核心层为将领主动技能加入跨回合冷却状态，释放后进入冷却，冷却随所属阵营开始行动回合递减。
2. 扩展 `GeneralSkillPreview`，暴露冷却剩余、冷却文案和不可用原因；冷却时核心层阻止再次释放。
3. 新增 `WarMeritStatus` 或等价只读模型，把 `ArmyUnit.experience` 转成军阶、战功、伤害加成、下一军阶进度和 UI 摘要。
4. 让 `GameViewModel` 暴露选中单位战功状态、技能冷却摘要和按钮 detail，不复制冷却规则或军阶规则。
5. 让 `BattleView` 在完整/紧凑情报、将领卡和技能按钮中展示冷却和战功状态；必要时给兵牌增加轻量冷却徽标，不能遮挡生命条、姿态和将领星标。
6. 让 AI 技能判断和 `.useSkill` 意图复用同一冷却/预览逻辑，且 `aiIntents` 预测不改变原始 `GameState`。
7. 新增 Swift Testing、Gameplay Smoke 和文档同步，CI artifact 版本更新到 v0.9。

## 非目标

- 不新增将领、技能、升级树、手动技能目标或新战斗数值体系。
- 不改变现有技能范围、恢复量、削城防量、行动消耗或经验伤害公式。
- 不做大规模 `BattleView.swift` 拆分，不引入第三方框架、图片素材、网络服务或存档 UI。
- 不创建 PR，不引入 `develop`、`smalldata_test`、`codeb/...` 或其他长期/候选分支。

## 当前架构依据

- `Sources/RomeLegionsCore/GameState.swift`
  - `ArmyUnit` 目前有 `experience`、`generalName`、`generalTrait`、`hasMoved`、`hasActed`，没有冷却字段；新增字段必须处理旧存档 Codable 默认值。
  - `GeneralSkillPreview` 是 v0.8 建立的只读技能预览入口，应继续承载冷却可用性，避免 UI 自己判断核心规则。
  - `useGeneralSkill(unitID:)` 已复用预览目标筛选，v0.9 必须继续保持预览和结算同源。
  - `endTurn()` 当前负责新 active faction 的行动重置，适合抽出“所属阵营开始回合”刷新 helper。
  - `aiIntents(for:limit:)` 当前使用 forecast copy 做只读预测；v0.9 需要在 forecast copy 上复用同一回合刷新 helper，不能改变原始 state。
- `RomeLegionsApp/App/GameViewModel.swift`
  - 负责 UI 友好的选择态和派生数据，适合暴露战功状态和技能冷却摘要。
- `RomeLegionsApp/Views/BattleView.swift`
  - 已有技能范围/目标叠层、将领卡、技能按钮和经验星点；本轮在这些入口补冷却和战功信息。
- `Tests/RomeLegionsCoreTests/GameStateTests.swift` 与 `Tools/GameplaySmoke/main.swift`
  - 已覆盖技能预览与释放一致性，需要新增冷却、战功和旧数据兼容基线。

## 实现步骤

1. 开始前读取本提示词、`AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`README.md`、`md/prompt/README.md`。
2. 同步仓库：

   ```sh
   git fetch origin
   git switch main
   git pull --ff-only origin main
   git status --short --branch
   ```

3. 在核心层新增 `GeneralTrait.skillCooldownTurns`，本轮建议统一为 2。
4. 在 `ArmyUnit` 新增 `generalSkillCooldownRemaining`，并实现 Codable 兼容：旧 JSON 缺字段时默认 0。
5. 新增 `GameRuleError.generalSkillOnCooldown` 或等价错误；`GeneralSkillPreview` 增加 `cooldownRemaining`、`cooldownText`，冷却时 `isExecutable == false`。
6. 抽出回合开始刷新 helper，同时重置行动、清空战术姿态并递减所属阵营单位冷却；`endTurn()` 和 `aiIntents(for:limit:)` 的 forecast 都复用该 helper。
7. `useGeneralSkill(unitID:)` 成功后设置冷却并继续消耗行动；冷却不得被移动、攻击、训练、休整、跳过或非所属阵营回合意外清零。
8. 新增 `WarMeritStatus` 或等价只读模型，至少表达经验值、军阶名、伤害加成 `experience * 3`、下一军阶进度和 UI 摘要。
9. `GameViewModel` 暴露 `selectedWarMeritStatus`、技能冷却摘要和按钮 detail。
10. `BattleView` 更新完整/紧凑情报、将领卡和技能按钮；UI 只读 ViewModel/核心派生数据，不复制冷却、军阶或可用性规则。
11. 更新测试、Smoke、README、flow、flowchart、test、update_log 和 workflow `CI_VERSION`。

## 验收标准

- 技能释放后进入冷却，冷却只在所属阵营开始行动回合递减。
- 冷却状态由核心层保护，不依赖 UI 禁用。
- 技能预览、释放和 AI 判断共享冷却规则。
- `aiIntents(for:limit:)` 不改变原始 `GameState`。
- 旧 `ArmyUnit` JSON 可解码，新增冷却字段默认 0。
- 战功状态可读展示经验、军阶、伤害加成和进度，不改变现有伤害公式。
- 三尺寸战斗页预览无明显裁切或重叠。
- 本地检查通过，`main` push 后 GitHub Actions 最新 run 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止新增冷却字段导致旧存档解码失败。
- 禁止冷却按每个敌方回合重复递减。
- 禁止让 `aiIntents` 为预测而改变原始 `GameState`。
- 禁止让技能预览和释放逻辑分叉。
- 禁止用 UI 禁用替代核心规则保护。
- 禁止提交 DerivedData、截图、缓存或旧 artifact 结果。
