# v0.7 / AI 意图与移动后攻击预览一致性

## 版本号和版本分配依据

- 版本号：v0.7。
- 版本标题：AI 意图与移动后攻击预览一致性。
- 分配依据：`update_log.md` 当前最新正式版本为 v0.6。v0.6 聚焦战斗地图 UI 和窄屏显示，本轮推进核心 AI、敌军意图和战斗预览一致性，因此递增为 v0.7。

## 背景

当前核心里 `attack` 已通过 `attackPreview` 结算，保证玩家直接攻击的预览与实际伤害一致。`aiIntents(for:limit:)` 会复制 `GameState` 做只读预测，并重置目标阵营单位行动状态；现有测试已覆盖直接攻击意图不改状态、城市夺取意图、AI 移动后攻击，以及普通 `attackPreview` / `attack` 一致性。

缺口是：AI 意图中的 `.attack` 和 `.advanceAttack` 的 `projectedDamage` 目前由 `estimatedDamage` 推导，真实 AI 在直接攻击或移动后攻击时会通过 `attack` -> `attackPreview` 结算。若将领指挥、友军支援、包夹、守军支援、战术姿态或移动后位置影响伤害，意图预测可能与真实预览/结算出现分叉。

本轮目标是收敛这条规则来源，让 AI 意图展示的预计伤害与规划态 `attackPreview` 一致，并用核心测试锁住直接攻击和移动后攻击两条路径。

## 目标

1. 让 AI 意图中 `.attack` 和 `.advanceAttack` 的 `projectedDamage` 与同一规划状态下的 `attackPreview(...).damage` 一致。
2. 让 AI 攻击评分、意图预测和真实结算尽量共享同一套战斗预览来源，避免展示与行动分叉。
3. 保持 `aiIntents(for:limit:)` 只读，不改变原始 `GameState`、单位位置、行动标记、战术姿态、资源或 event log。
4. 新增核心测试覆盖直接攻击意图和移动后攻击意图的 projectedDamage，确认真实 `performSimpleAI` 移动后攻击造成的伤害等于意图预测。
5. 更新 Gameplay Smoke、README、flow、flowchart、test、update_log 和 CI artifact 版本记录。

## 非目标

- 不重写 AI 为多回合搜索、蒙特卡洛、行为树或完整战略规划器。
- 不改变 SwiftUI 战斗地图、`GameViewModel` UI 状态或玩家交互语义。
- 不新增第三方框架、图片素材、网络服务或数据文件。
- 不改变 `CombatPreview` 公共模型，除非实现中发现必须新增字段。
- 不把 AI 预测做成会执行 `moveUnit`、`attack`、占城、招募或写 event log 的流程。
- 不删除既有 AI、战斗、任务、胜负保护测试。
- 不创建 PR，不引入 `develop`、`smalldata_test`、`codeb/...` 或其他长期/候选分支。

## 当前架构依据

- `Sources/RomeLegionsCore/GameState.swift`
  - `attackPreview(attackerID:defenderID:)` 是战斗预览入口。
  - `attack(attackerID:defenderID:)` 使用 `attackPreview` 结算伤害和反击。
  - `aiIntents(for:limit:)` 复制 state 做只读预测。
  - `performSimpleAI(for:)` 当前流程是：招募 -> 设置战术姿态 -> 休整/技能 -> 直接攻击 -> 移动 -> 移动后攻击。
  - `aiAttackScore`、`aiPositionScore`、`aiIntent(for:)` 共同决定 AI 攻击目标、移动目的地和敌军意图。
- `Tests/RomeLegionsCoreTests/GameStateTests.swift`
  - 已有 `aiMovesIntoRangeThenAttacks`、`aiIntentForecastPredictsDirectAttackWithoutMutatingState`、`aiIntentForecastPredictsCityCapture`。
  - 需要新增移动后攻击 projectedDamage 与规划态预览一致性测试。
- `Tools/GameplaySmoke/main.swift`
  - 用轻量集成路径覆盖核心玩法；本轮应加入 AI 意图与移动后攻击预览一致性断言。

3. 在 `GameState.swift` 中新增私有规划预览 helper。建议思路：
   - 复制当前 `GameState`。
   - 将复制态的 `activeFaction` 设置为规划 attacker 的 faction。
   - 用规划 attacker 替换同 id 单位，确保位置、战术姿态、`hasMoved`、`hasActed` 符合预览需要。
   - 调用复制态的 `attackPreview(attackerID:defenderID:)`。
   - helper 不得调用 `moveUnit`、`attack` 或任何会写 event log / 资源 / 城市归属的 mutating 命令。
4. 用规划预览 helper 替换 AI 意图中 `.attack` 和 `.advanceAttack` 的裸 `estimatedDamage`。
5. 让 `aiAttackScore` 优先使用规划预览 damage 评分；若 preview 因非法目标失败，再安全退回或给出极低分，保持 deterministic 排序。
6. 保持 `bestAIDestination(for:)`、`aiPositionScore`、`preferredAITacticalOrder` 的既有目标偏好和排序稳定性，不做大规模 AI 重写。
7. 新增或调整核心测试：
   - 直接攻击意图 `projectedDamage` 等于同姿态规划态 `attackPreview.damage`。
   - 移动后 `.advanceAttack` 意图的 `projectedDamage` 等于把单位放到 `intent.destination` 后的 `attackPreview.damage`。
   - `performSimpleAI` 在同一移动后攻击场景造成的真实生命变化等于意图 `projectedDamage`。
   - 测试场景应覆盖移动后位置影响支援、包夹、指挥或守军支援之一，防止 helper 仍读取旧位置。
   - `aiIntents` 调用前后原始 state 不变。
8. 更新 `Tools/GameplaySmoke/main.swift`，加入轻量移动后攻击意图/预览一致性断言。
9. 文档更新：
   - `README.md`：已实现能力补充 AI 意图 projectedDamage 与战斗预览一致。
   - `md/flow/flow.md` / `md/flow/flowchart.md`：在战斗与敌军意图流中说明 AI 意图预计伤害来自规划态预览。
   - `md/test/test.md`：更新 Swift Testing 用例数量和 Gameplay Smoke 覆盖点。
   - `update_log.md`：追加 v0.7 记录，写清核心变更、关键文件、验证结果和遗留事项。
   - `.github/workflows/ci-results.yml`：同步 `CI_VERSION` 到 v0.7，并同步 `md/test/test.md` 中 artifact 示例。
10. 不需要修改 `BattleView.swift`、`GameViewModel.swift`、存档结构或 Xcode project，除非编译错误证明必须调整。

## 验收标准

本轮通过必须满足：

- `aiIntents(for:limit:)` 对原始 `GameState` 只读，不改变 active faction、单位位置、行动标记、战术姿态或 event log。
- `.attack` 意图的 `projectedDamage` 等于同规划态 `attackPreview.damage`。
- `.advanceAttack` 意图的 `projectedDamage` 等于把单位放到意图 destination 后的规划态 `attackPreview.damage`。
- 同一移动后攻击场景中，`performSimpleAI` 实际造成的伤害等于意图预测伤害。
- 新增测试进入 Swift Testing 基线，Gameplay Smoke 包含轻量断言。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止让 AI 意图预测改变 `GameState`。
- 禁止让 `projectedDamage` 与实际 `attackPreview` 继续使用两套规则。
- 禁止为了修测试削弱战斗修正、将领、支援、包夹、守军支援或战术姿态。
- 禁止把核心规则写进 SwiftUI 或 `GameViewModel`。
- 禁止删除既有 AI、战斗、任务、胜负保护测试。
- 禁止用 UI 测试替代核心规则测试。
- 禁止把旧 artifact、旧 output 或 checkout 自带报告冒充 v0.7 云端结果。
