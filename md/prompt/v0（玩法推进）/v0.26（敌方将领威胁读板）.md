# v0.26（敌方将领威胁读板）

## 版本号

v0.26

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.25（军团成长优先级读板）。Agent X 总目标要求继续向“大征服者罗马”推进，持续优化战斗界面 UI、地图、AI、将领等体验。v0.25 已让玩家从我方视角看到军团成长优先级，但敌军将领威胁仍分散在敌军意图、AI 作战计划、热区和技能预览中，玩家无法一眼判断“哪个敌方将领最危险、危险来自技能还是集火、会影响哪个我方目标”。因此本轮递增为 v0.26，围绕“敌方将领威胁读板”推进敌情、AI 和将领可读性。

## 背景

当前 `GameState.aiIntents(for:limit:)` 能预测敌军攻击、接敌、夺城、固守、整备和技能意图；`aiOperationalPlanReports(against:perFactionLimit:limit:)` 能聚合敌军意图、战线压力、热区和敌方将领技能机会；`generalSkillPreview(unitID:)` 能只读预览将领技能范围、目标、恢复、削城防和冷却。

短板是这些信息没有被聚合成“敌方将领威胁”视角。玩家需要在敌情列表、计划卡、热区和地图上来回推断敌方将领的威胁优先级。更接近策略战棋的做法，是在战局层给出敌方将领威胁排行：敌方将领是谁、特性和技能是什么、是否可用、关联哪条敌军意图或作战计划、预计伤害/恢复/削城防/热区/压力如何影响罗马目标。读板必须只读，不改变真实 AI 行为。

## 目标

1. 在 `GameState.swift` 新增只读敌方将领威胁报告：
   - `EnemyCommanderThreatLevel`
   - `EnemyCommanderThreatReport`
   - `enemyCommanderThreatReports(against:limit:)`
   - `enemyCommanderThreatReport(unitID:against:)`
2. 报告至少包含：
   - unitID、faction、position、generalName、generalTrait
   - threatLevel、score
   - intentKind、targetPosition、targetUnitID、targetCityID、destination
   - projectedDamage、projectedRecovery、projectedFortificationReduction
   - skillReady、skillSummary、skillBlockedReason
   - pressureLevel、threatHeatLevel
   - title、summary、detail、reasons、impact
3. 核心报告必须复用现有只读来源：
   - `aiPlanningForecast(for:)`
   - `aiIntentReports(for:limit:)` / `AIIntent`
   - `aiOperationalPlanReports(against:perFactionLimit:limit:)`
   - `generalSkillPreview(for:)`
   - `frontlinePressureReports(against:perFactionLimit:limit:)`
   - `threatHeatZoneReports(for:limit:)`
   - `legionFormationReport(for:)`
4. 评分应解释清楚，至少考虑：
   - 将领技能是否就绪或冷却中。
   - 技能预计恢复或削城防。
   - 直接/接敌攻击预计伤害。
   - 目标是否处在战线压力或威胁热区。
   - AI 作战计划中是否由该将领作为 commander 或主攻参与。
   - 将领 trait 的被动攻防/机动/攻城价值。
5. 在 `GameViewModel.swift` 新增 UI 摘要：
   - `EnemyCommanderThreatSummary`
   - `enemyCommanderThreatSummaries`
   - `primaryEnemyCommanderThreatSummary`
   - 摘要需包含 title、commanderLabel、traitLabel、levelLabel、intentLabel、targetLabel、impactLabel、statusLabel、scoreLabel、accessibilityLabel。
6. 在 `BattleView.swift` 新增敌方将领威胁展示：
   - 顶部态势 chip 显示首要敌将威胁。
   - 完整战局面板展示敌将威胁行。
   - 敌情面板展示首要敌将威胁或在敌军计划附近补充威胁说明。
   - SwiftUI 只能展示 ViewModel 摘要，不得重新计算威胁分。
7. `Tools/RenderBattlePreview/main.swift` 增加云端断言：
   - `primaryEnemyCommanderThreatSummary` 存在。
   - `enemyCommanderThreatSummaries` 非空。
   - 摘要 title、commanderLabel、traitLabel、levelLabel、intentLabel、impactLabel、statusLabel、accessibilityLabel 非空。
   - 至少一个摘要关联 `.useSkill` 或包含可读技能摘要。
   - 失败抛出 `missingEnemyCommanderThreatSummary`。
8. Swift Testing 与 Gameplay Smoke 增加核心只读断言：
   - 敌方将领威胁报告不改变 `GameState`。
   - 可发动技能的敌将应高于纯整备/冷却敌将。
   - 攻城专家的削城防影响来自 `GeneralSkillPreview`。
   - 敌将攻击威胁的预计伤害来自 `AIIntent` / `attackPreview` 链路。
   - 条约保护或非交战阵营不应进入罗马敌将威胁。
9. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.26`。
10. 更新 README、flow、flowchart、test、prompt README、update_log，并保留 v0.25 最新云端通过证据。

## 非目标

- 不改变真实 AI 行动、AI 权重、AI 执行顺序或技能释放规则。
- 不新增多回合搜索、自动反制建议、敌将装备、升级树、将领池 UI、将领改名或技能选择。
- 不改变将领 trait 数值、技能范围、冷却、恢复、削城防、战斗伤害或城市规则。
- 不新增第三方框架，不拆分大型 SwiftUI 文件，不做 BattleView 全局重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GeneralTrait` 已提供被动修正、技能名、技能效果、范围、恢复、削城防和冷却。
- `generalSkillPreview(for:)` 已是技能影响和阻塞原因的核心只读来源。
- `AIIntent` 已公开 intent kind、目标、目的地、预计伤害和威胁分。
- `AIOperationalPlanReport` 已把敌军意图、将领技能、压力和热区聚合为作战计划，可作为敌方将领威胁的上下文。
- `FrontlinePressureReport` 与 `ThreatHeatZoneReport` 已提供罗马目标压力和热区等级。
- `GameViewModel` 已有敌情、计划、热区、战线、将令、成长等 UI 摘要范式。
- `BattleView` 顶部 chip、完整战局面板、敌情面板是本轮 UI 落点。

## 实现步骤

1. 在 `GameState.swift` 的 AI 作战计划类型附近新增敌方将领威胁 level 和 report 类型。
2. 实现 `enemyCommanderThreatReports(against:limit:)`：
   - 遍历交战敌方阵营中带将领 trait 的单位。
   - 对每个阵营使用 forecast，复用 `aiIntentReports`、`generalSkillPreview(for:)`、`aiOperationalPlanReports`、压力和热区报告。
   - 生成每名敌将的威胁报告，按 threatLevel、score、unitID 稳定排序并截断 limit。
3. 实现 `enemyCommanderThreatReport(unitID:against:)`：
   - 返回指定敌将报告。
   - 缺单位、无将领或非交战敌方时抛出合适错误或返回 nil，按现有错误风格处理。
4. 威胁评分建议：
   - 可发动 `.useSkill` 意图和技能就绪加高分。
   - `projectedDamage`、`projectedRecoveredHealth`、`projectedFortificationReduction` 进入分数。
   - 关联高压力/高热区/AI 作战计划 commander 的报告加分。
   - trait 被动攻防/机动/攻城价值进入理由，但不得改变结算。
5. 在 `GameViewModel.swift` 增加 `EnemyCommanderThreatSummary`，把 report 转成中文 UI 字段和无障碍文案。
6. 在 `BattleView.swift` 加入敌将 chip、战局面板威胁行、敌情面板补充卡，沿用现有紧凑信息密度。
7. 更新 `Tools/GameplaySmoke/main.swift`、`Tests/RomeLegionsCoreTests/GameStateTests.swift` 和 `Tools/RenderBattlePreview/main.swift`。
8. 更新 workflow 版本和文档。
9. 完成后只做只读 `git diff` / `git status` 检查，提交并 push `origin/main`，等待 GitHub Actions 和 Agent C artifact 复判。
