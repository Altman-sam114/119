# v0.33（战场目标线地图叠层）

## 版本号

v0.33

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.32（战场目标链路）。v0.32 已把战场焦点、将领协同、机动落点和本方军议串成“1 焦点、2 将令、3 机动、4 军议”的只读目标线，但这条目标线主要在卡片和文本中展示，地图上还不能直接读出这些阶段之间的空间关系。下一步应把这条已有链路地图化为只读叠层，递增为 v0.33。

## 背景

当前项目已有：

- `BattleObjectiveChainSummary`：组合 `BattlefieldFocusSummary`、`CommanderSynergySummary`、`ManeuverOptionSummary` 和 `TacticalRecommendationSummary`，输出阶段标签、链路摘要和无障碍文案。
- 地图路线叠层：敌军意图路线、本方军议路径、反制路线都已经使用 `RouteSegment` + SwiftUI layer 的模式展示。
- 地图位置叠层：机动落点、反制“1 回应/2 落点/3 目标”、敌军目的地/目标都通过 ViewModel 派生数据给 `BattleView` 展示。
- 主动地图图例：`MapOverlayLegendKind` 和 `activeMapOverlayLegendItems` 只读汇总当前实际存在的叠层。

当前短板是：玩家能在战场面板读到目标链路，但地图上无法直接看出“焦点目标、将令目标、机动落点、军议目标”之间的关系。v0.33 应只在 ViewModel 和 SwiftUI 层把 v0.32 目标链路派生为地图线段和阶段标记，不改变任何核心评分、排序、移动、攻击、AI 或技能结算。

## 目标

1. 在 `GameViewModel.swift` 增加只读战场目标线地图叠层，建议命名为：
   - `BattleObjectiveRouteSegment`
   - `BattleObjectiveMapRole`
   - `BattleObjectivePositionOverlay`
   - `BattleObjectiveMapOverlay`
   - `primaryBattleObjectiveMapOverlay`
   - `battleObjectiveRouteSegments`
   - `battleObjectiveOverlaysByPosition`
   - `battleObjectiveOverlayPositions`
2. 地图叠层至少覆盖四个阶段：
   - `1 焦点`：来自 `BattleObjectiveChainSummary.focus.targetPosition`。
   - `2 将令`：优先来自 `synergy?.targetPosition`，没有将令时可不生成该阶段位置标记。
   - `3 机动`：优先来自 `maneuver?.destination`，并可用 `maneuver?.targetPosition` 作为路线目标参考。
   - `4 军议`：优先来自 `recommendation?.targetPosition`，没有军议时可不生成该阶段位置标记。
3. 路线线段必须由已有链路摘要派生：
   - 按阶段顺序连接存在且位置不同的阶段点。
   - 若全部阶段都在同一格，也应至少保留一个可读的位置 overlay，避免图例和断言失效。
   - 不声明为真实移动路径，不参与可达、攻击、技能或 AI 决策。
4. 在 `BattleView.swift` 展示目标线地图叠层：
   - 在地图路线层增加目标线 layer，位置应低于按钮交互层，不影响点击。
   - 在 `HexTileView` 增加目标线位置 overlay，显示阶段序号和合适图标。
   - tile accessibility 应能读出目标线阶段、坐标和链路摘要。
5. 更新地图图例：
   - `MapOverlayLegendKind` 增加目标线类型，例如 `.battleObjective`。
   - `activeMapOverlayLegendItems` 在目标线 overlay 存在时加入图例项。
   - `MapOverlayLegendKind.legendTint` 增加对应颜色。
6. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 断言 `primaryBattleObjectiveMapOverlay` 存在。
   - 断言 `battleObjectiveOverlayPositions` 和 `battleObjectiveOverlaysByPosition` 非空。
   - 断言 overlay 引用当前 `primaryBattleObjectiveChainSummary`。
   - 断言焦点阶段位置存在；若链路有机动/军议/将令，也断言对应阶段可被 overlay 引用。
   - 断言 route segments、chain label、accessibility label、stage labels 非空。
   - 断言图例包含新目标线 kind。
   - 失败建议新增 `missingBattleObjectiveMapOverlay`，便于 Agent C 定位。
7. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.33`。
8. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.32 最新云端通过证据。

## 非目标

- 不新增核心规则字段，不修改 `GameState` 的评分、排序、路径、AI 或报告生成逻辑。
- 不把目标线变成自动执行、半自动执行、一键移动、一键攻击、一键技能或命令队列。
- 不改变真实移动、攻击、技能、姿态、AI、城市、外交、胜负结算。
- 不新增多回合搜索、装备、升级树、将领池 UI、外交界面、存档 UI 或建筑树。
- 不引入第三方框架，不做大型 SwiftUI 文件拆分或全局重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- v0.32 的 `BattleObjectiveChainSummary` 已经保证目标线引用当前焦点、将令、机动和军议摘要，适合作为地图 overlay 的唯一来源。
- `CountermeasureMapOverlay` 和 `CountermeasureTileOverlay` 已提供阶段标记、路线线段、无障碍文案和图例接入的本地模式，可作为实现参考。
- `TacticalRecommendationRouteLayerView` 和 `EnemyIntentRouteLayerView` 已提供 SwiftUI route layer 的绘制方式，可复用其轻量 Path 绘制思路。
- `WarMapView` 已集中读取 ViewModel 的 overlay 字典、路径和图例，是新增目标线地图叠层的主入口。
- `Tools/RenderBattlePreview/main.swift` 已断言 v0.32 目标链路和地图图例，适合扩展为 v0.33 的目标线地图叠层断言。

## 实现步骤

1. 在 `GameViewModel.swift` 增加目标线叠层数据结构：
   - `BattleObjectiveRouteSegment`：包含 `id`、`from`、`to`、`fromRole`、`toRole`、`isTargetLeg`。
   - `BattleObjectiveMapRole`：覆盖 `focus`、`synergy`、`maneuver`、`recommendation`，提供 `stageNumber`、`displayName`、`stageLabel`。
   - `BattleObjectivePositionOverlay`：包含 `chain`、`role`、`position`、`focusLabel`、`chainLabel`、`accessibilityLabel`。
   - `BattleObjectiveMapOverlay`：由 `BattleObjectiveChainSummary` 派生 `positionOverlays`、`routeSegments` 和 `accessibilityLabel`。
2. 在 `GameViewModel` 增加：
   - `primaryBattleObjectiveMapOverlay`
   - `battleObjectiveRouteSegments`
   - `battleObjectiveOverlaysByPosition`
   - `battleObjectiveOverlayPositions`
3. 更新 `MapOverlayLegendKind` 和 `activeMapOverlayLegendItems`，仅在目标线 overlay 存在时加入图例。
4. 在 `BattleView.swift`：
   - `WarMapView` 读取 `primaryBattleObjectiveMapOverlay` 和 `battleObjectiveOverlaysByPosition`。
   - 增加 `BattleObjectiveRouteLayerView` / `BattleObjectiveRouteSegmentView`。
   - `HexTileView` 增加 `battleObjectiveOverlay` 参数，并在 tile 上绘制 `BattleObjectiveTileOverlay`。
   - 保持叠层不接管 hit testing，不改变按钮 action 或 disabled 规则。
5. 在 `Tools/RenderBattlePreview/main.swift`：
   - 在 v0.32 目标链路断言之后增加地图 overlay 断言。
   - 新增 `missingBattleObjectiveMapOverlay` 错误枚举 case。
   - 扩展图例 required kind，要求包含新目标线 kind。
6. 更新文档和 workflow 版本号。
7. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

- 地图能显示战场目标线阶段标记，玩家可直接读出焦点、将令、机动和军议的空间关系。
- 地图路线线段由 v0.32 目标链路派生，且 overlay 能引用当前 `primaryBattleObjectiveChainSummary`。
- 主动地图图例能解释目标线叠层。
- tile accessibility 能读出阶段、坐标和链路摘要。
- SwiftUI 不重新计算评分、路径、目标、技能或移动规则，只消费 ViewModel 派生字段。
- `GameState` 核心规则、AI、技能、移动、攻击、姿态和胜负结算不变。
- 云端 RenderBattlePreview 断言覆盖目标线地图 overlay、位置集合、路线线段、图例和无障碍文案。
