# v0.45（选中军团军令窗口读板）

## 版本号

v0.45

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.44（将领战机威胁桥接读板），并已通过 `origin/main` 最新云端结果包复判。v0.44 已把选中将领的技能机会、敌将威胁、反制入口、目标线阶段和敌情闭环压缩进将领卡。下一步不继续堆将领卡，而是面向选中军团新增“军令窗口”短读板，把处境入口、反制指令、目标线阶段、将领战机、军议、机动、姿态和敌情闭环压成“先做什么、为什么、下一步接哪个入口”的行动顺序提示。递增为 v0.45。

## 背景

当前项目已有：

- `SelectedUnitSituationReadout`：选中军团压力、空间、机会、入口列表和命令入口 cue。
- `CountermeasureCommandPreview`：反制推荐姿态、落点、目标、下一步和阻塞原因。
- `BattleObjectiveStageCommandPreview`：目标线阶段的命令入口、推荐姿态、目标、技能和下一步。
- `SelectedCommanderOpportunityBridgeReadout`：选中将领的战机、敌将威胁、反制入口和敌情闭环。
- `TacticalRecommendationSummary`、`ManeuverOptionSummary`、`SelectedTacticalOrderPreview`：军议、机动落点和姿态预览。
- `EnemyEngagementLoopReadout` 与 `BattlefieldConvergenceSummary`：敌情闭环与战场态势交汇。

这些读板已经各自成立，但玩家选中军团后仍需要从处境卡、军令按钮、目标线、反制、将领卡和地图 HUD 中拼接行动顺序。v0.45 应新增一个只读军令窗口读板，直接告诉玩家当前军团优先看哪个入口、建议姿态/落点/打击窗口是什么。

## 目标

1. 在 `GameViewModel.swift` 新增选中军团军令窗口只读派生：
   - 新增 `SelectedUnitOrderWindowStepKind`、`SelectedUnitOrderWindowStep`、`SelectedUnitOrderWindowReadout`。
   - 新增 `selectedUnitOrderWindowReadout` computed property。
   - 只读取既有 ViewModel 派生数据：`selectedUnitSituationReadout`、`selectedCountermeasureCommandPreview`、`selectedBattleObjectiveStageCommandPreview`、`selectedCommanderOpportunityBridgeReadout`、`selectedCommanderChainReadout`、`selectedCommanderActionGuidance`、`selectedTacticalOrderPreviews`、`selectedTacticalRecommendationSummary`、`primaryManeuverOptionSummary`、`primaryEnemyEngagementLoopReadout`、`primaryBattlefieldConvergenceSummary`。
   - 输出字段建议包含 `unitID`、`title`、`statusLabel`、`openingLabel`、`postureLabel`、`movementLabel`、`strikeLabel`、`commanderLabel`、`counterLabel`、`nextStepLabel`、`riskLabel`、`compactLabel`、`steps`、来源 ID、`accessibilityLabel`。
   - 提供 references 方法供 Render 断言核对同源。
2. 在 `BattleView.swift` 的选中单位情报面板展示：
   - 在完整/紧凑选中单位情报中，放在 `SelectedUnitSituationReadoutView` 之后、军团编制/成长/军议卡之前。
   - 紧凑版只显示一行行动窗口和 2-3 个短 step。
   - 完整版显示开局、姿态、机动/打击、反制/将令和下一步；不要新增按钮。
   - SwiftUI 只展示 ViewModel 字段，不重新计算入口优先级、姿态、目标线或反制。
3. 在 `Tools/RenderBattlePreview/main.swift` 更新云端断言：
   - 新增错误 case：`missingSelectedUnitOrderWindowReadout`。
   - 在 v0.44 `missingCommanderOpportunityBridgeReadout` 断言之后、目标线地图叠层断言之前新增断言。
   - 断言固定单位场景 `rome-legion-1` 的军令窗口引用处境读板、反制指令、目标线阶段、将领战机桥接、军议、机动、敌情闭环和战场态势交汇。
   - 断言标题、状态、开局、姿态、机动、打击、将令、反制、下一步、风险、compact、steps 和无障碍文案非空。
   - 断言读取读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.45`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.44 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、AI 权重、反制评分、目标线评分、将领技能规则或姿态规则。
- 不新增命令队列、自动技能、自动反制、自动移动、自动攻击、一键连招或多回合搜索。
- 不新增地图叠层规则；本轮只解释既有入口和下一步。
- 不重做 BattleView 总布局，不引入第三方框架，不引入新素材。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameViewModel` 是 UI 派生数据边界，v0.40-v0.44 已经通过只读 readout 串联处境、指挥链、敌情和战机。
- `SelectedUnitSituationReadout.commandEntries` 已提供入口列表；v0.45 不替代它，而是将入口压成行动顺序窗口。
- `CountermeasureCommandPreview`、`BattleObjectiveStageCommandPreview`、`CommanderOpportunityBridgeReadout`、`EnemyEngagementLoopReadout` 和 `BattlefieldConvergenceSummary` 已提供同源 ID 和 references，适合 Render 做稳定断言。
- 选中单位情报面板是最小 UI 落点，避免继续加重地图 HUD 或将领卡。
