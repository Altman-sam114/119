# v0.34（战场目标线阶段聚焦）

## 版本号

v0.34

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.33（战场目标线地图叠层）。v0.33 已把“1 焦点、2 将令、3 机动、4 军议”目标线展示到地图阶段标记和金色连线，但玩家仍只能观看叠层，不能从目标线卡片快速定位各阶段对应的地块或单位。下一步应把目标线阶段转成只读定位入口，递增为 v0.34。

## 背景

当前项目已有：

- `BattleObjectiveChainSummary`：组合战场焦点、将领协同、机动落点和本方军议，输出目标线阶段文案。
- `BattleObjectiveMapOverlay`：把同一目标线派生为地图阶段标记和路线线段。
- `focusCountermeasure(_:)`：反制建议已有“只改变 ViewModel 选择态、位置、聚焦 ID 和 banner，不移动、不攻击、不切姿态”的聚焦模式。
- `Tools/RenderBattlePreview/main.swift`：已经断言反制聚焦行为和 v0.33 目标线地图叠层。

当前短板是：目标线阶段虽然在地图上可见，但卡片没有“定位 1 焦点 / 2 将令 / 3 机动 / 4 军议”的明确入口；玩家需要自己在地图上寻找阶段标记。v0.34 应复用现有 ViewModel 选择态，为目标线阶段增加只读定位，不改变核心规则。

## 目标

1. 在 `GameViewModel.swift` 增加战场目标线阶段聚焦能力：
   - 建议新增 `@Published var focusedBattleObjectiveRole: BattleObjectiveMapRole?`。
   - 建议新增 `focusedBattleObjectiveOverlay` 或同等只读派生，用于识别当前被定位的目标线阶段。
   - 建议新增 `focusPrimaryBattleObjectiveStage(_ role: BattleObjectiveMapRole)`，从 `primaryBattleObjectiveMapOverlay.positionOverlays` 查找阶段并更新选择态。
2. 聚焦行为必须只读：
   - 若阶段对应地块有罗马单位，优先选中该单位，并同步驻守城市。
   - 若阶段对应地块只有城市，选中城市。
   - 若阶段对应地块无单位/城市，则只更新 `selectedPosition`。
   - 更新 `focusedBattleObjectiveRole` 和 `bannerMessage`，banner 应说明阶段、位置和目标线链路。
   - 不调用 `GameState` mutating 方法，不移动、不攻击、不发动技能、不切换姿态、不结束回合。
3. 在 `BattleView.swift` 为目标线卡片增加阶段定位入口：
   - `BattleObjectiveChainCardView` 应接收可选 `focusStageAction` 闭包。
   - 展示 1 到 4 阶段的紧凑按钮或分段式按钮，按钮标签使用阶段序号和名称。
   - 阶段不存在时应禁用或不显示对应按钮，不伪造目标。
   - 按钮只调用 ViewModel 聚焦方法，不改变现有地图 tile 点击、攻击按钮、姿态按钮的 action 或 disabled 规则。
   - 增强无障碍文案，使按钮可读出阶段、目标和位置。
4. 如实现代价低，可在地图阶段徽标上为当前聚焦阶段增加轻量视觉高亮，但不得接管 tile 点击或扩大布局。
5. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingBattleObjectiveStageFocus`。
   - 在 v0.33 目标线 overlay 断言后，调用目标线阶段聚焦方法。
   - 断言聚焦焦点阶段后 `focusedPosition` 等于焦点位置、`focusedBattleObjectiveRole == .focus`、banner 包含“目标线”或阶段文案。
   - 断言聚焦机动阶段后 `focusedPosition` 等于机动落点，且不会让 `GameState` 中相关单位位置、生命、行动状态发生变化。
   - 断言聚焦军议或将令阶段时，如果对应位置有单位/城市，选择态与该位置一致。
6. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.34`。
7. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.33 云端通过证据。

## 非目标

- 不新增核心规则字段，不修改 `GameState` 的评分、排序、路径、AI 或报告生成逻辑。
- 不把目标线变成自动执行、半自动执行、一键移动、一键攻击、一键技能、一键切姿态或命令队列。
- 不改变真实移动、攻击、技能、姿态、AI、城市、外交、胜负结算。
- 不新增多回合搜索、装备、升级树、将领池 UI、外交界面、存档 UI 或建筑树。
- 不引入第三方框架，不做大型 SwiftUI 文件拆分或全局重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameViewModel.focusedPosition` 已统一把 `selectedPosition`、选中单位位置和选中城市位置转成地图焦点，适合作为阶段定位后的观察点。
- `focusCountermeasure(_:)` 已证明只读聚焦可以只改 ViewModel 选择态和 banner，不需要触碰核心规则。
- `BattleObjectivePositionOverlay` 已包含 role、position、stageLabel、focusLabel、chainLabel 和 accessibilityLabel，可作为聚焦入口的数据源。
- `BattleObjectiveChainCardView` 是目标线卡片的集中展示点，最适合承载阶段定位按钮。
- `Tools/RenderBattlePreview/main.swift` 目前已经在同一场景中拿到焦点、将令、机动、军议和目标线 overlay，适合扩展为聚焦行为断言。

## 实现步骤

1. 在 `GameViewModel.swift`：
   - 增加 `focusedBattleObjectiveRole` 并在 `start(mode:)` 清空。
   - 增加 `focusedBattleObjectiveOverlay` 或同等 helper，从 `primaryBattleObjectiveMapOverlay.positionOverlays` 中按 role 查找。
   - 增加 `focusPrimaryBattleObjectiveStage(_:)`，查找 overlay 后更新 `selectedPosition`、`selectedUnitID`、`selectedCityID`、`focusedBattleObjectiveRole` 和 `bannerMessage`。
   - 普通 `selectTile(_:)` 可以清空 `focusedBattleObjectiveRole`，避免手动选格后继续显示旧聚焦。
2. 在 `BattleView.swift`：
   - 找到所有 `BattleObjectiveChainCardView` 调用点，为可交互场景传入 `viewModel.focusPrimaryBattleObjectiveStage`。
   - 更新 `BattleObjectiveChainCardView` 参数，增加阶段按钮行。
   - 阶段按钮从 summary 的阶段数据和 overlay 可用性派生，避免 SwiftUI 重新计算目标。
   - 保持按钮尺寸稳定，移动端不要造成文字溢出。
3. 如需地图高亮：
   - `BattleObjectiveTileOverlay` 或 `BattleObjectiveStageBadge` 可接收当前聚焦 role，并对匹配阶段增加描边或亮度。
   - 不改变 hit testing，不新增地图 gesture。
4. 在 `Tools/RenderBattlePreview/main.swift`：
   - 在 `missingBattleObjectiveMapOverlay` 断言后记录聚焦前关键 `GameState` 快照。
   - 调用 `.focusPrimaryBattleObjectiveStage(.focus)` 和 `.focusPrimaryBattleObjectiveStage(.maneuver)`。
   - 断言选择态、`focusedPosition`、`focusedBattleObjectiveRole` 和 banner。
   - 断言聚焦前后核心单位位置/生命/行动状态不变。
   - 新增 `missingBattleObjectiveStageFocus` 错误 case。
5. 更新 workflow 版本和文档。
6. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 玩家能从目标线卡片快速定位 1 焦点、2 将令、3 机动、4 军议阶段。
- 聚焦后地图焦点、选中单位/城市和 banner 与阶段位置一致。
- 聚焦行为不改变 `GameState` 核心状态，不移动、不攻击、不发动技能、不切换姿态。
- 阶段按钮和无障碍文案可读，移动端布局不溢出。
- SwiftUI 不重新计算评分、路径、目标、技能或移动规则，只消费 ViewModel 派生字段。
- 云端 RenderBattlePreview 断言覆盖目标线阶段聚焦、核心状态不变、目标线地图 overlay 和图例。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。
