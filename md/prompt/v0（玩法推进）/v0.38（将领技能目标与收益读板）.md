# v0.38（将领技能目标与收益读板）

## 版本号

v0.38

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.37（将令技能入口链路提示），并已通过 `origin/main` 最新云端结果包复判。v0.37 已把选中单位将领简报、主动技能预览、当前将令协同和目标线“2 将令”阶段 cue 串成同源技能入口提示。下一步应让玩家更清楚主动技能会影响哪些单位或城市、预计收益多少、地图叠层对应哪里，递增为 v0.38。

## 背景

当前项目已有：

- `GeneralSkillPreview`：核心层只读预览技能范围、影响单位、影响城市、影响位置、预计恢复、预计削城防、冷却、可执行状态和阻塞原因。
- `GameViewModel.selectedGeneralSkillPreview`、`selectedGeneralSkillRangePositions`、`selectedGeneralSkillTargetPositions`、`selectedGeneralSkillTargetUnitIDs`、`selectedGeneralSkillTargetCityIDs`：把技能预览转给地图叠层和按钮。
- `SelectedCommanderBrief`：选中单位将领读板，展示将领名、被动贡献、技能状态、技能效果和战功摘要。
- `CommanderActionGuidance`：v0.37 新增的技能入口链路提示，把将领简报、技能预览、将令协同和目标线阶段 cue 串成同源入口。
- `BattleView` 已在地图上显示技能范围和技能目标叠层，但将领卡仍主要展示“范围、总恢复/削城防、冷却”，缺少“目标是谁、收益落到哪里”的明确读板。

当前短板是：玩家能看到紫色技能范围/目标叠层和技能按钮，但需要自己从地图上推断技能具体会救哪支军团或削哪座城。v0.38 应新增只读的技能目标与收益读板，把核心技能预览的目标集合、收益和地图叠层说明集中展示。

## 目标

1. 在 `GameViewModel.swift` 增加将领技能目标与收益只读派生：
   - 建议新增 `GeneralSkillTargetReadoutTarget` 和 `SelectedGeneralSkillTargetReadout` 或同等轻量结构。
   - 新增 `selectedGeneralSkillTargetReadout`，只读取 `selectedGeneralSkillPreview`、`selectedCommanderActionGuidance`、`state.unit(withID:)`、`state.city(withID:)` 和既有 skill target/range positions。
   - 输出字段建议包含 `title`、`effectLabel`、`targetCountLabel`、`targetLabels`、`mapCueLabel`、`statusLabel`、`targets`、`accessibilityLabel`。
   - 目标文案应区分友军单位和敌方城市；恢复类显示预计恢复总量和目标单位数，攻城类显示预计削城防总量和目标城市数。
   - 目标列表最多展示 2-3 个目标，超出时有“等 N 个”或类似短文案，避免挤爆紧凑 UI。
   - 不新增 `@Published` 状态，不修改 `GameState`，不保留 stale readout。
2. 在 `BattleView.swift` 展示目标读板：
   - `GeneralTraitCardView` 增加 `skillTargetReadout` 参数，在完整将领卡现有技能 pill 附近展示目标、收益和地图 cue。
   - `CompactGeneralTraitView` 增加同一参数，在紧凑将领卡显示一行短读板，例如“目标 1 友军 · 恢复 12 · 地图紫标”。
   - 读板只消费 ViewModel 派生字段，不在 SwiftUI 中重新计算技能目标、收益、范围或可执行状态。
   - 技能按钮 action、`.disabled(...)`、目标线 cue、反制 cue 和地图叠层逻辑不变。
3. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingGeneralSkillTargetReadout`。
   - 在 commander brief / commander action guidance 断言附近检查 `selectedGeneralSkillTargetReadout` 非空。
   - 断言 readout 的目标数与 `GeneralSkillPreview.affectedUnitIDs.count + affectedCityIDs.count` 同源。
   - 断言 readout 的收益文案与 `projectedRecoveredHealth` 或 `projectedFortificationReduction` 同源。
   - 断言 readout 的地图 cue 与 `selectedGeneralSkillTargetPositions` / `affectedPositions` 同源，并且无障碍文案非空。
   - 断言读取 readout 不改变单位位置、生命、行动标记、姿态、技能冷却、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.38`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.37 云端通过证据。

## 非目标

- 不新增真实命令、自动发动技能、一键技能、命令队列、自动战斗或多回合规划。
- 不改变现有技能按钮、攻击按钮、姿态按钮、移动、结束回合、城市或 AI 的 action / disabled 逻辑。
- 不把读板状态写入 `GameState` 或新增持久存档字段。
- 不在 SwiftUI `body` 中重新计算技能目标、技能范围、收益、目标线阶段或可执行状态。
- 不新增装备、升级树、将领池 UI、外交界面、存档 UI、建筑树或第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GeneralSkillPreview` 已提供 `affectedUnitIDs`、`affectedCityIDs`、`affectedPositions`、`projectedRecoveredHealth`、`projectedFortificationReduction`、`summary`、`detail`、`cooldownText` 和 `blockedReason`。
- `GameViewModel` 已有 `selectedGeneralSkillTargetPositions`、`selectedGeneralSkillTargetUnitIDs` 和 `selectedGeneralSkillTargetCityIDs` 供地图叠层使用。
- `SelectedCommanderBrief` 已提供技能名、状态、效果、阻塞原因和无障碍文案。
- `CommanderActionGuidance` 已提供技能入口链路、目标线阶段 cue 和技能按钮 detail 前缀。

## 实现步骤

1. 在 `GameViewModel.swift`：
   - 新增目标读板结构，字段保持 UI 友好、短文案优先、可无障碍朗读。
   - 新增 `selectedGeneralSkillTargetReadout` computed property。

   - readout 的目标数量、收益、地图 cue 必须完全来自 `GeneralSkillPreview` 和既有 skill target positions。
2. 在 `BattleView.swift`：
   - 将 `viewModel.selectedGeneralSkillTargetReadout` 传给 `CompactGeneralTraitView` 和 `GeneralTraitCardView`。
   - 新增一个轻量 `GeneralSkillTargetReadoutView` 或在既有将领卡内展示短行；若新增 View，保持局部小组件，不引入无关抽象。
   - 完整卡展示目标数量、收益、地图 cue 和最多 2-3 个目标标签；紧凑卡只展示单行摘要。
   - 注意短横屏和竖屏文本不要溢出，使用 `lineLimit`、`minimumScaleFactor` 和稳定高度。
3. 在 `Tools/RenderBattlePreview/main.swift`：
   - 在现有 `selectedGeneralSkillPreview` / `selectedCommanderActionGuidance` 断言附近增加 `missingGeneralSkillTargetReadout`。
   - 断言 readout 与 preview 的目标数、收益数、位置集合和状态文案一致。
   - 断言 `accessibilityLabel` 非空。
4. 更新 workflow 版本和文档。
5. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 选中带将领的罗马单位后，将领卡能显示技能目标与收益读板。
- 读板目标数量与 `GeneralSkillPreview.affectedUnitIDs + affectedCityIDs` 同源。
- 恢复类技能显示目标友军和预计恢复收益；攻城类技能显示目标城市和预计削城防收益。
- 地图 cue 能解释当前技能目标叠层来源，并与 `selectedGeneralSkillTargetPositions` 同源。
- 完整/紧凑将领卡都能读出短文案，不在短横屏或竖屏明显溢出。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算技能范围、目标、收益、可用性、将令评分或目标线阶段。
- 读板不改变 `GameState` 核心状态，不移动、不攻击、不发动技能、不切换姿态。
- 云端 RenderBattlePreview 断言覆盖技能目标收益读板、同源性、地图 cue 和无障碍文案。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。
