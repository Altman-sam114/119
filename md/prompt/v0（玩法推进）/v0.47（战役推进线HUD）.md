# v0.47（战役推进线 HUD）

## 版本号

v0.47

## 版本分配依据

`update_log.md` 当前最新正式玩法版本为 v0.46（地图侦察视角 HUD），本轮继续围绕“向大征服者罗马推进”的战斗地图可读性和战役目标表达迭代，因此顺延为 v0.47。

## 背景

当前战斗页已经具备战役任务、战场目标线、战线压力、敌情闭环和地图侦察视角 HUD。玩家可以看到“元老院目标”和“当前战场读板”，但两者之间还缺少一条直接说明：当前战役目标如何落到地图推进线、前线压力和下一步命令入口上。

## 目标

- 新增战役推进线 HUD / 读板，串联当前首要任务、战役进度、战线压力、战场目标线、地图侦察视角和战场态势交汇。
- `GameViewModel` 新增只读 `CampaignAdvanceReadout` 系列派生数据。
- `BattleView` 在顶部状态条和元老院任务面板展示战役推进线，不新增自动命令。
- 云端 RenderBattlePreview 增加 `missingCampaignAdvanceReadout` 断言，覆盖字段、同源引用和只读快照。

## 非目标

- 不新增 `GameState` 字段、存档字段、AI 权重、任务结算规则或胜负条件。
- 不自动选择单位、自动聚焦目标线、自动切换侦察视角、自动移动、攻击、发动技能或切换姿态。
- 不新增地图实体叠层；本轮只解释既有目标线、压力、侦察和态势交汇。
- 不引入第三方框架，不改技术栈，不做无关重构。

## 当前架构依据

- `GameState` 已提供 `missions`、`campaignStatus` 和任务 requirement 判断。
- `GameViewModel` 负责 UI 友好的只读派生数据，可组合 `primaryMission`、`campaignStatus`、`primaryFrontlinePressureSummary`、`primaryBattleObjectiveChainSummary`、`activeBattleObjectiveStageCommandPreview`、`mapReconPerspectiveHUDReadout` 和 `primaryBattlefieldConvergenceSummary`。
- `BattleView` 只展示 `GameViewModel` 派生数据，不在 SwiftUI 中重新计算任务进度、城市归属、压力评分或目标线阶段。
- `Tools/RenderBattlePreview/main.swift` 已覆盖战斗页关键读板和只读快照，是本轮新增断言的云端入口。

## 实现步骤

1. 在 `GameViewModel.swift` 新增 `CampaignAdvanceSignalKind`、`CampaignAdvanceSignal`、`CampaignAdvanceReadout`。
2. 新增 `primaryCampaignAdvanceReadout`，只读组合：
   - 当前首要任务：`primaryMission`。
   - 战役进度：`campaignStatus.progressText`。
   - 前线风险：`primaryFrontlinePressureSummary`。
   - 地图推进：`primaryBattleObjectiveChainSummary` 和 `activeBattleObjectiveStageCommandPreview`。
   - 侦察入口：`mapReconPerspectiveHUDReadout`。
   - 态势汇合：`primaryBattlefieldConvergenceSummary`。
3. readout 输出标题、状态、任务目标、进度、前线、目标线、地图 cue、下一步、风险、紧凑文案、无障碍文案、signal 列表和同源引用 ID。
4. 在 `BattleView.swift` 顶部战术状态条增加短的“推进” chip，并在元老院任务面板增加战役推进线卡片。
5. `Tools/RenderBattlePreview/main.swift` 新增 `missingCampaignAdvanceReadout` 断言，检查字段、同源引用、signal、首要任务和读取不改变核心状态 / ViewModel 聚焦选择态。
6. `.github/workflows/ci-results.yml` 的 `CI_VERSION` 更新为 v0.47。
7. 同步 README、AGENTS、flow、flowchart、test、prompt README 和 update_log。

## 测试要求

当前按人工要求使用云端-only 验证。Agent B 不得运行本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

云端 RenderBattlePreview 必须覆盖：

- `primaryCampaignAdvanceReadout` 存在，字段非空。
- 读板引用当前首要任务和 `campaignStatus.progressText`。
- 读板引用战线压力、战场目标线、当前活动阶段命令预览、地图侦察视角或战场态势交汇中的至少几组同源数据。
- signal 列表覆盖任务、进度、前线、目标线和侦察/态势来源。
- 读取读板不改变单位、城市、资源、回合、当前势力、选中单位/城市、目标线聚焦和侦察视角。
- 断言失败时抛出 `missingCampaignAdvanceReadout`。
