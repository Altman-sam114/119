# v0.51 HUD 信号胶囊 UI 重构

## 版本号

v0.51

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.50（战斗读板标签行 UI 重构），本轮沿玩法推进序列递增为 v0.51。人工总目标要求继续向“大征服者罗马”推进，并在新模型下持续重构既有代码，尤其是战斗界面 UI；本轮延续 v0.50 的低风险 UI 结构重构，聚焦地图 HUD 与战役读板中的信号胶囊重复结构。

## 背景

`BattleView.swift` 中 `MapReconPerspectiveHUDView`、`EnemyEngagementLoopHUDView` 和 `CampaignAdvanceReadoutView` 都展示“图标 + 标题 + tint 背景”的 20pt 信号胶囊。当前侦察 HUD 有 `MapReconPerspectiveSignalPill`，敌情闭环和战役推进线则各自内联一套近似样式，后续继续优化地图 HUD 时会重复调整同一视觉结构。

## 目标

- 在 `BattleView.swift` 内新增通用 `ReadoutSignalPill`，统一 system image、标题、tint、字体、透明度、padding、高度和圆角。
- 将 `MapReconPerspectiveSignalPill` 改成调用 `ReadoutSignalPill` 的轻量 wrapper，保留既有类型名和调用点语义。
- 将 `EnemyEngagementLoopHUDView.signalStrip(limit:)` 和 `CampaignAdvanceReadoutView.signalStrip(limit:)` 中的内联信号胶囊替换为 `ReadoutSignalPill`。
- 保持所有 symbol / tint 映射仍在原 view 内，避免把业务或读板语义塞进通用组件。
- 同步 CI artifact 版本和必要文档到 v0.51。

## 非目标

- 不修改 `GameState`、`GameViewModel`、AI 评分、AI 作战计划、敌军意图、战役任务、地图 overlay、移动、攻击、技能、姿态、城市、训练、任命、外交、存档或胜负结算。
- 不新增新的 HUD、读板字段、按钮、自动命令、命令队列、自动反制、自动技能或自动移动。
- 不调整 RenderBattlePreview 断言语义，不为了本轮重构放松任何云端检查。
- 不拆分新 Swift 文件，不修改 Xcode 工程结构，不引入第三方框架。
- 不重做整体视觉主题、地图布局、按钮体系、装备、升级树、将领池 UI、外交 UI、存档 UI 或建筑树。

## 当前架构依据

- `BattleView` 只展示 `GameViewModel` 已提供的 HUD/readout 数据，不计算敌情、反制、目标线或战役推进规则。
- `MapReconPerspectiveHUDView`、`EnemyEngagementLoopHUDView` 和 `CampaignAdvanceReadoutView` 的 signal strip 只消费各自 readout.signals。
- `md/flow/flow.md` 已允许 SwiftUI 读板组件复用通用展示行、状态胶囊或卡片 chrome，但不得成为规则计算层。
- 云端 RenderBattlePreview 已覆盖地图侦察视角 HUD、敌情交战闭环 HUD 和战役推进线 HUD 的存在性与读板链路。

## 风险和禁止项

- 风险：统一 pill 若高度、padding 或缩放参数不一致，可能让横屏 HUD 宽度发生变化；必须保留 20pt 高度、5pt 横向 padding、5pt 圆角和 0.58 minimumScaleFactor。
- 风险：symbol/tint 映射若被错误移动到通用组件，会混淆不同 HUD 的语义；映射必须留在原 view。
- 禁止修改核心规则、AI 行为、ViewModel 派生字段、RenderBattlePreview 断言语义或真实命令行为来迁就 UI 重构。
- 禁止本地运行测试或构建，也不得把未运行的本地命令写成验证结果。
