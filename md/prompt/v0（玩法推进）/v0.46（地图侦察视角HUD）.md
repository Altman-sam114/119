# v0.46（地图侦察视角 HUD）

## 版本号

v0.46

## 版本分配依据

`update_log.md` 当前最新正式玩法版本为 v0.45（选中军团军令窗口读板），本轮继续围绕“向大征服者罗马推进”的战斗地图可读性迭代，因此顺延为 v0.46。

## 背景

当前战斗页已经具备敌军路线/目标、威胁热区、地图控区、军议路径、机动落点、战场目标线、反制落点/目标和敌情交战闭环 HUD。随着叠层和读板增多，玩家需要一个地图层面的侦察视角入口，快速切换查看“敌路、反制、目标线、控图/热区”的当前重点，而不是在多个卡片之间来回扫读。

## 目标

- 新增地图侦察视角 HUD，作为既有叠层和读板的上层扫描入口。
- 支持至少四类视角：敌路、反制、目标线、热区/控区。
- HUD 展示当前视角标题、状态、简短说明、下一步、风险和同源 signal。
- HUD 提供交互按钮切换视角；切换只改变 `GameViewModel` UI 状态和 banner，不改变 `GameState`。
- 顶部战术状态条新增短 chip，提示当前侦察视角。
- 云端 RenderBattlePreview 增加断言，覆盖 HUD readout、视角切换、同源引用和只读快照。

## 非目标

- 不新增 `GameState` 字段、存档字段、AI 权重、移动/攻击/技能/姿态规则。
- 不过滤、隐藏或替换现有地图叠层。
- 不把图例改成交互控制面板。
- 不新增自动反制、自动技能、自动目标线执行或命令队列。
- 不引入第三方框架，不改技术栈。

## 当前架构依据

- `GameViewModel` 负责 UI 选择态、命令态、预览数据、战局汇总和 UI 派生数据。
- `BattleView` 只展示 `GameViewModel` 派生数据并触发 ViewModel 方法，不在 SwiftUI 中重新计算规则。
- 敌情交战闭环、反制、目标线、热区/控区已经有同源 summary、preview 和 overlay positions。
- `Tools/RenderBattlePreview/main.swift` 已固定用 `rome-legion-1` 单位场景断言战斗页关键读板和只读快照。

## 实现步骤

1. 在 `GameViewModel.swift` 新增 `MapReconPerspectiveKind`、`MapReconPerspectiveSignalKind`、`MapReconPerspectiveSignal`、`MapReconPerspectiveHUDReadout`。
2. 在 `GameViewModel` 新增 `@Published selectedMapReconPerspective`，`start(mode:)` 重置为 `.enemyIntent`。
3. 新增 `mapReconPerspectiveHUDReadout`，按当前视角只读组合：
   - 敌路：`primaryEnemyEngagementLoopReadout`、`enemyIntentMapOverlays`。
   - 反制：`primaryCountermeasureSummary`、`primaryCountermeasureCommandPreview`。
   - 目标线：`primaryBattleObjectiveChainSummary`、`activeBattleObjectiveStageCommandPreview`。
   - 热区/控区：`primaryThreatHeatZoneSummary`、`primaryMapControlSummary`、`primaryBattlefieldConvergenceSummary`。
4. 新增 `selectMapReconPerspective(_:)`，只更新视角选择和 `bannerMessage`。
5. 在 `WarMapView` 底部图例上方加入 `MapReconPerspectiveHUDView`，按钮可点；现有敌情闭环 HUD 继续不拦截点击。
6. `TacticalStatusStripView` 新增“侦察” chip，值保持短文案。
7. `Tools/RenderBattlePreview/main.swift` 新增 `missingMapReconnaissanceViewHUD`，断言 readout 字段、同源引用、视角切换和核心状态不变。
8. `.github/workflows/ci-results.yml` 的 `CI_VERSION` 更新为 v0.46。
9. 同步 README、AGENTS、flow、flowchart、test、prompt README 和 update_log。

## 测试要求

当前按人工要求使用云端-only 验证。Agent B 不得运行本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

云端 RenderBattlePreview 必须覆盖：

- `mapReconPerspectiveHUDReadout` 默认敌路视角存在，字段非空。
- HUD 视角列表覆盖敌路、反制、目标线、热区/控区。
- 敌路视角引用同一敌情闭环和敌军意图 overlay。
- 反制视角引用同一反制 summary / command preview。
- 目标线视角引用同一目标线 chain / active stage preview。
- 热区/控区视角引用同一热区、控区或态势交汇 summary。
- 调用 `selectMapReconPerspective` 切换视角只改变 ViewModel UI 选择态和 banner，不改变单位、城市、资源、回合和当前势力。
- 断言失败时抛出 `missingMapReconnaissanceViewHUD`。

## 验收标准

- 地图底部可见侦察视角 HUD，并可切换敌路、反制、目标线、热区/控区。
- HUD 不遮挡现有图例和敌情闭环 HUD 的关键文字。
- 顶部状态条能提示当前侦察视角。
- 所有新增 readout 均同源引用既有 summary / preview / overlay，不重新评分。
- GitHub Actions 最新 run 成功，Agent C 核对 artifact 后通过。

## 风险和禁止项

- 禁止把侦察视角切换写进 `GameState` 或存档。
- 禁止让视角切换自动选择单位、聚焦反制、聚焦目标线、移动、攻击、发动技能或切姿态。
- 禁止按侦察视角过滤已有叠层，避免破坏历史云端截图断言。
- 禁止用旧 artifact 或旧 run 冒充 v0.46 验收。
