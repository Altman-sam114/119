# v0.31（反制焦点链路）

## 版本号

v0.31

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.30（反制命令链高亮）。v0.30 已经让反制推荐姿态按钮和当前可攻击反制目标按钮显示“反制”提示，但地图上的回应位置、推荐落点、威胁目标、反制卡和军令面板之间仍主要靠颜色和短文案关联。下一步应把这条空间关系和命令入口整理成更明确的只读“焦点链路”，递增为 v0.31。

## 背景

当前项目已有：

- v0.27 敌情反制建议读板：核心层 `CountermeasureReport` 与 ViewModel `CountermeasureSummary`。
- v0.28 反制落点/目标地图叠层：`CountermeasureMapOverlay`、`CountermeasurePositionOverlay` 和反制路线。
- v0.29 反制指令聚焦与执行预览：`CountermeasureCommandPreview`、`focusedCountermeasureID` 和 `focusCountermeasure(_:)`。
- v0.30 反制命令链高亮：推荐姿态按钮和当前可攻击反制目标攻击按钮显示反制 cue。

当前短板是：玩家看到反制地图叠层时，还不能一眼确认“1 回应军团 -> 2 推荐落点 -> 3 威胁目标”与军令面板中的下一步、姿态按钮、攻击按钮属于同一条反制链。v0.31 应把地图叠层、反制卡和军令面板进一步联动，但仍保持只读提示，不做自动命令。

## 目标

1. 扩展 `CountermeasurePositionOverlay` / `CountermeasureMapOverlay` / `CountermeasureCommandPreview` 的只读 UI 派生字段，至少覆盖：
   - 反制链路序号或阶段标签，例如“1 回应”“2 落点”“3 目标”。
   - 地图 overlay 与军令面板共享的焦点短文案，例如“回应军团”“推荐落点”“威胁目标”。
   - 当前聚焦反制建议的地图无障碍文案中包含命令链短标签、推荐姿态 cue、移动 cue 或攻击 cue。
   - 可判断某个地图 overlay 是否是当前选中反制链的回应、落点或目标。
2. 强化 `BattleView.swift` 的反制地图 overlay：
   - `CountermeasureTileOverlay` 显示阶段序号或等价符号，不只依赖颜色/图标。
   - 反制目标格在与当前攻击目标重合时仍能让用户理解其“反制目标”身份，避免被普通攻击红框完全吞掉。
   - tile accessibility 读出阶段、目标、命令链和当前下一步。
3. 强化 `CountermeasureCommandPreviewView`：
   - 展示“1 回应 -> 2 落点 -> 3 目标”或等价链路摘要。
   - 紧凑布局只显示最关键信息，避免挤压军令按钮。
   - 完整布局可以显示阶段 chip，但不要引入大规模 UI 重构。
4. 强化完整/紧凑 `ActionsPanelView` 与地图目标的联系：
   - 反制攻击按钮 detail 可以引用“3 目标”或同链路 cue。
   - 保持按钮 action 仍为 `viewModel.attack(target.id)`。
   - 保持 disabled 规则不变。
5. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 断言反制地图 overlay 阶段标签、链路摘要和无障碍文案非空。
   - 聚焦首要反制后，断言 `selectedCountermeasureCommandPreview` 能提供链路摘要，并能识别目标 overlay / 反制攻击目标。
   - 失败继续使用 `missingCountermeasureCommandPreview` 或 `missingCountermeasureOverlay`，不必新增错误枚举，除非实现确实需要。
6. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.31`。
7. 更新 README、flow、flowchart、test、prompt README、update_log；保留 v0.30 最新云端通过证据。

## 非目标

- 不自动执行反制，不做一键移动、一键攻击、一键切姿态或命令队列。
- 不改变 `GameState.countermeasureReports` 评分、排序、目标匹配或核心字段。
- 不改变移动、攻击、技能、姿态、AI、城市、外交、胜负结算。
- 不新增多回合搜索、装备、升级树、将领池 UI、外交界面、存档 UI 或建筑树。
- 不重构大型 SwiftUI 文件结构，不引入第三方框架。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `CountermeasureSummary` 已经持有回应单位、目标单位/城市、回应位置、推荐落点、威胁目标、收益、风险和命令文案。
- `CountermeasureMapOverlay` 已经把首要反制建议派生成回应、落点、目标三类 `CountermeasurePositionOverlay`。
- `CountermeasureCommandPreview` 已经持有推荐姿态、落点可达性、目标可攻击性、命令链短标签、姿态 cue、移动 cue、攻击 cue 和步骤。
- `WarMapView` 已经把 `countermeasureOverlaysByPosition` 传给每个 `HexTileView`。
- `CountermeasureTileOverlay` 已经渲染反制地图标记。
- 完整/紧凑 `ActionsPanelView` 已经读取 `selectedCountermeasureCommandPreview` 来标记反制攻击按钮。
- `Tools/RenderBattlePreview/main.swift` 已经断言反制 summary、地图叠层、命令预览和聚焦行为。

## 实现步骤

1. 在 `GameViewModel.swift` 中扩展反制 UI 派生结构：
   - 给 `CountermeasureMapRole` 增加阶段序号、短标签或图标文案。
   - 给 `CountermeasurePositionOverlay` 增加 `stageLabel`、`focusLabel`、`chainLabel` 或等价字段。
   - 给 `CountermeasureMapOverlay` 增加链路摘要字段，表达“回应 -> 落点 -> 目标”。
   - 给 `CountermeasureCommandPreview` 增加共享链路摘要和目标地图 cue helper，例如 `chainSummaryLabel`、`targetStageCueLabel`、`isTargetOverlay(_:)` 等轻量字段。
2. 在 `BattleView.swift` 中扩展地图展示：
   - `CountermeasureTileOverlay` 在圆形标记中显示阶段序号或短标签，同时保留现有系统图标语义。
   - `HexTileView.accessibilityLabel` 继续从 `CountermeasurePositionOverlay` 读取无障碍文案，不在 SwiftUI 中重新拼核心规则。
   - 保持叠层优先级和 hit testing 不改变地图点击行为。
3. 在 `CountermeasureCommandPreviewView` 中展示链路摘要：
   - 紧凑视图显示一行链路摘要或阶段 cue。
   - 完整视图可以在现有步骤上方增加阶段 chip。
   - 避免新增过多字体尺寸或不稳定布局。
4. 在完整/紧凑 `ActionsPanelView` 的反制攻击按钮 detail 中使用目标阶段 cue：
   - 只修改 text/detail/accessibilityLabel。
   - 不修改 `viewModel.attack(target.id)`。
   - 不修改 `.disabled(viewModel.isCampaignOver)` 或紧凑面板现有规则。
5. 更新 `Tools/RenderBattlePreview/main.swift`：
   - 断言 `primaryCountermeasureMapOverlay.chainLabel` 或等价字段非空。
   - 断言 `countermeasureOverlaysByPosition` 中回应、落点、目标 overlay 的阶段字段和 accessibility 非空。
   - 断言 `primaryCountermeasureCommandPreview` 链路摘要和目标阶段 cue 非空。
6. 更新 workflow 版本号和文档。
7. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 反制地图叠层能区分回应、落点、目标三个阶段，不只依赖颜色。
- 反制卡/军令面板能显示与地图一致的链路摘要。
- 反制目标攻击按钮 detail 或无障碍文案能指向同一“目标阶段”。
- SwiftUI 不重新计算反制评分、路径、攻击目标或姿态规则，只消费 ViewModel 派生字段。
- 真实移动、攻击、姿态、AI 和胜负结算不变。
- 云端 RenderBattlePreview 断言覆盖新增链路字段和地图 overlay 阶段字段。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止把焦点链路实现成自动执行或半自动执行命令。
- 禁止在 SwiftUI `body` 中重新计算反制评分、路径、目标匹配、攻击范围或姿态规则。
- 禁止绕过 `GameViewModel.setSelectedTacticalOrder(_:)`、`attack(_:)`、`selectTile(_:)` 等现有入口。
- 禁止让地图叠层覆盖真实可移动、可攻击、技能目标等已有交互语义。
- 禁止扩大为自动战斗、命令队列、多回合搜索或全局 UI 重构。
- 禁止运行任何本地测试、build、typecheck、RenderBattlePreview、`verify_project`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。
- 如果 GitHub 权限、网络或 artifact 下载失败，必须明确阻塞，不能伪造云端通过。
