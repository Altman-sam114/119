# v0.35（目标线阶段命令预览）

## 版本号

v0.35

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.34（战场目标线阶段聚焦），并已通过 `origin/main` 最新云端结果包复判。v0.34 让玩家能从目标线卡片定位“1 焦点、2 将令、3 机动、4 军议”阶段，但阶段定位后仍需要玩家自行判断应使用姿态、移动、攻击或将领技能等既有命令入口。下一步应把每个目标线阶段转成只读命令预览，递增为 v0.35。

## 背景

当前项目已有：

- `BattleObjectiveChainSummary`：组合战场焦点、将领协同、机动落点和本方军议，输出目标线阶段文案。
- `BattleObjectiveMapOverlay` 和 `BattleObjectivePositionOverlay`：把同一目标线派生为地图阶段标记、阶段位置和链路线段。
- `focusPrimaryBattleObjectiveStage(_:)`：从目标线阶段按钮只读定位地图位置、罗马执行单位或城市。
- `CountermeasureCommandPreview`：反制建议已经有只读命令链预览，可作为字段结构和 SwiftUI 展示参考。
- `Tools/RenderBattlePreview/main.swift`：已经断言目标线链路、地图叠层和阶段聚焦，并验证聚焦不会改变核心 `GameState`。

当前短板是：目标线已经能展示“应该关注什么”，但没有把阶段与当前已有命令入口连接起来。v0.35 应在 ViewModel 中生成阶段命令预览，让 UI 展示“该阶段对应哪个命令入口、当前是否可执行、下一步是什么”，但不得新增自动执行、命令队列或新核心规则。

## 目标

1. 在 `GameViewModel.swift` 增加目标线阶段命令预览：
   - 建议新增 `BattleObjectiveStageCommandStep`。
   - 建议新增 `BattleObjectiveStageCommandPreview`。
   - 建议新增 `battleObjectiveStageCommandPreviews`、`focusedBattleObjectiveStageCommandPreview`、`selectedBattleObjectiveStageCommandPreview` 和 `primaryBattleObjectiveStageCommandPreview` 等只读派生。
2. 每个预览必须与 `primaryBattleObjectiveChainSummary` 和 `primaryBattleObjectiveMapOverlay` 同源：
   - `.focus` 阶段读取 `BattlefieldFocusSummary`。
   - `.synergy` 阶段读取 `CommanderSynergySummary`。
   - `.maneuver` 阶段读取 `ManeuverOptionSummary`。
   - `.recommendation` 阶段读取 `TacticalRecommendationSummary`。
3. 预览必须解释既有命令入口：
   - 焦点阶段：说明目标位置、执行单位、推荐姿态或观察入口。
   - 将令阶段：说明协同状态、技能或支援入口，以及可用/受阻原因。
   - 机动阶段：说明移动落点、推荐姿态、可达性和风险。
   - 军议阶段：说明推荐姿态、目标是否当前可攻击、攻击或等待原因。
4. UI 展示：
   - 在目标线卡片阶段按钮下方展示当前聚焦阶段或主阶段的命令预览。
   - 在完整军令面板和紧凑军令面板展示选中罗马单位相关的目标线阶段命令预览。
   - 可以在姿态、攻击或技能按钮 detail 中追加轻量“目标线”提示，但不得改变 action 或 disabled 规则。
5. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingBattleObjectiveStageCommandPreview`。
   - 断言预览存在、字段非空、阶段与目标线 overlay 同源。
   - 断言聚焦每个存在阶段后 `focusedBattleObjectiveStageCommandPreview` 指向对应 role。
   - 断言聚焦和读取预览不会改变核心 `GameState`。
6. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.35`。
7. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.34 云端通过证据。

## 非目标

- 不新增真实命令、自动移动、自动攻击、自动技能、一键执行、命令队列或自动战斗。
- 不修改 `GameState` 的核心评分、路径、AI、战斗结算、技能结算、姿态规则或报告排序。
- 不把阶段命令预览写成 SwiftUI `body` 内的规则计算。
- 不改变现有攻击、移动、姿态、技能、结束回合、城市或 AI 的 action / disabled 逻辑。
- 不新增装备、科技树、外交、存档 UI、建筑树、多回合搜索或新第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `BattleObjectiveChainSummary` 已经把焦点、将令、机动、军议串成一个目标线，是阶段命令预览的业务来源。
- `BattleObjectivePositionOverlay` 已包含 role、position、stageLabel、focusLabel、chainLabel 和 accessibilityLabel，可用于证明预览和地图阶段同源。
- `focusPrimaryBattleObjectiveStage(_:)` 已经只选择罗马执行单位，避免把敌军目标误当成命令入口。
- `selectedTacticalOrderPreviews`、`attackTargets`、`attackPreview(for:)`、`selectedGeneralSkillPreview` 和 `selectedCommanderSynergySummary` 已经提供当前命令状态，不应在 SwiftUI 重新计算。
- `CountermeasureCommandPreviewView` 已提供“只读命令链卡片”的视觉模式，可复用紧凑文本、步骤列表和无障碍结构。

## 实现步骤

1. 在 `GameViewModel.swift`：
   - 新增 `BattleObjectiveStageCommandStep` 和 `BattleObjectiveStageCommandPreview`，字段应覆盖 role、position、stageLabel、chain/source id、执行单位、目标单位/城市、推荐姿态、目标位置、命令入口、下一步、阻塞原因、步骤和无障碍文案。
   - 新增阶段预览派生属性，不保存旧预览，不保留 stale 状态。
   - 从 `primaryBattleObjectiveChainSummary` 和 `primaryBattleObjectiveMapOverlay.positionOverlays` 生成预览。
   - 只把罗马单位作为命令执行单位；敌军单位或城市只能作为目标展示。
   - 姿态状态复用 `selectedTacticalOrderPreviews` 或现有 helper，攻击状态复用 `state.attackTargets(for:)` 和 `state.attackPreview(...)`，技能状态复用 `state.generalSkillPreview(unitID:)` 或现有 ViewModel 派生。
2. 在 `BattleView.swift`：
   - 新增 `BattleObjectiveStageCommandPreviewView`，参考 `CountermeasureCommandPreviewView`，支持完整和紧凑模式。
   - 更新 `BattleObjectiveChainCardView` 参数，在阶段按钮后展示 `stageCommandPreview`。
   - 更新完整和紧凑军令面板，在反制预览后展示 `selectedBattleObjectiveStageCommandPreview`。
   - 保持按钮尺寸稳定，移动端文本短，不在卡片里再嵌套卡片。
   - 不改变现有命令按钮的 action / disabled。
3. 在 `Tools/RenderBattlePreview/main.swift`：
   - 在现有目标线链路、地图叠层和阶段聚焦断言后增加阶段命令预览断言。
   - 检查 `.focus`、`.synergy`、`.maneuver`、`.recommendation` 中当前 chain 实际存在的阶段。
   - 检查预览的 `role`、`position`、`stageLabel`、`chainID`、`sourceSummaryID` 与 overlay/summary 同源。
   - 检查 `title`、`statusLabel`、`commandEntryLabel`、`nextStepLabel`、按钮文案和 `accessibilityLabel` 非空。
   - 聚焦每个阶段后检查 `focusedBattleObjectiveStageCommandPreview` 切换正确。
   - 记录聚焦前后核心状态快照，断言单位、城市、资源、回合、主动方和技能冷却不变。
4. 更新 workflow 版本和文档。
5. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 玩家定位目标线阶段后，能看到该阶段对应的既有命令入口、当前状态和下一步。
- 完整和紧凑军令面板能展示选中罗马单位关联的目标线阶段命令预览。
- 预览与目标线 summary/overlay 同源，不出现旧阶段、不把敌军当执行单位。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算路径、攻击、技能、姿态或评分规则。
- 阶段命令预览不改变 `GameState` 核心状态，不移动、不攻击、不发动技能、不切换姿态。
- 云端 RenderBattlePreview 断言覆盖阶段命令预览、聚焦切换、核心状态不变和关键 UI 文案。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止把阶段命令预览实现成真实命令、自动执行、自动移动、自动攻击或命令队列。
- 禁止在 SwiftUI `body` 中重新计算焦点评分、将令评分、机动评分、战术建议、路径、攻击目标或技能目标。
- 禁止把敌军单位、城市或空地误当成罗马命令执行单位。
- 禁止覆盖反制建议的命令 cue；若同一按钮同时命中反制和目标线，反制优先，目标线只做次级说明。
- 禁止改变 `GameState` 核心报告排序、评分、目标或路径。
- 禁止扩大为自动战斗、多回合搜索、全局 UI 重构或第三方框架引入。
- 禁止运行任何本地测试、build、typecheck、RenderBattlePreview、`verify_project`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。
- 如果 GitHub 权限、网络或 artifact 下载失败，必须明确阻塞，不能伪造云端通过。
