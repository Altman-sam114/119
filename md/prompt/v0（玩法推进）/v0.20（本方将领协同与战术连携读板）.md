# v0.20（本方将领协同与战术连携读板）

## 版本号

v0.20

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.19（AI 作战计划与将领协同读板）。本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”推进玩家侧将领和战术连携可读化：把本方将领技能、编制、攻击修正、战术建议和焦点整合成“将令/连携”读板，因此版本递增为 v0.20。

## 背景

项目已有选中单位将领详情、主动技能预览、战功、军团编制、战术命令建议、战场焦点、敌军意图和 AI 作战计划。玩家仍需要自己判断“哪个罗马将领能带动友军”“当前攻击有哪些支援/包夹/指挥修正”“受伤友军是否值得等将领技能”“本方补线或推进由谁承担”。

本轮新增只读本方将领协同报告：核心层复用 `GeneralSkillPreview`、`LegionFormationReport`、`TacticalRecommendationReport` 和 `CombatPreview`，输出将领技能、合击、补线、推进或整备类连携；ViewModel 转成 UI 摘要；BattleView 在顶部 chip、选中单位情报卡和战局面板中展示。该能力只解释和提示，不自动执行技能、攻击、移动或姿态切换。

## 目标

1. 在 `RomeLegionsCore` 新增玩家侧只读将领协同模型。
2. 协同类型至少覆盖：将领技能、合击攻击、补线、推进、整备。
3. 协同步骤能表达将领、主攻、支援、受益和预备等角色。
4. 协同报告必须包含阵营、类型、主执行单位、将领单位、目标单位/城市、目标位置、支援单位、受益单位、推荐姿态、编制状态、风险、预计伤害、支援/包夹/指挥修正、预计恢复或削城防、是否可执行、阻塞原因、评分、标题、摘要、详情和步骤。
5. 报告入口必须只读，不新增 Codable 存档字段。
6. 不改变 `attackPreview`、`attack`、`useGeneralSkill`、`performSimpleAI(for:)`、战术姿态数值、将领技能数值、AI 权重、真实移动、攻击、技能、招募或胜负结算。
7. 冷却或已行动将领仍能被读板解释，但必须标记不可执行并给出阻塞原因，不能伪装成可发动技能。
8. `GameViewModel` 新增 `CommanderSynergySummary`、全局协同摘要、首要协同摘要和选中单位协同摘要，把核心报告转成中文文案和无障碍文案。
9. `BattleView` 在顶部状态条增加“将令” chip，在选中单位情报区展示协同卡，在完整战局面板展示前两条协同行；紧凑界面仍保持军令入口优先。
10. Swift Testing、Gameplay Smoke、RenderBattlePreview 覆盖将领技能协同、合击修正解释、冷却/已行动阻塞、条约过滤、只读不变性和 ViewModel 主链路字段。

## 非目标

- 不实现自动释放将领技能、自动执行攻击、自动移动补线或一键连携命令。
- 不修改现有战斗数值、支援/包夹/指挥修正、将领技能效果、冷却、战功、战术姿态或 AI 行为。
- 不新增多回合规划、真实补给线、建筑树、装备、人口、外交界面或存档 UI。
- 不拆分 `BattleView.swift` 作专项重构，只允许新增本轮必要的小型视图和摘要。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、战斗预览、将领技能、军团编制、战术建议、战场焦点和 AI 作战计划位于 `Sources/RomeLegionsCore/GameState.swift`。
- `GeneralSkillPreview` 已只读公开技能范围、受益单位/城市、预计恢复/削城防、冷却、可执行状态和阻塞原因。
- `CombatPreview` 已公开 `supportBonus`、`flankingBonus` 和 `commandBonus`，可用于解释合击，不需要改真实伤害。
- `LegionFormationReport` 已公开职责、战备、将领、技能摘要、建议姿态和命令建议。
- `TacticalRecommendationReport` 已公开选中单位攻击、补线、推进、坚守或整备建议、路径、风险和推荐姿态。
- `GameViewModel` 已有 `AIOperationalPlanSummary`、`BattlefieldFocusSummary`、`LegionFormationSummary` 和 `TacticalRecommendationSummary` 等 UI 摘要模式。
- `BattleView` 已有顶部 `TacticalStatusStripView`、选中单位情报卡、`StrategicBalancePanelView` 和行/卡片视图，可增量接入将令 chip、卡片和战局行。

## 实现步骤

1. 在 `GameState.swift` 新增：
   - `CommanderSynergyKind`
   - `CommanderSynergyRole`
   - `CommanderSynergyStepReport`

2. 新增只读入口：
   - `commanderSynergyReports(for:limit:) -> [CommanderSynergyReport]`

3. 协同派生规则：
   - 对本方单位读取 `LegionFormationReport` 和 `TacticalRecommendationReport`。
   - 有将领单位读取 `GeneralSkillPreview`，可执行、有目标时高分展示；冷却/已行动/缺目标时仍可作为低分解释。
   - 可攻击单位通过 `attackPreview` 读取伤害、反击、支援、包夹和指挥修正，输出合击报告；不得改 `CombatPreview` 或结算规则。

   - 条约保护势力不得进入合击目标；复用 `attackTargets` / `nearbyEnemyUnits` / `diplomaticStatus`。
   - 排序稳定：score 降序，同分按 id。
