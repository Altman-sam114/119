# v0.8 / 将领技能范围与目标预览体验

## 版本号和版本分配依据

- 版本号：v0.8。
- 版本标题：将领技能范围与目标预览体验。
- 分配依据：`update_log.md` 当前最新正式玩法版本为 v0.7。v0.7 对齐 AI 意图与移动后攻击预览，本轮在既有将领主动技能、AI 技能意图和战斗地图叠层基础上推进玩家可读性，因此递增为 v0.8。

## 背景

当前 `GeneralTrait` 已有四类主动技能：鹰旗鼓舞、攻城布阵、战地补给、盾墙号令。`useGeneralSkill(unitID:)` 以施法单位为中心自动影响范围内友军或敌城，并消耗行动；AI 也会调用 `shouldAIUseGeneralSkill` 与 `.useSkill` 意图。

缺口是：核心层没有只读技能预览模型；`GameViewModel` 只暴露 `canUseSelectedGeneralSkill`；`BattleView` 只有技能按钮，没有显示技能范围、受影响友军/城市、预计恢复或削城防效果。AI 技能意图目前只对攻城技能填 `targetCityID`，治疗类技能没有目标摘要。

本轮目标是把将领主动技能从“按钮式黑箱”推进为“地图上可读的战术工具”：核心层提供只读预览，ViewModel 只做 UI 派生，SwiftUI 展示范围和目标，释放逻辑继续由 `GameState` 统一结算。

## 目标

1. 在 `GameState` 增加只读将领技能预览能力，统一描述技能范围、受影响单位/城市、预计恢复量或城防削弱量。
2. 让 `useGeneralSkill` 与预览共享目标筛选逻辑，避免预览和结算分叉。
3. 在 `GameViewModel` 暴露选中单位的技能预览、范围格、受影响目标和按钮说明。
4. 在 `BattleView` 为将领技能增加地图范围叠层、目标高亮、命令按钮 detail 和将领卡片摘要；紧凑面板也要有可读的技能状态。
5. 让 AI `.useSkill` 意图尽量复用同一预览来源，攻城技能保留 `targetCityID`，治疗类可填最主要受益 `targetUnitID`。
6. 新增核心测试和 Gameplay Smoke，锁定预览不改状态、预览与释放结果一致、AI 技能意图目标来自预览。
7. 同步 README、flow、flowchart、test、update_log 和 CI artifact 版本记录。

## 非目标

- 不新增将领、技能冷却、升级树、手动点选单体技能目标。
- 不改变现有技能数值、范围和消耗规则，除非为了修正预览/结算一致性必须做最小调整。
- 不把核心规则写入 SwiftUI 或 `GameViewModel`。
- 不大规模拆分 `BattleView.swift`，除非局部子视图提取是为了保持本轮代码可读。
- 不引入第三方框架、图片素材、网络服务或数据文件。
- 不创建 PR，不引入 `develop`、`smalldata_test`、`codeb/...` 或其他长期/候选分支。

## 当前架构依据

- `Sources/RomeLegionsCore/GameState.swift`
  - `GeneralTrait.commandRange`、`recoveryAmount`、`skillName`、`skillDetail` 已定义技能语义。
  - `useGeneralSkill(unitID:)` 是唯一技能结算入口。
  - `recoverFriendlyUnits` 和 `reduceEnemyFortifications` 是当前结算 helper。
  - `shouldAIUseGeneralSkill`、`aiSkillTargetCity`、`aiIntents(for:limit:)` 参与 AI 技能使用和意图展示。
  - `aiIntents(for:limit:)` 必须只读预测，不能改变原始状态。
- `RomeLegionsApp/App/GameViewModel.swift`
  - 负责 UI 友好的选择态、命令态、战斗预览和派生数据。
  - 应新增技能预览派生属性，但不得复制目标筛选或数值规则。
- `RomeLegionsApp/Views/BattleView.swift`
  - `WarMapView`、`HexTileView` 已有移动、攻击和敌军意图叠层，可扩展技能范围/目标叠层。
  - `GeneralTraitCardView`、`CommandPanelView`、`CompactCommandPanelView` 是技能状态展示入口。
- `Tests/RomeLegionsCoreTests/GameStateTests.swift` 和 `Tools/GameplaySmoke/main.swift`
  - 当前已验证技能释放结果，需要补技能预览基线。

## 实现步骤

1. 开始前读取本提示词、`AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`README.md`、`md/prompt/README.md`。
2. 同步仓库：

   ```sh
   git fetch origin
   git switch main
   git pull --ff-only origin main
   git status --short --branch
   ```

   若没有 `origin`、无法拉取、当前不是 `main` 或存在无法判断归属的无关改动，先报告阻塞，不要继续实现。

3. 在核心层新增公开只读模型，建议命名为 `GeneralSkillPreview`，至少包含：
   - `unitID`、`trait`、`origin`、`range`
   - `affectedUnitIDs`、`affectedCityIDs`
   - `affectedPositions`
   - `projectedRecoveredHealth`
   - `projectedFortificationReduction`
   - `summary` / `detail`
   - `isExecutable` 或等价字段，用于攻城技能无敌城时禁用 UI
4. 将治疗目标筛选和敌城筛选抽成纯 helper，预览和 `useGeneralSkill` 共同调用。
5. 保持既有行为：治疗类即使没有受伤友军也不要在核心层随意改成抛错；攻城技能无合法敌城继续抛 `invalidTarget`。
6. 调整 AI 技能判断和 `.useSkill` 意图，使用技能预览判断目标、威胁分和 target id，并保持 `aiIntents` 不改原始 `GameState`。
7. 在 `GameViewModel` 增加 `selectedGeneralSkillPreview`、技能范围位置、受影响单位/城市集合，以及技能按钮 detail。
8. 在 `BattleView` 增加展示：
   - `WarMapView` 读取技能预览派生数据。
   - `HexTileView` 增加技能范围和技能目标状态。
   - 新增轻量叠层视图，如 `SkillRangeOverlay`、`SkillTargetOverlay`。
   - `GeneralTraitCardView` 展示范围和预计影响摘要。
   - `CommandPanelView` / `CompactCommandPanelView` 的技能按钮展示 detail，并在攻城无目标时禁用和说明原因。
   - 攻击目标红色叠层优先级保持高于技能范围叠层。
9. 更新测试和文档，避免提交 DerivedData、截图或临时产物。

## 验收标准

本轮通过必须满足：

- `generalSkillPreview(unitID:)` 或等价接口只读，不改变 active faction、单位生命、行动标记、城市防御或 event log。
- `useGeneralSkill(unitID:)` 与技能预览共享同一套目标筛选逻辑。
- 治疗类技能预览的预计恢复量等于释放后的生命变化。
- 攻城技能预览的预计城防削弱量等于释放后的城防变化；无合法敌城时预览能说明不可执行，释放仍抛 `invalidTarget`。
- AI `.useSkill` 意图目标来自同一技能预览，且 `aiIntents` 不改变原始状态。
- `GameViewModel` 只暴露 UI 派生数据，不复制技能规则。
- `BattleView` 在地图、将领卡和命令面板中清晰展示技能范围、目标和效果摘要；三尺寸预览不裁切、不重叠。
- 新增测试进入 Swift Testing 基线，Gameplay Smoke 包含轻量断言。
- 本地轻量检查和预览渲染通过，`main` push 后 GitHub Actions 最新 run 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止预览调用 `useGeneralSkill` 或其他写状态命令。
- 禁止让技能预览和释放结果使用两套目标筛选逻辑。
- 禁止让 AI 意图改变 `GameState`。
- 禁止为 UI 方便改变现有技能数值或行动消耗。
- 禁止把核心规则写进 SwiftUI 或 `GameViewModel`。
- 禁止删除既有 AI、战斗、任务、胜负保护测试。
- 禁止用 UI 测试替代核心规则测试。
- 禁止提交构建缓存、DerivedData、临时截图。
- 禁止把旧 artifact、旧 output 或 checkout 自带报告冒充 v0.8 云端结果。
