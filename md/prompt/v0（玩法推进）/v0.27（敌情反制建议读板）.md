# v0.27（敌情反制建议读板）

## 版本号

v0.27

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.26（敌方将领威胁读板）。Agent X 总目标要求持续向“大征服者罗马”推进，优化战斗界面 UI、地图、AI、将领和战场决策体验。v0.26 已让玩家看见“哪个敌将危险”，但玩家仍需要自己在敌将威胁、AI 作战计划、战线压力、热区、本方将令、机动落点和战术建议之间拼接“我该怎么应对”。因此本轮递增为 v0.27，目标是新增只读“敌情反制建议读板”，把敌方威胁和我方可执行回应串成一条可读建议。

## 背景

当前项目已有这些只读报告：

- 敌方侧：`AIIntent`、`AIOperationalPlanReport`、`EnemyCommanderThreatReport`、`FrontlinePressureReport`、`ThreatHeatZoneReport`。
- 我方侧：`TacticalRecommendationReport`、`ManeuverOptionReport`、`CommanderSynergyReport`、`BattlefieldFocusReport`、`LegionFormationReport`。
- UI 层已有顶部 chip、敌情面板、完整战局面板、选中单位情报卡和 RenderBattlePreview 断言模式。

短板是缺少一个“反制建议”视角：当敌军计划集火、敌将准备攻城、战线压力升高或热区覆盖关键格时，玩家需要知道推荐由哪个罗马单位回应、回应方式是打断/补线/坚守/机动/将领技能、关联的敌方威胁是什么，以及预计收益和风险。

## 目标

1. 在 `GameState.swift` 新增只读敌情反制建议报告：
   - `CountermeasureKind`
   - `CountermeasurePriority`
   - `CountermeasureReport`
   - `countermeasureReports(for:limit:)`
   - `countermeasureReport(for:)`
2. 报告至少包含：
   - id、faction、kind、priority、score
   - threatTitle、threatSummary、targetPosition、targetUnitID、targetCityID
   - responseUnitID、responsePosition、recommendedOrder、destination
   - linkedEnemyCommanderThreatID、linkedAIOperationalPlanID、linkedFrontlinePressureID
   - linkedManeuverOptionID、linkedCommanderSynergyID、linkedTacticalRecommendationID
   - projectedDamagePrevented、projectedDamageDealt、projectedRecovery、risk
   - title、summary、detail、reasons、command
3. 核心报告必须复用现有只读来源，不重新实现第二套战术规则：
   - `enemyCommanderThreatReports(against:limit:)`
   - `aiOperationalPlanReports(against:perFactionLimit:limit:)`
   - `frontlinePressureReports(against:perFactionLimit:limit:)`
   - `threatHeatZoneReports(for:limit:)`
   - `tacticalRecommendation(unitID:)`
   - `maneuverOptionReports(unitID:limit:)`
   - `commanderSynergyReport(unitID:)`
   - `legionFormationReports(for:limit:)`
4. 反制类型建议：
   - `.interruptCommander`：敌将威胁高，推荐打击/压制敌将或削弱其目标。
   - `.holdLine`：战线压力高，推荐坚守或补线。
   - `.reinforceCity`：城市面临夺取/攻城威胁，推荐补防城市。
   - `.strikeThreat`：已有安全打击窗口，推荐攻击关键敌军。
   - `.commanderAction`：本方将领技能或协同能明显缓解威胁。
   - `.redeploy`：机动落点能降低热区或靠近关键目标。
5. `GameViewModel.swift` 新增 UI 摘要：
   - `CountermeasureSummary`
   - `countermeasureSummaries`
   - `primaryCountermeasureSummary`
   - 摘要包含 title、kindLabel、priorityLabel、threatLabel、responseLabel、unitLabel、impactLabel、riskLabel、commandLabel、accessibilityLabel。
6. `BattleView.swift` 展示反制建议：
   - 顶部态势 chip 显示首要“反制”建议。
   - 敌情面板在敌将/计划卡附近展示首要反制建议卡。
   - 完整战局面板展示前两条反制建议行。
   - SwiftUI 只能展示 ViewModel 摘要，不得重新计算评分、目标或路径。
7. `Tools/RenderBattlePreview/main.swift` 增加云端断言：
   - `primaryCountermeasureSummary` 存在。
   - `countermeasureSummaries` 非空。
   - title、kindLabel、priorityLabel、threatLabel、responseLabel、impactLabel、riskLabel、commandLabel、accessibilityLabel 非空。
   - 至少一条建议关联敌将威胁或 AI 作战计划。
   - 失败抛出 `missingCountermeasureSummary`。
8. Swift Testing 与 Gameplay Smoke 增加核心只读断言：
   - 反制建议报告不改变 `GameState`。
   - 敌将威胁存在时能生成 `.interruptCommander` 或 `.commanderAction` 建议。
   - 战线压力存在时能生成 `.holdLine` 建议，并复用 `FrontlinePressureReport` 的目标和预计伤害。
   - 反制建议的我方回应单位来自本方军团/战术/机动/将令报告，不应引用敌军作为 responseUnit。
   - 条约保护或无敌方威胁时不生成伪反制。
9. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.27`。
10. 更新 README、flow、flowchart、test、prompt README、update_log，并保留 v0.26 最新云端通过证据。

## 非目标

- 不自动执行反制，不新增“自动下令”按钮。
- 不改变真实 AI、AI 权重、AI 执行顺序、技能释放规则、攻击/移动/占城/姿态/城市/外交/胜负结算。
- 不新增多回合搜索、装备、将领池 UI、升级树、外交界面、存档 UI 或建筑树。
- 不引入第三方框架，不拆分大型 SwiftUI 文件，不做全局 UI 重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameState` 已集中建模核心规则和只读战场报告，适合新增 `CountermeasureReport`。
- `GameViewModel` 已有 `EnemyCommanderThreatSummary`、`AIOperationalPlanSummary`、`ManeuverOptionSummary`、`CommanderSynergySummary` 等摘要范式。
- `BattleView` 顶部态势、敌情面板和完整战局面板已有同类只读卡/行布局。
- `Tools/RenderBattlePreview/main.swift` 已在单位场景断言多个读板，可扩展新反制摘要断言。
- 当前验证固定走 GitHub Actions artifact，不能本地跑测试或渲染。

## 实现步骤

1. 在 `GameState.swift` 的战场报告类型区域新增 `CountermeasureKind`、`CountermeasurePriority`、`CountermeasureReport`。
2. 实现 `countermeasureReports(for:limit:)`：
   - 读取敌将威胁、AI 作战计划、战线压力、热区。
   - 读取本方军团编制、战术建议、机动落点、将领协同。
   - 将敌方威胁与我方回应按目标位置、目标单位/城市、风险、预计伤害/恢复和距离进行只读匹配。
   - 输出稳定排序的建议，按 priority、score、id 排序。
3. 实现 `countermeasureReport(for:)` 返回首要建议。
4. 在 `GameViewModel.swift` 增加 `CountermeasureSummary` 和相关 computed properties，把核心字段转成中文 UI 文案。
5. 在 `BattleView.swift` 加入反制 chip、敌情反制卡和战局反制行。
6. 更新 Swift Testing、Gameplay Smoke、RenderBattlePreview 断言。
7. 更新 workflow 版本号和文档。
8. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 和 Agent C artifact 复判。
