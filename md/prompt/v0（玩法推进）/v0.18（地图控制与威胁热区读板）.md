# v0.18（地图控制与威胁热区读板）

## 版本号

v0.18

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.17（战场焦点与将领机会读板）。本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”推进“地图”维度：让玩家能读出罗马控制区、敌军威胁热区、争夺格和城市推进风险，因此版本递增为 v0.18。

## 背景

项目已有敌军意图、战线压力、战场焦点、军团编制和战术建议，但地图本身仍主要显示地形、可移动格、攻击目标和路线叠层。玩家要判断“哪些格子安全”“哪里被敌军覆盖”“哪片区域正在争夺”仍需要自行拼接多个读板。

本轮新增只读地图控制与威胁热区报告：核心层按阵营派生每格控制状态和高风险热区；ViewModel 转为 UI 摘要和 overlay；BattleView 在地图、顶部 chip、战场面板和战局面板中展示。该能力只解释地图态势，不自动执行命令，不改变 AI、路径、战斗、城市或胜负结算。

## 目标

1. 在 `RomeLegionsCore` 新增只读地图控制和威胁热区模型。
2. 地图控制报告至少能表达友方控制、敌方控制、争夺和中立四种状态。
3. 威胁热度至少能表达安静、监视、争夺、危险和危急五档。
4. 报告必须包含位置、地形、视角阵营、城市/占据单位、友方/敌方影响值、来源单位、压力分、标题或摘要、详情。
5. 热区报告必须能聚合高风险位置，输出中心格、关联格、来源单位、相关城市、控制状态、热度、分数、标题和详情。
6. 报告调用不改变 `GameState`，不新增 Codable 存档字段。
7. 不改变真实移动、攻击、AI 决策、AI 评分、姿态切换、技能使用、招募、城市扩建或胜负结算。
8. `GameViewModel` 新增 UI 友好的 `MapControlSummary`、`ThreatHeatZoneSummary`，以及地图 overlay 所需 position 集合。
9. `BattleView` 在顶部状态条增加首要“热区”chip，在地图 hex 上显示低透明热区叠层，在战场面板展示控区/热区读板，在完整战局面板展示前两条热区行；紧凑界面仍保持军令优先。
10. Swift Testing、Gameplay Smoke、RenderBattlePreview 覆盖核心只读不变性、敌军热区、城市夺取热区、条约过滤、ViewModel 主链路字段。
11. 因人工要求本地不得测试，本轮必须把 RenderBattlePreview 的编译和三尺寸运行纳入 GitHub Actions，并把 render 日志和必要 PNG artifact 放入结果包，供 Agent C 云端复判。

## 非目标

- 不实现自动规避热区、自动执行命令、多回合路径规划、真实战略路线系统或玩家回合自动规划。
- 不修改 `performSimpleAI(for:)`、`aiIntent(for:)`、AI 权重、真实 `moveUnit`、`attack`、`useGeneralSkill`、城市占领或战役胜负结算。
- 不改变现有地形移动成本、战术姿态数值、将领技能数值、战功公式、城市招募成本或攻击预览规则。
- 不新增建筑树、人口、装备、科技树扩展、外交界面或真实补给线系统。
- 不拆分 `BattleView.swift` 作为专项重构；只允许新增本轮必要的小型视图和 overlay。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、地形、单位、城市、敌军意图、战线压力、战场焦点位于 `Sources/RomeLegionsCore/GameState.swift`。
- `Position.neighbors(width:height:)`、`hexDistance(to:)`、`tile(at:)`、`unit(at:)`、`city(at:)`、`effectiveMovement(for:)`、`effectiveAttack(for:)`、`aiIntents(for:limit:)` 和 `frontlinePressureReports(against:perFactionLimit:limit:)` 可作为只读计算来源。
- `GameViewModel` 已有 `FrontlinePressureSummary`、`BattlefieldFocusSummary`、`LegionFormationSummary` 和 `TacticalRecommendationSummary` 模式。
- `BattleView` 已有 `WarMapView`、`HexTileView`、`TacticalStatusStripView`、`BattlefieldFocusPanelView` 和 `StrategicBalancePanelView`，可增量接入低透明 overlay、chip、卡片和行。
- 当前 CI 尚未运行 RenderBattlePreview；本轮若把 RenderBattlePreview 作为验收项，必须同步修改 `.github/workflows/ci-results.yml`。

## 实现步骤

1. 在 `GameState.swift` 新增：
   - `MapControlState`
   - `ThreatHeatLevel`

   - `ThreatHeatZoneReport`
2. 新增只读入口：
   - `mapControlReport(at:for:) -> MapControlReport?`
   - `mapControlReports(for:) -> [MapControlReport]`
   - `threatHeatZoneReports(for:limit:) -> [ThreatHeatZoneReport]`
3. 控制与热区计算规则：
   - 单位占据格、攻击射程、相邻格、可移动投射和城市归属都可以贡献影响值。
