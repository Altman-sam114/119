# v0.42（将领指挥链读板）

## 版本号

v0.42

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.41（处境命令入口联动），并已通过 `origin/main` 最新云端结果包复判。v0.41 已让选中军团处境读板指向既有命令入口；下一步应把将领被动、技能目标、战功、将令协同、目标线阶段和处境入口压缩成一条“指挥链”，让玩家在将领卡内快速理解这名将领当前能贡献什么、技能是否有目标、战功处于什么阶段、应看哪个命令入口。递增为 v0.42。

## 背景

当前项目已有：

- `SelectedCommanderBrief`：选中单位将领名、特性、被动贡献、技能状态、技能效果、战功摘要。
- `SelectedGeneralSkillTargetReadout`：主动技能目标数量、收益、地图标记和目标短列表。
- `CommanderActionGuidance`：将领技能入口与目标线“2 将令”阶段的同源 cue。
- `CommanderSynergySummary`：本方将令协同、协同步骤、预计收益和阻塞原因。
- `BattleObjectiveStageCommandPreview`：目标线阶段命令入口、技能阶段 cue、推荐姿态和下一步。
- `SelectedUnitSituationReadout`：选中军团处境与主命令入口。
- `BattleView` 已在选中单位情报面板中展示处境、编制、成长、军议、机动、将令、姿态和将领卡。

但将领卡内部仍需要玩家跨多行拼接“被动加成、技能目标、战功、将令协同、目标线/处境入口”的关系。v0.42 应新增一条只读、短高度的将领指挥链读板，放在将领卡内部，不新增命令按钮，不复制完整目标线或处境卡。

## 目标

1. 在 `GameViewModel.swift` 新增将领指挥链只读派生：
   - 建议新增 `SelectedCommanderChainSignalKind`、`SelectedCommanderChainSignal` 和 `SelectedCommanderChainReadout`。
   - 新增 `selectedCommanderChainReadout` computed property。
   - 只读取 `selectedUnit`、`selectedCommanderBrief`、`selectedGeneralSkillTargetReadout`、`selectedWarMeritStatus`、`selectedCommanderActionGuidance`、`selectedCommanderSynergySummary`、`selectedBattleObjectiveStageCommandPreview`、`selectedUnitSituationReadout`。
   - 输出字段建议包含 `unitID`、`title`、`statusLabel`、`passiveLabel`、`skillTargetLabel`、`warMeritLabel`、`entryLabel`、`summaryLabel`、`signals`、同源 ID 和 `accessibilityLabel`。
   - signal 只保存 kind、标题、详情、sourceID；不要保存 SwiftUI `Color`。
   - 提供 references 方法或等价同源校验入口，供 `RenderBattlePreview` 检查。
2. 在 `BattleView.swift` 展示短读板：
   - 在 `CompactGeneralTraitView` 和 `GeneralTraitCardView` 内部展示，插入到被动贡献 strip 之后、技能状态/技能详情之前。
   - 紧凑版只显示一条低高度 row，最多三段信息：被动、技能目标、入口。
   - 完整版仍保持一条读板，可显示被动、技能目标、战功和入口，但不展开目标列表。
   - SwiftUI 只消费 ViewModel 派生字段，不重新计算将领入口、技能目标、战功或目标线关系。
3. 在 `Tools/RenderBattlePreview/main.swift` 更新云端断言：
   - 新增错误 case，建议 `missingCommanderChainReadout`。
   - 在现有 `selectedCommanderBrief`、`selectedCommanderActionGuidance`、`selectedGeneralSkillTargetReadout` 断言附近检查 `selectedCommanderChainReadout`。
   - 断言固定样本 `rome-legion-1` 的读板非空，引用同一将领 brief、技能目标 readout、战功状态、将令 guidance、将令 synergy、目标线阶段和处境入口。
   - 断言 `accessibilityLabel` 包含“被动/目标/战功/将令/入口”等核心语义。
   - 断言读取指挥链读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.42`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.41 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、将领技能规则、战功规则、AI 权重、目标线评分、将令评分、反制评分或处境入口优先级。
- 不新增命令队列、自动执行、一键发动技能、一键连招、多回合搜索或真实作战计划执行器。
- 不改变移动、攻击、姿态、技能、结束回合、城市、外交或存档逻辑。
- 不新增地图叠层规则；本轮只解释既有将领/将令/目标线/处境入口关系。
- 不新增第三方框架，不重排整个情报面板，不做大型 SwiftUI 拆分或无关重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `selectedCommanderBrief` 已集中提供将领名、被动贡献、技能状态和战功摘要。
- `selectedGeneralSkillTargetReadout` 已同源绑定 `selectedGeneralSkillPreview` 的目标位置、目标数量和收益。
- `selectedCommanderActionGuidance` 已把将领技能入口与目标线 synergy 阶段同源串联。
- `selectedCommanderSynergySummary` 已提供将令协同的目标、收益、状态和步骤。
- `selectedBattleObjectiveStageCommandPreview` 可证明选中将领是否参与目标线“2 将令”或其他阶段命令。
- `selectedUnitSituationReadout.primaryCommandEntry` 可证明将领指挥链与 v0.41 的处境入口一致。
- `CompactGeneralTraitView` 和 `GeneralTraitCardView` 是最小 UI 落点；不要新增顶层卡片，否则短横屏容易拥挤。

## 验收标准

- `selectedCommanderChainReadout` 能在确定性战斗预览场景中为 `rome-legion-1` 生成。
- 指挥链读板与 `selectedCommanderBrief`、`selectedGeneralSkillTargetReadout`、`selectedWarMeritStatus`、`selectedCommanderActionGuidance`、`selectedCommanderSynergySummary`、`selectedBattleObjectiveStageCommandPreview` 和 `selectedUnitSituationReadout` 中存在的来源同源。
- 标题、状态、被动、技能目标、战功、入口、summary、signals 和无障碍文案非空。
- 完整/紧凑将领卡都能读到指挥链短摘要，不在短横屏或竖屏明显溢出。
- 读取指挥链读板不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算技能目标、战功、将令、目标线阶段或处境入口。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：将领卡已有被动、技能、战功和目标收益，指挥链若做成大卡会重复且拥挤；实现必须压缩成一条短读板。
- 风险：固定 fixture 的技能目标数量可能随地图或单位位置变化；断言应核对同源数据，不硬编码目标列表文案。
- 风险：处境入口和目标线阶段可能因为聚焦状态改变而变化；断言应在稳定选择态下读取。
- 禁止把指挥链变成真实命令队列、自动技能、一键目标线或自动姿态系统。
- 禁止在 SwiftUI 中重新计算技能目标、战功、将令协同、目标线阶段或处境入口优先级。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
