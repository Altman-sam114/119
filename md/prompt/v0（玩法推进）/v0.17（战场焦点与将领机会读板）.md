# v0.17（战场焦点与将领机会读板）

## 版本号

v0.17

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.16（战术命令建议与补线路径读板）。本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”做可验证小步迭代，在敌军意图、战线压力、军团编制、战术建议和将领技能读板之上新增一个更接近大战略战棋体验的“战场焦点”：告诉玩家当前最该关注的是救线、打击、将领机会、推进城市还是整编军团，因此版本递增为 v0.17。

## 背景

项目已经能分别展示敌军意图、战线压力、本方军团编制、选中单位战术建议和将领技能状态。但这些读板目前仍相对分散：玩家需要自己把“哪里危险”“哪支军团能补线”“哪个将领技能可用”“当前该推进哪个目标”拼成局面判断。

本轮目标是新增只读战场焦点报告：核心层综合现有只读报告，输出一条或多条“当前焦点”；ViewModel 转成 UI 摘要；BattleView 在顶部态势、战场面板和战局面板中展示。该能力只解释战局，不自动执行命令，不改变 AI 决策和结算。

## 目标

1. 在 `RomeLegionsCore` 新增只读战场焦点模型，按阵营生成焦点报告列表。
2. 焦点类型至少覆盖：紧急防守、将领机会、攻击机会、补线支援、城市推进、整编恢复。
3. 焦点报告至少包含：阵营、类型、严重度、焦点位置、推荐单位、目标单位/城市、相关单位、建议姿态、分数、标题、摘要和详情。
4. 焦点必须由核心层生成，不在 SwiftUI 中重新评分或复制规则。
5. 焦点报告调用不改变 `GameState`，不新增 Codable 存档字段。
6. 不改变真实移动、攻击、AI 决策、AI 评分、姿态切换、技能使用、招募、城市扩建或胜负结算。
7. `GameViewModel` 新增 UI 友好的战场焦点摘要，并提供 `battlefieldFocusSummaries` 与 `primaryBattlefieldFocusSummary`。
8. `BattleView` 在顶部态势条展示首要“焦点”chip，在现有 `BattlefieldFocusPanelView` 展示焦点卡，在完整战局面板展示前几条焦点行；紧凑界面仍保持“军令”优先。
9. Swift Testing、Gameplay Smoke 和 RenderBattlePreview 覆盖核心焦点字段、只读不变性、压力驱动焦点、将领技能机会焦点和 UI 主链路所需字段。

## 非目标

- 不实现自动执行焦点命令、任务脚本、多回合规划、AI 搜索、真实战略路线系统或自动玩家回合。
- 不修改 `performSimpleAI(for:)`、`aiIntent(for:)`、AI 权重、真实 `moveUnit`、`attack`、`useGeneralSkill`、城市占领或战役胜负结算。
- 不改变现有战术姿态数值、将领技能数值、战功公式、城市招募成本或地形移动成本。
- 不引入真实编制槽位、装备、科技树扩展、人口、建筑树或外交界面。
- 不拆分 `BattleView.swift` 作为专项重构；只允许新增本轮必要的小型视图。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、敌军意图、战线压力、军团编制、战术建议、将领技能预览位于 `Sources/RomeLegionsCore/GameState.swift`。
- `frontlinePressureReports(against:perFactionLimit:limit:)` 已公开防守焦点候选。
- `tacticalRecommendation(unitID:)` 已公开本方单位攻击、补线、推进、坚守或整备建议。
- `legionFormationReports(for:limit:)` 已公开军团战备、支援、近敌、技能状态和建议姿态。
- `generalSkillPreview(unitID:)` 已公开将领技能机会和阻塞原因。
- `GameViewModel` 已有 `FrontlinePressureSummary`、`LegionFormationSummary` 和 `TacticalRecommendationSummary` 的摘要模式。
- `BattleView` 已有 `TacticalStatusStripView`、`BattlefieldFocusPanelView`、`StrategicBalancePanelView` 和对应行/卡片视图，可增量接入。

## 实现步骤

1. 在 `GameState.swift` 新增 `BattlefieldFocusKind`、`BattlefieldFocusSeverity` 和 `BattlefieldFocusReport`。
2. 新增 `GameState.battlefieldFocusReports(for:limit:) -> [BattlefieldFocusReport]` 和 `battlefieldFocusReport(for:) -> BattlefieldFocusReport?`：
   - 仅根据现有 `GameState` 派生，不写入任何字段。
   - 从战线压力报告生成紧急防守/补线焦点。
   - 从战术建议报告生成攻击机会、补线支援和城市推进焦点。
   - 从军团编制报告中技能就绪的将领生成将领机会焦点。
   - 从危急/吃紧军团编制报告生成整编恢复焦点。
   - 统一按分数排序并返回 limit 条；同分按 id 稳定排序。
3. `GameViewModel.swift` 新增 `BattlefieldFocusSummary`，提供：
   - `title`
   - `kindLabel`
   - `severityLabel`
   - `targetLabel`
   - `unitLabel`
   - `scoreLabel`
   - `detail`
   - `accessibilityLabel`
   并新增 `battlefieldFocusSummaries`、`primaryBattlefieldFocusSummary`。
4. `BattleView.swift`：
   - 顶部 `TacticalStatusStripView` 增加首要“焦点”chip。
   - `BattlefieldFocusPanelView` 在地形/补给信息附近展示焦点卡；紧凑模式使用一行短摘要。
   - `StrategicBalancePanelView` 在战线压力前展示前 2 条焦点行。
   - 不改变紧凑命令顺序：`CompactActionsPanelView` 仍在 `CompactSelectionPanelView` 前。
5. `Tests/RomeLegionsCoreTests/GameStateTests.swift`：
   - 新增战线压力驱动的紧急防守焦点测试，并断言只读不变性。
   - 新增将领技能就绪驱动的将领机会焦点测试，并断言只读不变性。
6. `Tools/GameplaySmoke/main.swift`：
   - 增加战场焦点主链路断言：焦点存在、类型/严重度/标题/详情可读、调用不改变状态。
7. `Tools/RenderBattlePreview/main.swift`：
   - 在单位场景断言中加入 `primaryBattlefieldFocusSummary` 非空、焦点类型、严重度、目标、详情和无障碍文案可用。
8. 更新文档：
   - `README.md` 已实现能力和预览断言说明。
   - `md/flow/flow.md` 当前数据流、战场焦点流程、核心对象和扩展点。
   - `md/flow/flowchart.md` 战斗流程图加入战场焦点报告。
   - `md/test/test.md` 测试基线、Gameplay Smoke、RenderBattlePreview 断言和 artifact 版本。
   - `md/prompt/README.md` 提示词目录追加 v0.17。
   - `update_log.md` 追加 v0.17 正式记录。
   - `.github/workflows/ci-results.yml` 的 `CI_VERSION` 更新为 `v0.17`。
