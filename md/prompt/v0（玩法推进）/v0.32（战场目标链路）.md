# v0.32（战场目标链路）

## 版本号

v0.32

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.31（反制焦点链路）。v0.31 已把敌情反制建议的“1 回应 -> 2 落点 -> 3 目标”串成可读链路，但战场焦点、将令协同、机动落点和本方军议仍分别展示，玩家还需要自己判断这些读板如何合成一条当前作战目标线。下一步应把这些已有只读报告组织成更清晰的“战场目标链路”，递增为 v0.32。

## 背景

当前项目已有：

- 战场焦点：`BattlefieldFocusSummary` 展示高优先目标、严重度、执行单位和建议姿态。
- 本方将令：`CommanderSynergySummary` 展示将领技能、合击、补线、推进或整备协同。
- 机动落点：`ManeuverOptionSummary` 展示选中单位可移动落点、风险、控区和目标。
- 战术军议：`TacticalRecommendationSummary` 展示选中单位攻击、补线、推进、坚守或整备建议。
- 地图叠层：已有本方建议路径/目标、机动落点、热区、控区、敌路和反制叠层。

当前短板是：这些读板都能单独说明一件事，但还缺少一个轻量的“本回合目标线”把焦点目标、将令机会、机动落点和军议目标串起来。v0.32 应只在 ViewModel 和 SwiftUI 层把已有派生报告连成一条可读链路，不改变任何核心评分、排序、移动、攻击、AI 或技能结算。

## 目标

1. 在 `GameViewModel.swift` 增加只读 `BattleObjectiveChainSummary` 或等价结构，至少覆盖：
   - 焦点阶段，例如“1 焦点 <目标>”。
   - 将令阶段，例如“2 将令 <目标/技能/协同>”。
   - 机动阶段，例如“3 机动 <落点>”。
   - 军议阶段，例如“4 军议 <目标>”。
   - 完整链路摘要、短标题、风险/优先文案和无障碍文案。
   - 能判断链路是否引用当前焦点、将令、机动和军议摘要。
2. 给 `BattlefieldFocusSummary`、`CommanderSynergySummary`、`ManeuverOptionSummary`、`TacticalRecommendationSummary` 增加轻量目标链路 cue 字段，供卡片和云端断言复用。
3. 在 `BattleView.swift` 中展示目标链路：
   - `BattlefieldFocusPanelView` 完整布局显示目标链路卡。
   - 紧凑布局显示一行链路摘要或短 cue，避免挤压命令按钮。
   - 焦点、将令、机动和军议卡片可以显示各自目标链路 cue，但不要大规模重构布局。
4. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 断言目标链路摘要存在且字段非空。
   - 断言目标链路引用当前首要战场焦点、选中/首要将令、首要机动落点和选中军议。
   - 断言四类摘要的目标链路 cue 和无障碍文案可读。
   - 失败可新增 `missingBattleObjectiveChainSummary`，或复用已有错误枚举；建议新增，便于 Agent C 定位。
5. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.32`。
6. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.31 最新云端通过证据。

## 非目标

- 不新增核心规则字段，不修改 `GameState` 的评分、排序、路径、AI 或报告生成逻辑。
- 不自动执行目标线，不做一键移动、一键攻击、一键技能或命令队列。
- 不改变真实移动、攻击、技能、姿态、AI、城市、外交、胜负结算。
- 不新增多回合搜索、装备、升级树、将领池 UI、外交界面、存档 UI 或建筑树。
- 不引入第三方框架，不做大型 SwiftUI 文件拆分或全局重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `BattlefieldFocusSummary`、`CommanderSynergySummary`、`ManeuverOptionSummary` 和 `TacticalRecommendationSummary` 都是 `GameViewModel` 中的 UI 派生数据，适合增加只读展示字段。
- `GameViewModel.primaryBattlefieldFocusSummary`、`selectedCommanderSynergySummary`、`primaryManeuverOptionSummary` 和 `selectedTacticalRecommendationSummary` 已能在选中单位场景下同时生成。
- `BattlefieldFocusPanelView` 已集中展示战场焦点、热区、控区和选中地块信息，适合作为目标链路主入口。
- `TacticalRecommendationCardView`、`ManeuverOptionCardView` 和 `CommanderSynergyCardView` 已在完整/紧凑单位信息中展示，适合附加单行 cue。
- `Tools/RenderBattlePreview/main.swift` 已构造选中罗马军团场景，并断言焦点、将令、机动和军议读板存在，适合新增链路一致性断言。

## 实现步骤

1. 在 `GameViewModel.swift` 增加四类摘要的目标链路 cue 字段：
   - `BattlefieldFocusSummary.objectiveCueLabel`
   - `CommanderSynergySummary.objectiveCueLabel`
   - `ManeuverOptionSummary.objectiveCueLabel`
   - `TacticalRecommendationSummary.objectiveCueLabel`
2. 在 `GameViewModel.swift` 新增 `BattleObjectiveChainSummary`：
   - 持有首要焦点、可选将令、可选机动、可选军议。
   - 输出 `chainLabel`、`compactLabel`、`focusStageLabel`、`synergyStageLabel`、`maneuverStageLabel`、`recommendationStageLabel`、`accessibilityLabel`。
   - 提供 `references(...)` helper，供云端断言判断链路引用的是当前摘要。
3. 在 `GameViewModel` 增加 `primaryBattleObjectiveChainSummary`，从现有只读摘要组合生成，不写回 `GameState`。
4. 在 `BattleView.swift` 新增目标链路卡或轻量 view：
   - 完整 `BattlefieldFocusPanelView` 在焦点卡附近展示链路卡。
   - 紧凑 `BattlefieldFocusPanelView` 显示 `compactLabel`。
   - 相关卡片展示各自 `objectiveCueLabel` 的一行文本，保持字体和布局克制。
5. 在 `Tools/RenderBattlePreview/main.swift` 新增断言：
   - 在焦点、将令、机动、军议摘要都获取后，断言目标链路存在。
   - 检查链路引用当前摘要，阶段标签、链路摘要、compact 文案、无障碍文案非空。
   - 检查四类 cue 非空。
6. 更新 workflow 版本号和文档。
7. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 战场面板能读出“焦点 -> 将令 -> 机动 -> 军议”的目标链路。
- 焦点、将令、机动和军议卡片能提供与目标链路一致的 cue。
- SwiftUI 不重新计算评分、路径、目标、技能或移动规则，只消费 ViewModel 派生字段。
- `GameState` 核心规则、AI、技能、移动、攻击、姿态和胜负结算不变。
- 云端 RenderBattlePreview 断言覆盖目标链路字段、引用一致性和四类 cue。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

- 禁止绕过 `GameViewModel` 现有命令入口。
- 禁止改变 `GameState` 核心报告的排序、评分、目标或路径。
