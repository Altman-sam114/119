# v0.29（反制指令聚焦与执行预览）

## 版本号

v0.29

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.28（反制落点地图叠层）。v0.28 已把首要反制建议投射到地图，展示回应位置、推荐落点和威胁目标。但玩家仍需要手动寻找回应军团、判断姿态能否切换、落点是否可达、目标是否能攻击。本轮递增为 v0.29，目标是在不自动下令的前提下，把反制建议变成可聚焦、可读阻塞原因的指令预览，让战斗界面继续接近“大征服者罗马”的战场指挥体验。

## 背景

当前项目已有：

- v0.27 敌情反制建议读板：`CountermeasureReport`、`CountermeasureSummary`、顶部 chip、敌情反制卡、战局反制行。
- v0.28 反制地图叠层：`primaryCountermeasureMapOverlay`、`countermeasureRouteSegments`、`countermeasureOverlaysByPosition`、反制图例和 RenderBattlePreview 断言。
- 选中单位命令体系：`selectTile(_:)`、`reachablePositions`、`attackTargets`、`selectedTacticalOrderPreviews`、`canSetSelectedTacticalOrder(_:)`、`setSelectedTacticalOrder(_:)`、`attack(_:)`。
- 战斗面板中已有军令按钮、敌情卡、战局行和紧凑/完整布局。

短板是反制建议还不能把玩家带到“下一步该怎么操作”的状态：读板告诉玩家“谁去哪里、对付谁”，但 UI 没有一键聚焦回应军团，也没有把推荐姿态、落点可达性、目标攻击窗口和阻塞原因整理成指令预览。

## 目标

1. 在 `GameViewModel.swift` 新增只读反制指令预览摘要，例如：
   - `CountermeasureCommandPreview`
   - `primaryCountermeasureCommandPreview`
   - `countermeasureCommandPreview(for:)`
2. 指令预览至少包含：
   - 关联 `CountermeasureSummary`。
   - 回应军团是否存在、是否为罗马、是否当前阵营、是否已行动。
   - 推荐姿态 `recommendedOrder` 的当前状态、是否可切换和阻塞原因。
   - 推荐落点 `destination` 是否为当前位置、是否可达或为何不可达。
   - 目标单位/城市和目标距离；若目标单位在攻击范围内，提供可攻击状态。
   - title、statusLabel、orderLabel、destinationLabel、targetLabel、nextStepLabel、blockedReason、accessibilityLabel。
3. 在 `GameViewModel.swift` 新增聚焦方法：
   - `focusCountermeasure(_ id: String)` 或 `focusPrimaryCountermeasure()`
   - 只改变 ViewModel 选择态和 banner，不改变 `GameState`。
   - 成功后选中反制回应单位、同步 selectedPosition / selectedCityID，让现有军令面板、可达格、攻击目标和姿态预览自然刷新。
4. 在 `BattleView.swift` 展示反制指令预览：
   - `CountermeasureCardView` 支持可选执行预览和“定位回应”按钮。
   - 敌情面板的首要反制卡显示预览，并允许聚焦回应军团。
   - 完整战局面板的反制行可显示更短的下一步/状态文案；如改动行结构，保持紧凑稳定。
   - 军令面板可在选中回应军团后显示“反制建议”提示，说明推荐姿态、落点和目标，不自动执行移动/攻击/姿态切换。
5. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - `primaryCountermeasureCommandPreview` 存在。
   - title/status/order/destination/target/nextStep/accessibility 文案非空。
   - 预览关联首要 `primaryCountermeasureSummary`。
   - 聚焦方法能选择首要反制回应军团，并使 `selectedUnitID == responseUnitID`、`focusedPosition == responsePosition` 或该单位当前位置。
   - 失败抛出新的 `missingCountermeasureCommandPreview`。
6. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.29`。
7. 更新 README、flow、flowchart、test、prompt README、update_log；保留 v0.28 最新云端通过证据。

## 非目标

- 不自动执行反制，不新增“一键移动/一键攻击/一键切姿态”。
- 不改变 `GameState.countermeasureReports` 评分、排序、目标匹配或核心字段。
- 不改变 AI 评分、AI 执行、移动、攻击、技能、姿态、城市、外交、胜负结算。
- 不引入第三方框架，不拆分大型 SwiftUI 文件，不做全局 UI 重构。
- 不新增多回合搜索、装备、将领池 UI、科技树、建筑树、存档 UI 或复杂命令队列。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `CountermeasureReport` 已提供 responseUnitID、responsePosition、recommendedOrder、destination、targetPosition、targetUnitID、targetCityID。
- `GameViewModel` 已负责选择态、命令态、预览数据和 UI 派生数据，是新增反制指令预览和聚焦方法的正确位置。
- `BattleView` 的 `CountermeasureCardView`、`CountermeasureRowView`、`EnemyIntentPanelView`、`StrategicBalancePanelView` 和 `ActionsPanelView` 已有合适的展示入口。
- `Tools/RenderBattlePreview/main.swift` 已有反制摘要和地图叠层断言，可继续扩展指令预览与聚焦行为断言。
- 当前验证固定走 GitHub Actions artifact，不能本地跑测试或渲染。

## 实现步骤

1. 在 `GameViewModel.swift` 定义 `CountermeasureCommandPreview`，保持纯 UI 派生，不写入核心状态。
2. 实现 `countermeasureCommandPreview(for:)`：
   - 根据 `summary.report.responseUnitID` 查找回应单位。
   - 用 `state.reachablePositions(for:)` 判断 `destination` 是否可达。
   - 用 `state.attackTargets(for:)` 和目标 ID 判断目标是否可攻击。
   - 复用 `tacticalOrderBlockedReason` 或同等逻辑判断推荐姿态状态。
   - 输出稳定中文文案和无障碍文案。
3. 实现 `focusCountermeasure(_:)`：
   - 找到对应 summary 和 responseUnit。
   - 设置 `selectedUnitID`、`selectedPosition`、`selectedCityID`。
   - 更新 `bannerMessage`，提示推荐姿态、落点和目标。
4. 在 `BattleView.swift`：
   - 敌情反制卡传入 `viewModel.primaryCountermeasureCommandPreview` 和聚焦 action。
   - `CountermeasureCardView` 增加指令预览区域和“定位回应”按钮，按钮使用 `Button("定位回应", systemImage: ...)` 或带文本的 label，保证 VoiceOver 可读。
   - 军令面板选中回应军团时展示一个小型 `CountermeasureCommandPreviewView`，只提示下一步，不触发命令。
5. 在 `Tools/RenderBattlePreview/main.swift` 增加 `missingCountermeasureCommandPreview` 断言。
6. 更新 workflow 版本号和文档。
7. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 和 Agent C artifact 复判。

## 验收标准

- 首要反制建议有可读指令预览，说明推荐姿态、落点、目标、下一步和阻塞原因。
- 玩家可以从反制卡聚焦回应军团，现有军令面板随选择态刷新。
- 聚焦只改变 ViewModel 选择态和 banner，不改变核心 `GameState` 战局。
- SwiftUI 不重新计算反制评分或核心规则，只展示 ViewModel 派生数据。
- 云端 RenderBattlePreview 断言覆盖指令预览与聚焦行为。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止把“定位回应”实现成自动移动、自动攻击或自动切姿态。
- 禁止在 SwiftUI `body` 中重新实现核心规则、路径、攻击目标或反制评分。
- 禁止让聚焦方法写入 `GameState` 或消耗单位行动。
- 禁止扩大为自动战斗、命令队列、多回合搜索或全局 UI 重构。
- 禁止运行任何本地测试、build、typecheck、RenderBattlePreview、`verify_project`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。
- 如果 GitHub 权限、网络或 artifact 下载失败，必须明确阻塞，不能伪造云端通过。
