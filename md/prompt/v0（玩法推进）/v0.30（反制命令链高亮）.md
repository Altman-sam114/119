# v0.30（反制命令链高亮）

## 版本号

v0.30

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.29（反制指令聚焦与执行预览）。v0.29 已经让玩家从敌情反制卡定位回应军团，并在军令面板看到推荐姿态、落点、目标和下一步。但反制预览仍停留在“读板提示”层，现有姿态按钮和攻击按钮没有标出哪一个是本条反制建议对应的推荐命令。下一步应把反制读板继续接到已有命令入口，递增为 v0.30。

## 背景

当前项目已有：

- v0.27 敌情反制建议读板：`CountermeasureReport`、`CountermeasureSummary`、顶部反制 chip、敌情反制卡、战局反制行。
- v0.28 反制落点/目标地图叠层：首要反制建议的回应位置、推荐落点、威胁目标和地图引导线。
- v0.29 反制指令聚焦与执行预览：`CountermeasureCommandPreview`、`focusedCountermeasureID`、`focusCountermeasure(_:)`、军令面板反制预览和 RenderBattlePreview 断言。
- 现有命令入口：`TacticalOrderControlView`、`TacticalOrderPreviewButtonContent`、完整/紧凑 `ActionsPanelView` 的攻击按钮、地图可移动格和攻击目标徽标。

当前短板是：玩家看到“建议坚守 / 可直接攻击目标”等文字后，还要自己在按钮组中寻找对应姿态或攻击按钮。本轮目标是在不自动执行的前提下，让现有命令按钮明确显示“这是反制建议对应的下一步”。

## 目标

1. 扩展 `CountermeasureCommandPreview` 的只读 UI 派生字段，至少覆盖：
   - 推荐姿态是否已经是当前姿态。
   - 推荐姿态按钮是否应显示反制提示。
   - 当前可攻击目标中哪个是反制目标。
   - 反制目标攻击按钮的标题、详情和无障碍文案。
   - 当前命令链的短标签，例如“先切换坚守”“先移动至 (x,y)”“反制攻击目标”。
2. 在 `BattleView.swift` 中高亮现有姿态按钮：
   - `TacticalOrderControlView` 读取 `viewModel.selectedCountermeasureCommandPreview`。
   - `TacticalOrderPreviewButtonContent` 支持可选反制推荐标记。
   - 推荐姿态按钮显示“反制”或等价小标签，并用现有色彩系统增加边框/底色强化。
   - 不改变按钮原有 disabled 规则，不绕过 `canSetSelectedTacticalOrder(_:)`。
3. 在完整和紧凑军令面板中高亮反制攻击目标：
   - 如果 `selectedCountermeasureCommandPreview.targetUnit?.id` 出现在 `viewModel.attackTargets` 中，对应攻击按钮显示“反制目标”或“反制攻击”提示。
   - 保留原有攻击按钮实际 action：`viewModel.attack(target.id)`。
   - 不新增自动执行，不新增命令队列。
4. 强化 `CountermeasureCommandPreviewView`：
   - 展示当前命令链短标签、推荐姿态、落点和目标状态。
   - 文案必须在紧凑和完整布局中可读，避免明显溢出。
   - 使用带文本的按钮/Label 或可读 accessibilityLabel，符合 VoiceOver 要求。
5. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - `primaryCountermeasureCommandPreview` 除 v0.29 字段外，还要断言命令链短标签、推荐姿态高亮文案和攻击目标高亮文案非空。
   - 聚焦反制后，若反制目标在 `attackTargets` 中，应能通过 ViewModel 派生字段确认攻击按钮可标记为反制目标。
   - 失败继续使用或细分为 `missingCountermeasureCommandPreview`，不必新增多余错误枚举，除非实现需要。
6. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.30`。
7. 更新 README、flow、flowchart、test、prompt README、update_log；保留 v0.29 最新云端通过证据。

## 非目标

- 不自动执行反制，不做一键移动、一键攻击、一键切姿态或命令队列。
- 不改变 `GameState.countermeasureReports` 评分、排序、目标匹配或核心字段。
- 不改变移动、攻击、技能、姿态、AI、城市、外交、胜负结算。
- 不新增多回合搜索、装备、升级树、将领池 UI、外交界面、存档 UI 或建筑树。
- 不重构大型 SwiftUI 文件结构，不引入第三方框架。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `CountermeasureCommandPreview` 已经集中保存推荐姿态、落点、目标、可聚焦状态、可达性、目标攻击状态、步骤和无障碍文案。
- `selectedCountermeasureCommandPreview` 会优先返回 `focusedCountermeasureID` 对应建议，适合驱动当前选中单位的军令面板高亮。
- `TacticalOrderControlView` 已经从 `viewModel.selectedTacticalOrderPreview(for:)` 读取每个姿态按钮状态。
- 完整/紧凑 `ActionsPanelView` 已经从 `viewModel.attackTargets` 和 `viewModel.attackPreview(for:)` 渲染攻击按钮。
- `Tools/RenderBattlePreview/main.swift` 已经会聚焦首要反制建议并断言 `selectedCountermeasureCommandPreview`、`selectedTacticalOrderPreviews` 和 banner。

## 实现步骤

1. 在 `GameViewModel.swift` 中扩展 `CountermeasureCommandPreview`：
   - 增加 `commandChainLabel`、`recommendedOrderCueLabel`、`attackCueLabel`、`movementCueLabel` 等只读文案。
   - 增加 `func isCountermeasureAttackTarget(_ unit: ArmyUnit) -> Bool` 或 ViewModel 层等价 helper，例如 `selectedCountermeasureAttackCue(for:)`。
   - 如需抽象，新增轻量 `CountermeasureCommandCue`，但不要过度设计。
2. 在 `BattleView.swift` 中扩展 `CountermeasureCommandPreviewView`：
   - 在完整布局显示命令链短标签和步骤状态。
   - 紧凑布局只显示一行或两行关键提示。
3. 在 `TacticalOrderControlView` / `TacticalOrderPreviewButtonContent`：
   - 从 `viewModel.selectedCountermeasureCommandPreview` 判断当前 `order == preview.recommendedOrder`。
   - 推荐姿态按钮增加反制标签和边框/底色。
   - 保持 `.disabled(!(preview?.canSwitch ?? viewModel.canSetSelectedTacticalOrder(order)))` 等原逻辑不变。
4. 在完整/紧凑 `ActionsPanelView` 的攻击按钮：
   - 对反制目标攻击按钮显示反制 cue。
   - 保持 `viewModel.attack(target.id)` action 不变。
   - 保持 `PrimaryButtonStyle` / `SecondaryButtonStyle` 现有体系，不引入新设计系统。
5. 在 `Tools/RenderBattlePreview/main.swift` 添加断言：
   - 新增文案字段非空。
   - 聚焦反制后推荐姿态 cue 可被读取。
   - 如果目标当前可攻击，攻击 cue 可被读取。
6. 更新 workflow 版本号和文档。
7. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- 聚焦反制回应军团后，军令面板能显示反制命令链短标签。
- 推荐姿态按钮有反制高亮或标签，但不改变原有 disabled 和 action。
- 如果反制目标当前可攻击，对应攻击按钮有反制高亮或标签，但 action 仍为普通攻击入口。
- SwiftUI 不重新计算反制评分、路径、攻击目标或姿态规则，只消费 ViewModel 派生字段。
- 云端 RenderBattlePreview 断言覆盖新增 cue 字段和聚焦后的命令入口标记。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。

## 风险和禁止项

- 禁止把高亮实现成自动执行或半自动执行命令。
- 禁止在 SwiftUI `body` 中重新计算反制评分、路径、目标匹配、攻击范围或姿态规则。
- 禁止绕过 `GameViewModel.setSelectedTacticalOrder(_:)`、`attack(_:)` 等现有入口。
- 禁止让高亮覆盖真实 disabled 状态。
- 禁止扩大为自动战斗、命令队列、多回合搜索或全局 UI 重构。
- 禁止运行任何本地测试、build、typecheck、RenderBattlePreview、`verify_project`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。
- 如果 GitHub 权限、网络或 artifact 下载失败，必须明确阻塞，不能伪造云端通过。
