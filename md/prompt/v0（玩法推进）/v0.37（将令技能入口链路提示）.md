# v0.37（将令技能入口链路提示）

## 版本号

v0.37

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.36（目标线阶段联动高亮），并已通过 `origin/main` 最新云端结果包复判。v0.36 已把目标线阶段、地图徽标、姿态、攻击和技能入口做了联动高亮，但将领卡、将令协同卡、目标线“2 将令”阶段和技能按钮之间仍缺少同源行动引导。下一轮应让玩家更清楚“当前将领机会对应哪个阶段、哪个按钮、是否可发动”，递增为 v0.37。

## 背景

当前项目已有：

- `CommanderSynergySummary`：本方将令协同摘要，展示将领技能、合击、补线、推进或整备机会。
- `SelectedCommanderBrief`：选中单位将领读板，展示将领名、被动贡献、技能状态、技能效果和战功。
- `BattleObjectiveStageCommandPreview`：目标线阶段命令预览，其中 `.synergy` 阶段对应“2 将令”。
- `selectedGeneralSkillPreview`、`selectedGeneralSkillButtonDetail`、`canUseSelectedGeneralSkill`：技能按钮现有状态来源。
- `BattleView` 完整/紧凑军令面板已经在技能按钮 detail 中拼接目标线技能 cue，但两处逻辑重复，且将领卡本身没有读出同一行动链路。

当前短板是：玩家能看到将令协同、目标线“2 将令”和技能按钮，但仍需要自己判断这些是否属于同一个将领行动窗口。v0.37 应新增只读的将领行动引导，把将令协同、目标线阶段、技能预览和按钮 detail 统一起来。

## 目标

1. 在 `GameViewModel.swift` 增加将领行动引导的只读派生：
   - 建议新增 `CommanderActionGuidance` 或同等轻量结构，包含 `title`、`stageCueLabel`、`skillCueLabel`、`buttonDetailPrefix`、`statusLabel`、`accessibilityLabel`、`isLinkedToBattleObjectiveStage` 等字段。
   - 建议新增 `selectedCommanderActionGuidance`，从 `selectedCommanderSynergySummary`、`selectedBattleObjectiveStageCommandPreview`、`selectedCommanderBrief` 和 `selectedGeneralSkillPreview` 只读派生。
   - 建议新增 `selectedGeneralSkillCommandButtonDetail`，统一完整/紧凑技能按钮 detail 拼接，避免 `BattleView` 两处重复逻辑。
   - 不新增 `@Published` 状态，不修改 `GameState`，不保留 stale guidance。
2. 在 `BattleView.swift` 展示将领行动引导：
   - `GeneralTraitCardView` / `CompactGeneralTraitView` 或 `CommanderSkillStatusRow` 应展示同源的将领行动引导 cue，例如“2 将令 · 将领技能可用 / 将令需确认”。
   - 完整/紧凑技能按钮 detail 改用 `viewModel.selectedGeneralSkillCommandButtonDetail`。
   - 若 `selectedBattleObjectiveStageCommandPreview` 为空或不是 `.synergy`，应降级为现有技能预览文案，不显示误导性目标线阶段。
   - 反制、目标线、技能按钮 action 与 disabled 规则不变。
3. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingCommanderActionGuidance`。
   - 断言 `selectedCommanderActionGuidance` 与 `selectedCommanderSynergySummary`、`selectedGeneralSkillPreview`、目标线 `.synergy` 阶段预览同源。
   - 断言 `selectedGeneralSkillCommandButtonDetail` 包含同一引导 cue，并与 `canUseSelectedGeneralSkill` / `selectedCommanderBrief.skillStatusLabel` 一致。
   - 继续证明只读：不得改变单位位置、生命、行动标记、姿态、技能冷却、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.37`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.36 云端通过证据。

## 非目标

- 不新增真实命令、自动发动技能、一键技能、命令队列、自动战斗或多回合规划。
- 不修改 `GameState` 的将领技能规则、技能目标筛选、冷却、战斗结算、AI、目标线排序或将令评分。
- 不改变现有技能按钮、攻击按钮、姿态按钮、移动、结束回合、城市或 AI 的 action / disabled 逻辑。
- 不把引导状态写入 `GameState` 或新增持久存档字段。
- 不在 SwiftUI `body` 中重新计算将令评分、技能目标、技能范围、目标线阶段或可执行状态。
- 不新增装备、升级树、将领池 UI、外交界面、存档 UI、建筑树或第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 验收标准

- 选中带将领的罗马单位后，将领卡能显示将令技能入口链路提示。
- 若目标线 `.synergy` 阶段与当前选中单位同源，提示应带“2 将令”阶段 cue，并与技能按钮 detail 使用同一来源。
- 若没有目标线 `.synergy` 阶段或技能不可用，提示应降级为现有技能状态/阻塞原因，不误导玩家。
- 完整/紧凑技能按钮 detail 由同一个 ViewModel 字段生成，不再两处手动重复拼接。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算技能范围、目标、可用性、将令评分或目标线阶段。
- 引导不改变 `GameState` 核心状态，不移动、不攻击、不发动技能、不切换姿态。
- 云端 RenderBattlePreview 断言覆盖将领行动引导、同源性、按钮 detail 和只读性。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。
