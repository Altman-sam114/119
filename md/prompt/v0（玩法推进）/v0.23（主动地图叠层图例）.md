# v0.23（主动地图叠层图例）

## 版本号

v0.23

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.22（AI 主攻优先执行）。Agent X 总目标要求继续向“大征服者罗马”推进，持续优化战斗界面 UI、地图、AI 和将领。本轮接续 v0.18 地图控制/威胁热区、v0.21 机动落点和 v0.22 AI 执行一致性，处理地图可读性的下一个短板：地图已经同时显示敌军路线、目标格、热区、控区、军议路径和机动落点，但底部图例仍只解释阵营颜色。因此版本递增为 v0.23。

## 背景

`WarMapView` 当前会收集并展示多种地图叠层：敌军意图路线、敌军目的地/目标格、本方军议路径和目标、机动落点、地图控制、威胁热区、可移动格和攻击目标等。玩家在地图上看到折线、虚线落点、热区火焰或控区底色时，需要回到侧栏推断含义。

本轮新增“主动地图叠层图例”：`GameViewModel` 根据当前实际存在的叠层派生少量图例项，`BattleView` 在地图底部展示图标、标题和短说明。它只解释当前可见叠层，不新增玩法规则，不重新计算路径、热区、控区或目标。

## 目标

1. 在 `GameViewModel.swift` 新增 UI 派生模型：
   - `MapOverlayLegendKind`
   - `MapOverlayLegendItem`
   - `activeMapOverlayLegendItems`
2. 图例项必须只来自现有 ViewModel 派生数据，不直接在 SwiftUI 中重新判断复杂规则。
3. 动态图例至少覆盖当前实际出现的：
   - 敌军路线
   - 敌军目标/目的地
   - 威胁热区
   - 地图控区
   - 军议路径/目标
   - 机动落点
   - 可移动格
   - 可攻击目标
   - 将领技能范围/目标（存在时）
4. 每个图例项包含稳定 id、kind、symbol、title、detail、accessibilityLabel。
5. `BattleView` 将 `MiniLegendView()` 替换或扩展为 `MapOverlayLegendView(items:)`：
   - 保留阵营色说明。
   - 优先展示当前动态叠层项。
   - 在紧凑宽度中允许横向滚动或换行，避免挤压地图和文本溢出。
   - 图例项必须有图标 + 文本，不只靠颜色。
6. `MapOverlayLegendView` 整体提供无障碍 label，汇总当前图例含义。
7. `Tools/RenderBattlePreview/main.swift` 增加云端断言：
   - `viewModel.activeMapOverlayLegendItems` 非空。
   - 确定性单位场景中至少包含敌军路线、敌军目标/目的地、威胁热区、地图控区、军议路径或目标、机动落点。
   - 每项 title/detail/accessibilityLabel 非空。
   - 失败时抛出 `missingMapOverlayLegend`。
8. 更新 README、flow、flowchart、test、prompt README、update_log 和 CI 版本号。

## 非目标

- 不改变 `GameState` 核心规则、AI 行为、移动/攻击/夺城/技能/姿态/热区/控区/机动评分。
- 不新增地图叠层算法，不重算敌军路线、热区、控区、机动落点或军议路径。
- 不做 BattleView 大拆分或全局视觉重做。
- 不新增第三方框架，不引入原游戏素材，不改变技术栈。
- 不实现可配置图层开关、过滤器、缩放、教程或新手引导。

## 当前架构依据

- `GameViewModel.enemyIntentMapOverlays`、`enemyIntentDestinationOverlays(for:)` 和 `enemyIntentTargetOverlays(for:)` 已提供敌军路线、目的地和目标格。
- `selectedTacticalRecommendationSummary`、`selectedTacticalRecommendationPathPositions` 和 `selectedTacticalRecommendationTargetPosition` 已提供本方军议路径和目标。
- `maneuverOptionOverlaysByPosition` / `maneuverOptionOverlayPositions` 已提供机动落点。
- `threatHeatZoneOverlaysByPosition` / `threatHeatOverlayPositions` 和 `mapControlOverlayPositions` 已提供热区/控区叠层位置。
- `reachablePositions`、`attackTargets`、`selectedGeneralSkillRangePositions`、`selectedGeneralSkillTargetPositions` 已提供可移动、可攻击和技能范围/目标。
- `BattleView.WarMapView` 当前在底部调用 `MiniLegendView()`，而 `MiniLegendView` 只显示 `Faction.turnOrder` 阵营色。
- `RenderBattlePreview` 已在渲染前验证多种 ViewModel 派生叠层存在，可增加图例字段断言。
