# v0.40（选中军团处境读板）

## 版本号

v0.40

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.39（战场态势交汇链路），并已通过 `origin/main` 最新云端结果包复判。v0.39 把目标线、反制、阶段命令、将令、机动、热区和控区压缩成战场总览；下一步应把同一批战场信号落到玩家当前选中的军团上，让“这个军团现在承压多少、有什么机会、下一步该看什么命令”更直接，递增为 v0.40。

## 背景

当前项目已有：

- `FrontlinePressureSummary`：敌军对本方单位或城市形成的战线压力。
- `ThreatHeatZoneSummary`：威胁热区、来源单位、预计伤害和控区状态。
- `MapControlSummary`：指定位置的控区、威胁等级、来源和影响。
- `LegionFormationSummary`：选中军团职责、战备、友军/近敌、姿态建议和将领技能状态。
- `TacticalRecommendationSummary`、`ManeuverOptionSummary`、`CommanderSynergySummary`：分别说明军议、机动落点和将令协同。
- `BattleView` 已在选中单位情报面板里连续展示编制、成长、军议、机动、将令、姿态和将领信息。

但玩家选中一个军团时，仍需要跨多张卡片拼接“它现在被谁压迫、脚下空间是否危险、当前最大收益来自哪里、下一步应该优先看哪条建议”。v0.40 应新增一张小型只读处境读板，把这些既有 summary 绑定到选中军团，不新增规则和不改变命令行为。

## 目标

1. 在 `GameViewModel.swift` 新增选中军团处境只读派生：
   - 建议新增 `SelectedUnitSituationSignalKind`、`SelectedUnitSituationSignal` 和 `SelectedUnitSituationReadout`。
   - 新增 `selectedUnitSituationReadout` computed property。
   - 只读取 `selectedUnit`、选中军团对应的 `FrontlinePressureSummary`、覆盖选中位置的 `ThreatHeatZoneSummary`、选中位置的 `MapControlSummary`、`selectedLegionFormationSummary`、`selectedTacticalRecommendationSummary`、`primaryManeuverOptionSummary`、`selectedCommanderSynergySummary`。
   - 输出字段建议包含 `unitID`、`position`、`title`、`statusLabel`、`pressureLabel`、`spaceLabel`、`opportunityLabel`、`nextStepLabel`、`riskLabel`、`signals`、`accessibilityLabel`。
   - signal 只保存 kind、标题、详情、位置、来源 ID；不要保存 SwiftUI `Color`。
   - 必须提供 references 方法或等价同源校验入口，供 `RenderBattlePreview` 检查。
2. 在 `BattleView.swift` 展示处境读板：
   - 在 `CompactSelectionPanelView` 和 `SelectionPanelView` 的选中单位基础属性/姿态之后、编制卡之前展示。
   - 紧凑版只显示标题、状态 pill 和一行下一步；完整版显示“压力 / 机会 / 下一步”三行短项。
   - SwiftUI 只消费 ViewModel 派生字段，不重新计算压力、热区、控区、机动收益或命令建议。
   - 保持短横屏和竖屏稳定，使用 `lineLimit`、`minimumScaleFactor` 和稳定最小高度。
3. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingSelectedUnitSituationReadout`。
   - 在初始选中 `rome-legion-1` 且位置仍为 `(3,3)` 时检查 `selectedUnitSituationReadout` 非空。
   - 断言它引用同一 `frontlinePressure`、覆盖 `(3,3)` 的 `threatHeat`、选中位置的 `mapControl`、编制、军议、机动和将令等有效来源。
   - 断言 `unitID == "rome-legion-1"`、`position == (3,3)`，标题、状态、压力、空间、机会、下一步、风险、无障碍文案和 signals 非空。
   - 断言读取处境读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.40`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.39 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、AI 权重、AI 行为、敌军意图、压力评分、热区评分、控区评分、军议评分、机动评分或将令评分。
- 不新增自动移动、自动攻击、自动技能、一键执行、命令队列、多回合搜索或真实作战计划执行器。
- 不改变按钮 action、`.disabled(...)`、移动、攻击、姿态、技能、结束回合、城市或外交逻辑。
- 不新增地图叠层规则；本轮只解释既有压力、热区、控区、军议、机动、将令和编制信号。
- 不新增持久存档字段，不新增第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameViewModel.frontlinePressureSummaries` 已能按 `report.targetKind == .unit && report.targetID == selectedUnitID` 找到选中军团承压来源。
- `GameViewModel.threatHeatZoneOverlaysByPosition` 已能按选中军团坐标取得覆盖该格的热区；不要强行要求 `primaryThreatHeatZoneSummary` 就是该格，避免重踩 v0.39 的排序误判。
- `GameViewModel.selectedMapControlSummary` 在当前选择焦点仍为军团位置时可复用；也可直接用 `state.mapControlReport(at: selectedUnit.position, for: .rome)` 派生。
- `selectedLegionFormationSummary`、`selectedTacticalRecommendationSummary`、`primaryManeuverOptionSummary`、`selectedCommanderSynergySummary` 已分别提供编制、军议、机动和将令短文案。
- `BattleView` 已有 `TacticalRecommendationCardView`、`ManeuverOptionCardView`、`CommanderSynergyCardView`、`LegionFormationCardView` 的 7px 卡片视觉语言，本轮复用小卡模式。

5. 完成后只做只读 `git diff` / `git status` 检查，提交并 push 到 `origin/main`，等待 GitHub Actions 与 Agent C artifact 复判。

## 验收标准

- `GameViewModel.selectedUnitSituationReadout` 能在确定性战斗预览场景中为 `rome-legion-1` 生成。
- 处境读板与选中军团的压力、热区、控区、编制、军议、机动和将令同源。
- signals 至少覆盖压力、热区、控区、编制、军议、机动或将令中的有效子集，并包含非空标题、详情、位置或来源 ID。
- 完整/紧凑选中单位情报面板都能读到处境摘要，不在短横屏或竖屏明显溢出。
- 读取处境读板不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算核心路径、热区、控区、收益、可用性或优先级。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：选中军团处境读板可能与编制、军议、机动、将令卡重复；实现时要压缩成“压力 / 机会 / 下一步”三类短文案。
- 风险：`primaryThreatHeatZoneSummary` 不一定覆盖选中军团坐标；实现与断言必须按选中位置匹配热区。
- 风险：选中单位情报面板已较长；紧凑版只能显示一行下一步，不再展开 signal 列表。
- 禁止把处境读板变成真实命令队列或自动执行系统。
- 禁止在 SwiftUI 中重新计算优先级、热区、控区、路径、反制收益或目标线阶段。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
