# v0.21（机动落点与地图风险读板）

## 版本号

v0.21

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.20（本方将领协同与战术连携读板）。Agent X 总目标要求继续向“大征服者罗马”推进，重点优化战斗界面 UI、地图、AI 和将领。本轮在 v0.18 地图控制/威胁热区、v0.16 战术建议路径和 v0.20 本方将令协同基础上，继续推进地图可读性：把“选中单位能走到哪里”升级为“推荐走到哪里、为什么、落点危险与收益是什么”，因此版本递增为 v0.21。

## 背景

当前战斗页已经能显示可移动格、敌军意图路线、本方战术建议路径、地图控制、威胁热区、战场焦点、AI 作战计划和本方将领协同。玩家仍需要在多个读板之间手动合并判断：某个可移动格是否能形成攻击、是否能夺城、是否能补线、是否会踩进高危热区、是否比当前战术建议更安全。

本轮新增“机动落点”只读报告：核心层基于选中单位的真实可达格，模拟候选落点后的地图控制、热区、可攻击目标、可夺取城市、补线距离和路径，输出少量高价值候选；ViewModel 转成中文摘要和 overlay 集合；BattleView 在地图可达格上显示落点类型/风险，并在选中单位情报区和完整战局面板展示首要机动建议。该能力只解释落点，不自动移动单位，不改变 AI、移动、战斗或结算。

## 目标

1. 在 `RomeLegionsCore` 新增玩家侧只读机动落点模型。
2. 机动类型至少覆盖：打击落点、夺城落点、补线落点、推进落点和避险/稳固落点。
3. 机动报告必须包含单位、阵营、起点、落点、路径、目标单位/城市、目标位置、推荐姿态、地图控制状态、热区等级、友敌影响、预计伤害、反击、支援/包夹/指挥修正、补线距离或推进距离、是否可执行、阻塞原因、评分、标题、摘要和详情。
4. 公开只读入口：
   - `maneuverOptionReports(unitID:limit:) throws -> [ManeuverOptionReport]`
   - `maneuverOptionReport(unitID:) throws -> ManeuverOptionReport?`
5. 报告必须复用现有可达格、路径、攻击预览、地图控制、威胁热区、战线压力、城市目标和外交规则；不得写另一套移动、攻击、夺城或热区规则。
6. 报告入口必须只读，不新增 Codable 存档字段。
7. 不改变 `moveUnit`、`attackPreview`、`attack`、`captureCityIfPossible`、`performSimpleAI(for:)`、AI 权重、战术姿态数值、将领技能数值、真实移动、招募、城市或胜负结算。
8. `GameViewModel` 新增 `ManeuverOptionSummary`、选中单位机动摘要列表、首要机动摘要、机动落点 overlay 字典/位置集合，把核心报告转成中文文案和无障碍文案。
9. `BattleView` 在地图可达格增加机动落点叠层，能区分打击/夺城/补线/推进/避险和低/中/高风险；在顶部状态条增加“机动” chip；在选中单位情报区展示机动卡；完整战局面板展示前两条机动行。
10. Swift Testing、Gameplay Smoke、RenderBattlePreview 覆盖机动落点的打击、夺城、补线、风险排序、条约过滤、只读不变性和 ViewModel 主链路字段。

## 非目标

- 不实现自动移动、自动攻击、一键执行机动、自动规避热区或多回合路径规划。
- 不修改现有移动范围、移动成本、地形、防御、攻击、支援/包夹/指挥修正、将领技能、战术姿态或 AI 行为。
- 不新增真实补给线、建筑树、装备、人口、外交界面或存档 UI。
- 不把机动评分写入 SwiftUI；SwiftUI 只能展示核心报告。
- 不拆分 `BattleView.swift` 作专项重构，只允许新增本轮必要的小型视图、叠层和摘要。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- `GameState.reachablePositions(for:)` 和私有 `reachablePositions(for:)` 已实现真实可达格。
- `tacticalPath(from:to:for:)` 和 `tacticalFallbackPath(from:to:)` 已用于战术建议路径，可复用为候选落点路径。
- `attackPreview(attackerID:defenderID:)` 已公开预计伤害、反击、支援、包夹和指挥修正；机动落点若模拟落点后打击，必须继续从该预览读取数值。
- `capturableCity(at:by:)`、`nearestAICityObjective(from:for:)`、`frontlinePressureReports`、`mapControlReport(at:for:)`、`threatHeatZoneReports` 已提供夺城、推进、补线和风险输入。
- `GameViewModel` 已有 `TacticalRecommendationSummary`、`CommanderSynergySummary`、`MapControlSummary`、`ThreatHeatZoneSummary`、`BattlefieldFocusSummary` 等摘要模式。
- `BattleView` 已有 `WarMapView`、`HexTileView`、`ReachableTileOverlay`、`TacticalRecommendationPathOverlay`、`ThreatHeatTileOverlay`、`MapControlTileOverlay`、顶部 `TacticalStatusStripView`、选中单位情报区和 `StrategicBalancePanelView`，可增量接入机动落点 overlay、chip、卡片和行。

## 实现步骤

1. 在 `GameState.swift` 新增：
   - `ManeuverOptionKind`
   - `ManeuverOptionReport`
2. 新增只读入口：
   - `maneuverOptionReports(unitID:limit:) throws -> [ManeuverOptionReport]`
   - `maneuverOptionReport(unitID:) throws -> ManeuverOptionReport?`
3. 机动派生规则：
   - 只对存在、属于当前行动势力且未移动的单位返回候选；已移动或非当前势力返回空列表或不可执行报告时必须明确阻塞原因。
   - 候选落点来自真实 `reachablePositions(for:)`，路径来自 `tacticalPath`，不得把不可达格放入报告。
   - 对每个候选落点创建只读 projected state，把该单位临时放到落点，再读取 projected state 的地图控制/热区、可攻击目标和攻击预览。
   - 若落点可直接攻击，选择最高收益目标，输出 `.strike`，预计伤害和修正来自 projected `attackPreview`。
   - 若落点可占领中立或敌方城市，输出 `.capture`，目标城市来自既有城市/外交判断。
   - 若落点靠近当前战线压力目标并缩短距离，输出 `.reinforce`，补线距离来自战线压力目标。
   - 若落点靠近最近可争夺城市，输出 `.advance`，推进距离来自城市目标。
   - 若没有明显打击/夺城/补线/推进，但落点显著降低热区或提高友方控制，输出 `.secure`。
   - 排序稳定：可执行优先，score 降序，同分按 id。
4. `GameViewModel.swift` 新增：
   - `ManeuverOptionSummary`
   - `selectedManeuverOptionSummaries`
   - `primaryManeuverOptionSummary`
   - `maneuverOptionOverlaysByPosition`
   - `maneuverOptionOverlayPositions`
   - summary 映射 helper。
5. `BattleView.swift`：
   - `WarMapView` 把机动 overlay 字典传入 `HexTileView`。
   - `HexTileView` 展示机动落点叠层，并把机动信息加入 accessibility label。
   - 顶部状态条增加“机动” chip。
   - 选中单位完整/紧凑情报区增加 `ManeuverOptionCardView`。
   - 完整战局面板展示前两条 `ManeuverOptionRowView`。
   - 新增 `ManeuverOptionKind` / 风险的图标和颜色扩展；图标优先用 SF Symbols。
6. `Tests/RomeLegionsCoreTests/GameStateTests.swift` 新增核心测试：
   - 可达落点报告只读，路径起点/终点正确。
   - 打击落点的预计伤害、反击、支援/包夹/指挥修正来自 projected `attackPreview`。
   - 可占领城市落点标记为夺城，不越过条约保护。
   - 补线落点能缩短到高压友方目标的距离。
   - 高危热区落点风险高于安全落点，排序不让高风险无收益落点压过可执行打击/夺城。
   - 已移动或非当前势力单位没有可执行机动落点且不改变状态。
7. `Tools/GameplaySmoke/main.swift` 增加机动落点主链路断言。

## 验收标准

- 核心机动落点报告能输出打击、夺城、补线、推进和避险/稳固类读板。
- 机动落点只来自真实可达格，路径起点/终点正确。
- 打击落点的预计伤害、反击、支援、包夹和指挥修正来自 projected `attackPreview`。
- 夺城落点尊重外交条约，不把被条约保护城市作为可夺取目标。
- 报告调用不改变 `GameState`，不新增存档字段。
- `GameViewModel` 能提供完整中文摘要、overlay 位置集合和无障碍文案。
- 地图可达格能看到机动类型和风险，顶部态势、选中单位情报区和完整战局面板可见机动信息；紧凑界面不挤掉军令首屏。
- 既有敌军计划、敌情路线、战线压力、威胁热区、战场焦点、军团编制、战术建议、本方将令、将领详情、战术姿态、城市经营/招募读板无回归。
- 推送 `main` 后 GitHub Actions 成功，Agent C 下载最新 artifact 复判通过。
