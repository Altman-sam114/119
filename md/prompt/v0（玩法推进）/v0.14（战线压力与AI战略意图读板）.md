# v0.14（战线压力与 AI 战略意图读板）

## 版本号

v0.14

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.13（城市经营与招募预览读板），本轮继续围绕“接近《大征服者罗马》的战棋体验”小步推进，版本递增为 v0.14。

## 背景

当前项目已经具备 AI 回合、单兵敌军意图、敌军路线/目标地图叠层、将领详情、战术姿态和城市经营读板。现有 `EnemyIntentSummary` 适合展示每个敌军单位下一步动作，但玩家还缺少更高层的“哪条战线最危险、哪座城或哪支部队被多路威胁、应该优先防哪里”的战略读板。

本轮目标是新增只读的战线压力聚合报告：核心层基于 `AIIntent` 汇总敌军对罗马单位和城市的压力，ViewModel 转成 UI 摘要，BattleView 在现有敌情/战局区域展示。该能力让敌情从“单个单位要做什么”进化到“整个战线正在压哪里”。

## 目标

1. 在 `RomeLegionsCore` 新增只读战线压力报告模型，聚合敌方 `AIIntent` 对罗马单位和城市的威胁。
2. 报告至少包含：目标单位或城市、目标坐标、来源阵营、来源单位、意图数量、攻击/夺城数量、预计伤害合计、最高威胁值、总压力值和压力等级。
3. 报告必须由核心层生成，不能在 SwiftUI 中重新评分或复制 AI 规则。
4. `GameViewModel` 新增 UI 友好的战线压力摘要，整理目标名称、来源、压力等级、预计伤害/夺城风险和无障碍文案。
5. `BattleView` 在完整战局面板展示前 2-3 条战线压力；紧凑界面只显示一条短摘要或小型指标，不能挤占军令首屏。
6. Swift Testing 和 Gameplay Smoke 覆盖报告聚合、城市夺取压力、只读不变性和 UI 主链路需要的字段。
7. `Tools/RenderBattlePreview` 渲染前断言战线压力摘要存在，并在三尺寸预览中保留已有敌军路线、将领、姿态和城市读板覆盖。

## 非目标

- 不实现真正多回合 AI、搜索树、长期战略计划或难度系统。
- 不改变现有 AI 行为顺序、评分含义、移动/攻击/招募结算或战术姿态数值。
- 不新增建筑树、人口、军团编制、地图大改或存档 UI。
- 不拆分 `BattleView.swift`，除非编译需要极小范围辅助视图。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。
- 不改变 `AIIntent` 既有字段语义，除非新增字段完全向后兼容。

## 当前架构依据

- 核心规则和 AI 预测位于 `Sources/RomeLegionsCore/GameState.swift`。
- `GameState.aiIntents(for:limit:)` 已经在 forecast copy 中刷新 AI 行动状态并返回只读意图，不能改变原始 `GameState`。
- `GameViewModel.enemyIntentSummaries` 将核心 `AIIntent` 转成单兵 UI 摘要；`enemyIntentMapOverlays` 再派生地图路径。
- `BattleView` 只展示 `GameViewModel` 派生数据，不应写 AI 聚合逻辑。
- `md/test/test.md` 当前 Swift Testing 基线为 45 个用例；RenderBattlePreview 已覆盖三尺寸城市场景和 `*-unit.png` 单位场景。

## 实现步骤

1. 在 `GameState.swift` 新增 `FrontlinePressureLevel` 和 `FrontlinePressureReport` 等只读核心模型。
2. 新增 `GameState.frontlinePressureReports(for:limit:)` 或同等清晰 API：
   - 默认用于 `defendingFaction == .rome`，但实现不要硬编码 UI。
   - 遍历与防守方处于战争关系的非中立阵营。
   - 对每个敌方阵营调用 `aiIntents(for:limit:)`，聚合目标为防守方单位或防守方城市的意图。
   - 按目标单位或目标城市分组；没有防守方目标的整备/固守意图不进入报告。
   - 计算来源单位、来源阵营、意图数量、攻击类数量、夺城数量、预计伤害合计、最高 `threatScore`、总压力值和等级。
   - 输出按压力值降序、同分按目标 id 稳定排序，并限制为 `limit` 条。
3. 压力等级建议：
   - `.critical`：预计伤害足以击败目标单位，或防守方城市存在明确 `.captureCity` 意图，或攻击类意图数量不少于 2。
   - `.threatened`：存在攻击/接敌攻击，或总压力值达到高阈值。
   - `.contested`：存在推进到防守方城市/单位目标，或中等压力。
   - `.watch`：低强度关注。
   阈值可按现有 `AIIntent.threatScore` 规模保守选择，并用测试锁定关键场景。
4. `GameViewModel.swift` 新增 `FrontlinePressureSummary`，把报告转成：
   - `targetLabel`
   - `sourceLabel`
   - `pressureLabel`
   - `impactLabel`
   - `detail`
   - `accessibilityLabel`
   - `level`
   并新增 `frontlinePressureSummaries`、`primaryFrontlinePressureSummary`。
5. `BattleView.swift`：
   - 完整 `StrategicBalancePanelView` 在兵力/城池/态势与阵营行之间展示战线压力列表。
   - 没有压力时显示简短空状态。
   - 新增小型 `FrontlinePressureRowView` / `CompactFrontlinePressureView`，使用现有 Panel/Card 风格，不做大视觉体系重写。
   - 紧凑 `BattlefieldFocusPanelView(isCompact: true)` 只显示首要压力的一行短摘要，避免提高军令区域高度。
   - 使用 `Label` 或带文本的按钮/图标语义，避免纯图标无标签。
6. `Tests/RomeLegionsCoreTests/GameStateTests.swift`：
   - 新增多敌军攻击同一罗马单位时聚合为同一报告，报告含来源单位、预计伤害和 critical/threatened 等级。
   - 新增敌军夺取罗马城市时生成城市压力报告。
   - 每个核心报告测试都保留 `let before = state` 并断言调用报告后 `state == before`。
7. `Tools/GameplaySmoke/main.swift`：
   - 增加战线压力主链路断言：至少一条报告，目标 id、来源单位、预计伤害或夺城数量、压力等级符合预期。
8. `Tools/RenderBattlePreview/main.swift`：
   - 在现有单位场景断言中加入 `frontlinePressureSummaries` 非空、首要目标为罗马单位或城市、detail/impact 文案非空。
   - 保留 v0.10-v0.13 的敌军路径、将领、姿态、城市读板断言。
9. 更新文档：
   - `README.md` 已实现能力和预览命令说明。
   - `md/flow/flow.md` 当前数据流、AI/敌情流、核心对象、扩展点。
   - `md/flow/flowchart.md` 战斗与敌军意图图中加入战线压力聚合。
   - `md/test/test.md` 当前基线、Gameplay Smoke 与 RenderBattlePreview 断言说明。
   - `update_log.md` 追加 v0.14 正式记录。
   - `.github/workflows/ci-results.yml` 的 `CI_VERSION` 更新为 `v0.14`。
