# v0.41（处境命令入口联动）

## 版本号

v0.41

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.40（选中军团处境读板），并已通过 `origin/main` 最新云端结果包复判。v0.40 已把压力、热区、控区、编制、军议、机动和将令压缩为选中军团处境读板；下一步应让这张读板明确指向现有可操作命令入口，让玩家从“这个军团处境如何”继续读到“应该去看哪个命令面板”。递增为 v0.41。

## 背景

当前项目已有：

- `SelectedUnitSituationReadout`：选中军团处境摘要，已同源引用压力、热区、控区、编制、军议、机动和将令。
- `selectedCountermeasureCommandPreview`：选中军团作为反制回应单位时的反制入口预览。
- `selectedBattleObjectiveStageCommandPreview`：选中军团参与目标线阶段时的阶段命令入口预览。
- `selectedCommanderActionGuidance`：将领技能入口和目标线阶段关联提示。
- `primaryManeuverOptionSummary`、`selectedTacticalRecommendationSummary`、`selectedTacticalOrderPreviews`：机动、军议和姿态入口来源。
- `BattleView` 已在选中单位动作面板中展示反制预览、目标线阶段预览、姿态建议和将领技能按钮状态。

但 v0.40 的处境读板仍停留在“下一步”文字，无法显式证明它和现有命令入口同源。v0.41 应只在 ViewModel 派生层追加命令入口引用，并在 UI 中用一行短提示展示，不改变任何规则、按钮行为或 AI 权重。

## 目标

1. 在 `GameViewModel.swift` 扩展选中军团处境读板：
   - 建议新增 `SelectedUnitSituationCommandEntryKind` 和 `SelectedUnitSituationCommandEntry`。
   - `SelectedUnitSituationReadout` 新增 `commandEntries`、主入口短标题、入口摘要、反制/目标线/姿态等引用字段或等价同源校验方法。
   - 入口来源优先级建议为：反制命令预览 > 目标线阶段命令预览 > 将领技能入口 > 机动 > 军议 > 姿态建议。
   - 每个 entry 至少包含 kind、title、detail、cueLabel、position、sourceID、isPrimary，字段保持纯 Swift 数据，不写 SwiftUI 类型。
   - `selectedUnitSituationReadout` 只读取现有 computed properties，不新增 `@Published` 状态。
2. 在 `BattleView.swift` 更新处境读板：
   - 紧凑版在原一行下一步之外体现主命令入口，保持最多两行。
   - 完整版增加一行“入口”或替换原“下一步”为更明确的命令入口行。
   - SwiftUI 只展示 ViewModel 结果，不重新计算反制、目标线、机动、军议、技能或姿态优先级。
3. 在 `Tools/RenderBattlePreview/main.swift` 更新云端断言：
   - `missingSelectedUnitSituationReadout` 区块继续作为 v0.41 验收入口。
   - 断言 `commandEntries` 非空、主入口标签和无障碍文案非空。
   - 若存在选中反制预览或目标线阶段预览，断言处境读板引用对应 preview。
   - 断言入口读取仍不改变 `GameState` 单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.41`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.40 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、AI 权重、敌军意图、反制评分、目标线评分、军议评分、机动评分、姿态评分或将领技能规则。
- 不新增命令队列、自动执行、一键连招、多回合搜索或真实作战计划执行器。
- 不改变移动、攻击、姿态、技能、结束回合、城市、外交或存档逻辑。
- 不新增地图叠层规则；本轮只把处境读板连接到既有命令入口文案。
- 不新增第三方框架，不做大型 SwiftUI 拆分或无关重构。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 验收标准

- `selectedUnitSituationReadout` 能在确定性战斗预览场景中为 `rome-legion-1` 生成至少一个命令入口。
- 主入口标题、摘要、cue 文案、无障碍文案非空。
- 若选中反制或目标线阶段预览存在，处境读板可用 references 方法证明同源引用。
- 完整/紧凑选中单位情报面板都能读到处境命令入口，不在短横屏或竖屏明显溢出。
- 读取处境命令入口不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算核心路径、热区、控区、收益、可用性或优先级。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：处境入口可能和动作面板里的反制/目标线预览重复；实现时要压缩成短 cue，而不是复制整张卡。
- 风险：固定 fixture 中选中反制或目标线 preview 未来可能因样本扩展而变化；断言必须先确认 preview 归属于选中军团，再要求同源引用。
- 风险：选中单位情报面板已较长；紧凑版不要展开 command entry 列表。
- 禁止把命令入口变成真实命令队列或自动执行系统。
- 禁止在 SwiftUI 中重新计算优先级、反制收益、目标线阶段、机动落点或姿态可用性。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
