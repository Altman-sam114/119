# v0.19（AI作战计划与将领协同读板）

## 版本号

v0.19

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.18（地图控制与威胁热区读板）。本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”推进“AI + 将领”维度：把零散敌军意图、战线压力、热区和敌方将领技能机会聚合成可读的敌军作战计划，因此版本递增为 v0.19。

## 背景

项目已有敌军意图、敌军路线、战线压力、战场焦点、地图控制和威胁热区，但玩家仍需要自己把“敌军下一步主攻哪里”“哪些单位协同”“敌方将领是否准备发动技能”“这是夺城、集火还是整备”串起来。

本轮新增只读 AI 作战计划报告：核心层按敌方阵营和目标位置聚合 AIIntent、战线压力、热区和敌方将领技能预览；ViewModel 转成 UI 摘要；BattleView 在顶部 chip、敌情面板和战局面板中展示。该能力只解释敌方计划，不改变真实 AI 行为、AI 评分、移动、攻击、技能、招募或胜负结算。

## 目标

1. 在 `RomeLegionsCore` 新增只读 AI 作战计划模型。
2. 计划类型至少覆盖：集火攻击、夺城、将领技能、推进、固守、整备。
3. 计划步骤能表达主攻、支援、将领、预备等协同角色。
4. 计划报告必须包含敌方阵营、类型、目标位置、目标单位/城市、来源单位、将领单位、意图种类、压力等级、热区等级、预计伤害、评分、标题、摘要、详情和步骤。
5. 计划入口必须只读，不新增 Codable 存档字段。
6. 不改变 `performSimpleAI(for:)`、`aiIntent(for:)`、AI 权重、真实移动、攻击、技能、城市、招募或胜负结算。
7. 敌方将领技能协同必须在敌方 forecast copy 中计算，避免罗马回合下误判敌方技能不可用。
8. `GameViewModel` 新增 `AIOperationalPlanSummary`、计划摘要列表和首要计划，把核心报告转成中文文案和无障碍文案。
9. `BattleView` 在顶部状态条增加“计划”chip，在敌情面板展示首要计划卡，在完整战局面板展示前两条计划行；紧凑界面仍保持军令优先。
10. Swift Testing、Gameplay Smoke、RenderBattlePreview 覆盖集火计划、夺城计划、敌方将领技能计划、整备计划、条约过滤、只读不变性和 ViewModel 主链路字段。

## 非目标

- 不实现多回合 AI 搜索、真实路线规划、自动执行计划或玩家自动应对。
- 不修改现有 AI 评分、AI 行动顺序、真实 `performSimpleAI(for:)`、`moveUnit`、`attack`、`useGeneralSkill` 或城市占领逻辑。
- 不改变战斗数值、将领技能数值、战术姿态数值、热区评分、控图评分或战役目标。
- 不新增建筑树、人口、装备、外交界面、补给线或存档 UI。
- 不拆分 `BattleView.swift` 作专项重构，只允许新增本轮必要的小型视图和摘要。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、AI 意图、战线压力、战场焦点、地图控制、威胁热区、将领技能和军团编制位于 `Sources/RomeLegionsCore/GameState.swift`。
- `aiIntents(for:limit:)` 已通过 forecast copy 只读预测敌军行动。
- `frontlinePressureReports(against:perFactionLimit:limit:)` 已按罗马单位/城市聚合多路敌方意图。
- `threatHeatZoneReports(for:limit:)` 已按地图位置输出高风险热区。
- `legionFormationReport(for:)` 和 `generalSkillPreview(for:)` 可给计划步骤补充敌方将领状态，但必须在对应敌方 forecast copy 上调用。
- `GameViewModel` 已有 `EnemyIntentSummary`、`FrontlinePressureSummary`、`ThreatHeatZoneSummary`、`BattlefieldFocusSummary` 等 UI 摘要模式。
- `BattleView` 已有 `TacticalStatusStripView`、`EnemyIntentPanelView`、`StrategicBalancePanelView` 和多个行/卡片视图，可增量接入计划 chip、卡片和行。

## 实现步骤

1. 在 `GameState.swift` 新增：
   - `AIOperationalPlanKind`
   - `AIPlanCoordinationRole`
   - `AIPlanStepReport`
   - `AIOperationalPlanReport`
2. 新增只读入口：
   - `aiOperationalPlanReports(against:perFactionLimit:limit:) -> [AIOperationalPlanReport]`
   - `aiOperationalPlanReport(against:) -> AIOperationalPlanReport?`
3. 抽取或复用敌方 forecast helper：
   - `private func aiPlanningForecast(for faction: Faction) -> GameState`
   - `aiIntents(for:)` 和新计划报告都复用同一刷新逻辑。
4. 计划聚合规则：
   - 对每个与防守方处于战争状态的敌方阵营取 `aiIntents(for:limit:)`。
   - 攻击/移动后攻击按目标单位聚合为集火计划。
   - 夺城按目标城市聚合为夺城计划。
   - 技能按技能目标或施法者位置聚合为将领技能计划。
   - 推进、固守、整备无明确压力目标时仍产生 fallback 计划，不能被过滤。
   - 每个计划关联对应 `FrontlinePressureReport` 和 `ThreatHeatZoneReport`，但不得让热区报告反向依赖计划，避免循环。
   - 步骤排序稳定：主攻、将领、支援、预备；同级按威胁分和 unit id。
   - 计划排序稳定：score desc，同分按阵营、目标位置、id。
5. `GameViewModel.swift` 新增：
   - `AIOperationalPlanSummary`
   - `aiOperationalPlanSummaries`
   - `primaryAIOperationalPlanSummary`
   - plan summary 映射 helper。
6. `BattleView.swift`：
   - `TacticalStatusStripView` 在“敌情”后或“战线”前增加“计划”chip。
   - `EnemyIntentPanelView` 在意图列表前展示首要 AI 计划卡。
   - `StrategicBalancePanelView` 展示前两条 AI 计划行。
   - 新增小型 `AIOperationalPlanCardView` / `AIOperationalPlanRowView` 和颜色/图标扩展。
7. `Tests/RomeLegionsCoreTests/GameStateTests.swift` 新增核心测试：
   - 多支敌军集火同一罗马军团时聚合作战计划且只读。
   - 敌军夺取罗马城市时产生夺城计划。
   - 敌方将领技能计划在罗马回合仍能通过敌方 forecast 识别。
   - 低生命敌军整备不因没有压力目标被过滤。
   - 条约势力不产生敌对计划。
8. `Tools/GameplaySmoke/main.swift` 增加 AI 作战计划主链路断言。
9. `Tools/RenderBattlePreview/main.swift` 增加 ViewModel 字段断言：
   - `primaryAIOperationalPlanSummary`
   - `aiOperationalPlanSummaries`
   - 计划标题、类型、来源、目标、影响、详情、步骤和无障碍文案可读。
   - 错误枚举增加 `missingAIOperationalPlanSummary`。
