# v0.39（战场态势交汇链路）

## 版本号

v0.39

## 版本分配依据

`update_log.md` 当前最新功能版本为 v0.38（将领技能目标与收益读板），并已通过 `origin/main` 最新云端结果包复判。v0.38 已把选中单位主动技能目标、收益和地图叠层读板补齐；下一步应把当前已经分散在战场目标线、反制建议、阶段命令预览、将令协同、机动落点、威胁热区和地图控区里的关键态势压缩成一张只读总览，递增为 v0.39。

## 背景

当前项目已有：

- `BattlefieldFocusSummary`：首要战场焦点。
- `BattleObjectiveChainSummary`：只读组合焦点、将令、机动和军议，形成“1 焦点、2 将令、3 机动、4 军议”的目标线。
- `BattleObjectiveStageCommandPreview`：目标线阶段对应的既有命令入口、状态、下一步和阻塞说明。
- `CountermeasureSummary` 与 `CountermeasureCommandPreview`：敌情反制建议、回应单位、落点、目标、收益、风险和执行预览。
- `CommanderSynergySummary`、`ManeuverOptionSummary`、`ThreatHeatZoneSummary`、`MapControlSummary`：分别说明将令、机动、热区和控区。
- `BattleView` 已分别展示这些卡片和行，但玩家仍需要在多个面板之间拼接“当前主线是什么、威胁在哪里、应该先看哪个命令入口”。

v0.39 应新增一张只读的战场态势交汇读板，把这些既有 summary 的首要结论串成同源链路，帮助玩家更快读图。它不是新 AI，不是命令队列，也不是自动执行。

## 目标

1. 在 `GameViewModel.swift` 增加战场态势交汇只读派生：
   - 建议新增 `BattlefieldConvergenceRole`、`BattlefieldConvergenceSignal` 和 `BattlefieldConvergenceSummary`。
   - 新增 `primaryBattlefieldConvergenceSummary` computed property。
   - 只读取 `primaryBattleObjectiveChainSummary`、`primaryCountermeasureSummary`、`primaryCountermeasureCommandPreview`、`activeBattleObjectiveStageCommandPreview`、`selectedCommanderSynergySummary ?? primaryCommanderSynergySummary`、`primaryManeuverOptionSummary`、`primaryThreatHeatZoneSummary`、`selectedMapControlSummary ?? primaryMapControlSummary`。
   - 输出字段建议包含 `title`、`compactLabel`、`priorityLabel`、`objectiveLabel`、`responseLabel`、`spaceLabel`、`nextStepLabel`、`riskLabel`、`signals`、`accessibilityLabel`。
   - signal 只保存角色、标题、详情、位置、来源 ID、状态或下一步文案；不要保存 SwiftUI `Color`。
   - 必须提供 references 方法或等价同源校验入口，供 RenderBattlePreview 检查。
2. 在 `BattleView.swift` 展示交汇读板：
   - 在 `BattlefieldFocusPanelView` 中展示 `BattlefieldConvergenceCardView`，完整面板显示目标线、反制、空间和下一步；紧凑面板显示一行短摘要。
   - 在 `StrategicBalancePanelView` 顶部资源/评分附近展示一条紧凑交汇行，避免继续堆长列表。
   - SwiftUI 只消费 ViewModel 派生字段，不重新判断优先级、不重新计算热区、路径、收益或命令状态。
   - 保持短横屏和竖屏文本稳定，使用 `lineLimit`、`minimumScaleFactor` 和固定最小高度。
3. 更新 `Tools/RenderBattlePreview/main.swift` 云端断言：
   - 新增错误 case，建议 `missingBattlefieldConvergenceSummary`。
   - 在目标线 summary 断言之后、目标线聚焦操作之前，检查 `primaryBattlefieldConvergenceSummary` 非空。
   - 断言它引用同一 `objectiveChain`、`countermeasure`、`countermeasureCommandPreview`、`stagePreview`、`synergy`、`maneuver`、`threatHeat`、`mapControl`。
   - 断言 signals 覆盖目标、反制、阶段、将令、机动、热区或控区中的有效子集，且标题、详情、位置、来源 ID、无障碍文案非空。
   - 断言读取交汇读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.39`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.38 云端通过证据。

- 不新增自动移动、自动攻击、自动技能、一键反制、命令队列、多回合搜索或真实作战计划执行器。
- 不改变按钮 action、`.disabled(...)`、移动、攻击、姿态、技能、结束回合、城市或外交逻辑。
- 不新增地图叠层规则；本轮只解释已有目标线、反制、热区、控区、机动和技能叠层。
- 不新增持久存档字段，不新增第三方框架。
- 不做大型 SwiftUI 文件拆分、全局重构或无关文档整理。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameViewModel.primaryBattleObjectiveChainSummary` 已提供目标线、阶段文案、紧凑文案、优先级和同源 references。
- `GameViewModel.primaryCountermeasureSummary` 与 `primaryCountermeasureCommandPreview` 已提供反制回应单位、落点、目标、收益、风险、命令入口和焦点链路。
- `GameViewModel.activeBattleObjectiveStageCommandPreview` 已提供当前目标线阶段的命令入口、状态和下一步。
- `selectedCommanderSynergySummary ?? primaryCommanderSynergySummary`、`primaryManeuverOptionSummary`、`primaryThreatHeatZoneSummary`、`selectedMapControlSummary ?? primaryMapControlSummary` 已覆盖将令、机动、热区和控区。
- `BattleView` 已有 `BattleObjectiveChainCardView`、`CountermeasureRowView`、`ThreatHeatCardView`、`MapControlCardView` 等组件，可复用视觉语言；本轮只新增小型交汇卡/行。

## 验收标准

- `GameViewModel.primaryBattlefieldConvergenceSummary` 能在确定性战斗预览场景中生成。
- 交汇读板与首要目标线、反制建议、反制指令预览、活动目标线阶段、将令、机动、热区和控区同源。
- signals 至少覆盖目标线、反制、阶段、将令、机动、热区或控区中的有效子集，并包含非空标题、详情、位置或来源 ID。
- 完整/紧凑战场面板和战局面板都能读到交汇摘要，不在短横屏或竖屏明显溢出。
- 读取交汇读板不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算核心路径、热区、控区、收益、可用性或优先级。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：交汇读板可能与目标线、反制、热区卡重复；实现时要压缩成“主线 / 回应 / 空间 / 下一步”四类短文案。
- 风险：`StrategicBalancePanelView` 已较长；战局面板只增加一条紧凑行，不再展开 signal 列表。
- 风险：Render 断言必须在焦点操作前执行，避免后续选择态变化影响稳定性。
- 禁止把聚合读板变成真实命令队列或自动执行系统。
- 禁止在 SwiftUI 中重新计算优先级、热区、控区、路径、反制收益或目标线阶段。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
