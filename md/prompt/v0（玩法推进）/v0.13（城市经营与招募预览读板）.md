# v0.13 / 城市经营与招募预览读板

## 版本号和版本分配依据

- 版本号：v0.13。
- 版本标题：城市经营与招募预览读板。
- 分配依据：`update_log.md` 当前最新正式版本为 v0.12（敌军意图路径贴合六边形地图）。本轮继续响应 Agent X 的“大征服者罗马”长期目标，补足城市、资源、招募和部署可读性，因此在 v0.12 后递增为 v0.13。

## 背景

当前战斗页的单位、将领、姿态和敌军意图已经逐步可读，但城市面板仍只显示基础产出和城防，招募按钮也只显示单位短名和金币。玩家无法在点城市时直接判断下一回合收入、扩建收益、招募总成本、资源缺口、舰队是否有港口、城市格或相邻部署格是否可用。这使战略层仍不像《大征服者罗马》式的城市经营读板。

## 目标

1. 在 `GameState` 新增城市经营与招募只读预览，并让真实扩建/招募复用或对齐预览：
   - 扩建预览公开成本、产出增量、城防增量、是否可执行和阻塞原因。
   - 招募预览公开成本、预计部署位置、是否可执行和阻塞原因。
   - 预览不得修改原始 `GameState`。
2. 在 `GameViewModel` 新增选中/命令城市的 UI 派生模型：
   - 城市名、归属、坐标、城防。
   - 本城产出和所属势力总回合收入。
   - 罗马当前资源池。
   - 扩建成本、扩建后产出/城防预览、是否可执行和阻塞原因。
   - 补给/部署摘要：城市格是否被占用、可用相邻部署格数量、舰队港口可用性。
3. 在 `GameViewModel` 新增招募选项 UI 预览：
   - 覆盖 `UnitKind.allCases`。
   - 显示单位类型、攻击、防御、机动、射程、生命、完整资源成本。
   - 显示预计部署位置或不可部署原因。
   - 显示资源是否足够、缺口摘要、是否可招募。
4. `BattleView` 在完整情报栏、紧凑情报栏和军令面板中展示上述读板：
   - 点城市时能看到收入、城防、扩建收益和招募可行性。
   - 招募按钮使用 ViewModel 的预览数据，不再在视图中直接过滤 `UnitKind` 或只显示金币。
   - 不可执行按钮应禁用并展示简短原因。
5. `Tools/RenderBattlePreview` 增加确定性城市场景断言：
   - 选中罗马城市后，`selectedCityBrief` 存在。
   - 扩建预览存在成本和收益。
   - 招募预览覆盖军团、骑兵、弓兵、舰队，并至少包含一个可招募陆军选项和舰队港口判断。
   - 保留 v0.12 敌军路径、v0.11 将领详情/姿态预览和紧凑命令区像素检查。
6. 同步 Swift Testing、Gameplay Smoke、README、flow、flowchart、test、update_log 和 CI artifact 版本到 v0.13。

## 非目标

- 不修改 `GameState` 的城市产出数值、扩建成本、扩建收益、招募成本、招募位置规则、AI 招募策略、真实招募行为或 Codable 存档字段。
- 不引入新科技树、建筑树、城市等级、人口、军团编制或外交 UI。
- 不把核心规则写入 SwiftUI `body`、按钮闭包或生命周期回调。
- 不创建新分支、PR、`develop`、`smalldata_test` 或候选分支流程。

## 当前架构依据

- 核心城市规则在 `Sources/RomeLegionsCore/GameState.swift`：
  - `City` 存储城市位置、归属、产出和城防。
  - `income(for:)` 统计所属势力城市产出。
  - `developCity(id:)` 固定消耗 `EmpireResources(gold: 70, grain: 40, iron: 35, science: 0, prestige: 0)`，产出提升 `gold +10 / grain +8 / iron +6 / science +4 / prestige +1`，城防 `+3`。
  - `recruit(_:at:)` 调用私有 `spawnPosition(for:from:)` 后消耗 `UnitKind.recruitmentCost` 并生成单位。
  - `UnitKind.canEnter(_:)` 和 `Position.neighbors(width:height:)` 可支持 ViewModel 做只读部署预览。
- `GameState` 应拥有会影响真实可执行性的规则预览，避免 `GameViewModel` 复制私有 `spawnPosition(for:from:)` 后与真实 `recruit` 漂移。
- `GameViewModel` 已负责 UI 友好派生数据，例如 `SelectedCommanderBrief`、`SelectedTacticalOrderPreview`、`EnemyIntentMapOverlay`；本轮城市读板中的文案、排序和可访问摘要应放在这里。
- `BattleView` 目前城市分支只展示基础产出，军令面板直接遍历 `UnitKind.allCases` 并只显示金币；这些应改为消费 ViewModel 预览。
- `Tools/RenderBattlePreview` 已有确定性战斗场景，适合追加城市场景断言，避免截图工具只覆盖单位读板。

## 验收标准

- 点选罗马城市后，完整和紧凑情报栏都能读到城市收入、城防、资源/总收入、扩建收益和部署摘要。
- 军令面板的扩建、招募按钮使用 ViewModel 预览数据；资源不足、缺少港口或无部署格时会禁用并展示原因。
- 招募选项覆盖军团、骑兵、弓兵、舰队，且成本展示不再只有金币。
- 真实招募、扩建和 AI 招募行为没有变化；核心预览与真实结算使用同一部署/成本规则。
- 三尺寸预览通过，短横屏/竖屏命令入口仍可见。
- GitHub Actions 最新 v0.13 run 通过，Agent C artifact 验收通过。

## 风险和禁止项

- 禁止把城市经济、招募或部署真实结算搬进 SwiftUI。
- 禁止更改 `GameState.spawnPosition(for:from:)` 的真实行为；只能公开只读预览并让结算复用同一来源。
- 禁止在 ViewModel 预览中调用 mutating 方法试探规则。
- 禁止提交 DerivedData、PNG、xcresult、本地缓存或历史 artifact。
- 如果城市读板导致紧凑布局拥挤，优先压缩文案和使用滚动/折行，不要删除已有攻击、技能、休整、跳过入口。
