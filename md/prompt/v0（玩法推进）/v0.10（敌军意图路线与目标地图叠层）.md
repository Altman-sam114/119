# v0.10 / 敌军意图路线与目标地图叠层

## 版本号和版本分配依据

- 版本号：v0.10。
- 版本标题：敌军意图路线与目标地图叠层。
- 分配依据：人工和 Agent X 已指定本轮为 v0.10；`update_log.md` 当前最新正式版本为 v0.9（将领技能冷却与战功状态可读化）。本轮继续向“大征服者罗马”推进，但只做敌军意图可读化，因此在 v0.9 后递增为 v0.10。

## 背景

v0.7 已让 `aiIntents(for:limit:)` 的攻击类 `projectedDamage` 与规划态 `attackPreview` 保持一致，并保证预测只读不改变原始 `GameState`。v0.8 / v0.9 又把将领技能预览、冷却和战功状态展示到地图和侧栏。

当前敌军意图已经包含足够的只读信息：`destination`、`targetUnitID`、`targetCityID`、`projectedDamage`、`tacticalOrder` 和 `threatScore`。但 SwiftUI 主要只在敌军单位头上显示一个小徽标，侧栏 `detail` 也只有单行文本；玩家难以一眼看出敌军会走向哪里、攻击谁、夺哪座城、预计伤害是多少。

本轮目标是把现有 `AIIntent` 字段转成更清晰的 ViewModel 派生数据和地图叠层：显示意图路线、目的地、目标格和侧栏摘要。不得新增或改变 AI 行为算法，不得改变真实 AI 决策。

## 目标

1. 保持 `GameState.aiIntents(for:limit:)` 的预测语义不变：只读、排序和 AI 决策逻辑不变，攻击类 `projectedDamage` 继续来自规划态战斗预览。
2. 在 `GameViewModel` 为敌军意图新增 UI 友好的派生模型或派生集合，明确表达：
   - 起点：敌军当前 `unit.position`。
   - 目的地：`intent.destination`，没有则视为当前位置。
   - 目标格：`targetUnitID` 对应单位位置或 `targetCityID` 对应城市位置。
   - 预计伤害：`projectedDamage`，没有则显示技能、夺城、推进、固守或整备语义。
   - 路线段：仅用于可视化的 `origin -> destination -> target` 线段，不做寻路或新 AI 规划。
3. 在 `BattleView` 地图中显示敌军意图叠层：
   - 移动/接敌意图显示从当前位置到 `destination` 的路线线段或箭头。
   - 攻击/接敌攻击/技能意图显示目标格高亮。
   - 夺城意图同时高亮目标城市和预计进入的目的地。
   - 固守/整备意图保持轻量，不制造大量视觉噪音。
4. 优化敌情侧栏摘要，让每条意图读起来能回答“谁、从哪、去哪里、打谁/夺哪、预计伤害或效果”。
5. 地图叠层必须响应三种尺寸：短横屏 932x430、竖屏 390x844、宽屏 1024x768；不得遮挡单位生命条、技能叠层、攻击目标按钮或主要命令入口。
6. 更新核心测试、ViewModel 派生验证、Gameplay Smoke、三尺寸预览图和文档；CI artifact 版本同步到 v0.10。

## 非目标

- 不新增 AI 行为算法、不新增路径搜索、不改变 `bestAIDestination`、`bestAITarget`、AI 评分、AI 战术姿态选择或真实 `performSimpleAI` 行为。
- 不改变 `AIIntent` 的核心含义；除非有非常明确的 Codable 兼容理由，否则不要改 `AIIntent` 存储字段。
- 不改变战斗伤害、反击、占城、技能、冷却、战功、外交、任务或胜负规则。
- 不做大规模 `BattleView.swift` 拆分；只在本轮需要的位置提取小型 View / 派生结构。
- 不引入第三方框架、图片素材、网络服务、动画引擎、存档 UI 或新长期分支流程。
- 不创建 PR，不引入 `develop`、`smalldata_test`、`codeb/...` 或其他候选分支。

## 当前架构依据

- `Sources/RomeLegionsCore/GameState.swift`
  - `AIIntent` 已包含 `unitID`、`faction`、`kind`、`tacticalOrder`、`targetUnitID`、`targetCityID`、`destination`、`projectedDamage` 和 `threatScore`。
  - `aiIntents(for:limit:)` 使用 forecast copy，并调用 `refreshFactionForNewTurn(faction)`，预测不得改变原始 `GameState`。
  - `aiIntent(for:)` 已覆盖 `.attack`、`.advanceAttack`、`.captureCity`、`.advance`、`.defend`、`.regroup`、`.useSkill`。
  - `.advanceAttack` 已提供 `destination` 和 `targetUnitID`；`.captureCity` 已提供 `destination` 和 `targetCityID`；攻击类 `projectedDamage` 已与规划态 `attackPreview` 对齐。
- `RomeLegionsApp/App/GameViewModel.swift`
  - `EnemyIntentSummary` 已封装 `AIIntent`、来源单位、目标单位和目标城市，并提供 `title`、`detail`、`shortTitle`、`badgeText`、`threatLabel`。
  - `enemyIntentSummaries` 当前从敌对阵营各取 `state.aiIntents(for:limit: 2)`，排序后最多展示 5 条。
  - ViewModel 是本轮新增路线、目标格、目标文案、伤害文案等 UI 派生数据的正确位置；SwiftUI 不应复制 AI 决策规则。
- `RomeLegionsApp/Views/BattleView.swift`
  - `WarMapView` 已按 `HexMetrics.center(for:)` 定位六边形、单位、攻击徽标和叠层。
  - `HexTileView` 已有技能范围叠层、技能目标叠层、可移动叠层、攻击目标叠层和敌军单位徽标，可参考这些模式添加敌军意图目的地/目标叠层。
  - `EnemyIntentPanelView` / `EnemyIntentRowView` 是侧栏敌情摘要入口。
  - `AIIntentKind` 已在 `BattleView.swift` extension 中提供系统图标和颜色。
- `Tests/RomeLegionsCoreTests/GameStateTests.swift`
  - 已覆盖 AI 意图只读、直接攻击、移动后攻击、夺城、技能意图和伤害一致性。
- `Tools/GameplaySmoke/main.swift`
  - 已覆盖核心 AI 意图只读、直接攻击、移动后攻击、伤害预览一致性和 AI 实际结算一致性。
- `Tools/RenderBattlePreview/main.swift`
  - 已编译 `GameState.swift`、`GameViewModel.swift` 和 `BattleView.swift` 并渲染战斗页 PNG，适合承载 UI / ViewModel 轻量断言和三尺寸视觉验收。

   - `.allowsHitTesting(false)`，不得拦截地图点击。
