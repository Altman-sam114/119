# v0.15（将领成长与军团编制可读化）

## 版本号

v0.15

## 版本分配依据

`update_log.md` 当前最新正式功能版本为 v0.14（战线压力与 AI 战略意图读板），本轮继续围绕 Agent X 总目标“继续向大征服者罗马推进，优化战斗界面 UI、地图、AI、将领等”做可验证小步迭代，版本递增为 v0.15。

## 背景

项目已经具备将领任命、将领特性、主动技能、技能冷却、战功状态、战术姿态、敌军意图和战线压力读板。玩家能看见单个单位的将领和姿态，但还缺少接近战略战棋体验的“这支军团在编制中承担什么职责、战备是否完整、附近有没有友军支援、下一步应该保持战列/突击/坚守/行军”的综合读板。

本轮目标是新增只读的军团编制与成长报告：核心层从现有单位、将领、战功、姿态、相邻友军、附近敌军和有效攻防移派生报告；ViewModel 转成 UI 摘要；BattleView 在顶部态势、战局面板和选中单位情报中展示。该能力让将领/军团从“单兵卡片”推进到“战略编制资产”。

## 目标

1. 在 `RomeLegionsCore` 新增只读军团编制/成长报告模型，按阵营和单位生成报告。
2. 报告至少包含：单位 id、兵种、坐标、编制职责、战备等级、生命、军阶、将领/特性、姿态、建议姿态、有效攻防移、相邻友军数、两格内友军数、两格内敌军数、技能是否就绪、编制完整度和一句命令建议。
3. 报告必须由核心层生成，不在 SwiftUI 中重新评分或复制规则。
4. 不新增 `ArmyUnit` / `GameState` 存档字段，不改变 Codable 结构。
5. `GameViewModel` 新增 UI 友好的军团编制摘要，并为选中单位提供 `selectedLegionFormationSummary`。
6. `BattleView` 在完整战局面板展示前几条军团编制行，在顶部状态条展示首要军团态势 chip，在完整/紧凑选中单位情报里展示本单位编制建议；紧凑界面仍保持“军令”优先，不把新增读板插到军令前。
7. Swift Testing、Gameplay Smoke 和 RenderBattlePreview 覆盖核心报告字段、只读不变性、危急孤军场景和 UI 主链路所需字段。

## 非目标

- 不实现真实编制槽位、装备、升级树、技能点、人口、建筑树或大型养成系统。
- 不改变 `experience * 3` 伤害公式、训练/任命成本、技能冷却、城市招募部署、AI 招募策略、AI 评分或真实 AI 行为。
- 不修改 `performSimpleAI`、`aiIntent(for:)`、真实移动、攻击、反击、占城或战役胜负结算。
- 不拆分 `BattleView.swift` 作为专项重构；只允许新增本轮必要的小型视图。
- 不引入第三方框架，不使用原游戏素材，不改变技术栈。

## 当前架构依据

- 核心规则、单位、将领、战功、AI 和战线压力位于 `Sources/RomeLegionsCore/GameState.swift`。
- `WarMeritStatus` 已将经验映射为军阶、伤害加成和进度。
- `GeneralSkillPreview` 已只读提供技能范围、目标、冷却和阻塞原因。
- `GameViewModel.selectedCommanderBrief`、`selectedTacticalOrderPreviews` 和 `frontlinePressureSummaries` 是现有 UI 派生模式。
- `BattleView` 目前完整侧栏顺序是情报、战场、敌情、战局、军令等；紧凑侧栏顺序是军令、情报、战场、战报，新增内容不得挤掉首屏军令。

## 实现步骤

1. 在 `GameState.swift` 新增 `LegionFormationRole`、`LegionFormationReadiness` 和 `LegionFormationReport` 等只读模型。
2. 新增 `GameState.legionFormationReports(for:limit:)` 和 `GameState.legionFormationReport(unitID:)`：

   - 给出建议姿态和一句命令建议。

   - `title`
   - `roleLabel`
   - `readinessLabel`
   - `compactTitle`

   - `recommendationLabel`
   - `accessibilityLabel`
   并新增 `legionFormationSummaries`、`primaryLegionFormationSummary`、`selectedLegionFormationSummary`。
4. `BattleView.swift`：
   - 顶部 `TacticalStatusStripView` 增加首要“军团” chip。
   - 完整 `StrategicBalancePanelView` 在战线压力后、阵营态势前展示前 3 条军团编制行。
   - 完整/紧凑选中单位情报加入本单位编制建议卡。
   - 紧凑命令顺序保持不变：`CompactActionsPanelView` 仍在 `CompactSelectionPanelView` 前。
5. `Tests/RomeLegionsCoreTests/GameStateTests.swift`：

6. `Tools/GameplaySmoke/main.swift`：
   - 增加军团编制主链路断言：报告存在、将领/战功/支援/建议字段可读、调用不改变状态。
7. `Tools/RenderBattlePreview/main.swift`：

8. 更新文档：
