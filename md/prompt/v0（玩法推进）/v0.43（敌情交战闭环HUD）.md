# v0.43（敌情交战闭环 HUD）

## 版本号

v0.43

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.42（将领指挥链读板），并已通过 `origin/main` 云端结果包复判。v0.42 已把选中将领的被动、技能目标、战功、将令、目标线和处境入口压缩成将领卡内短读板。下一步应把敌军路线、战线压力、敌将威胁、反制指令和我方回应将领链串成一条地图内战场闭环提示，让玩家在地图上直接读到“敌人从哪来、压到哪里、谁威胁最大、我方谁回应、下一步做什么”。递增为 v0.43。

## 背景

当前项目已有：

- `EnemyIntentSummary` 和 `EnemyIntentMapOverlay`：敌军意图、六边形路线、目的地和目标格。
- `FrontlinePressureSummary`：战线压力目标、来源、预计伤害和压力等级。
- `EnemyCommanderThreatSummary`：敌将、技能窗口、目标、影响和威胁等级。
- `CountermeasureSummary` 与 `CountermeasureCommandPreview`：反制建议、回应单位、推荐姿态、落点、目标、下一步和阻塞原因。
- `SelectedCommanderChainReadout`：选中回应军团后，将领 brief、技能目标、战功、将令、目标线和处境入口的只读指挥链。
- `BattlefieldConvergenceSummary`：目标线、反制、阶段、将令、机动、热区和控区的玩家侧交汇读板。

但敌情链路仍分散在地图路线、敌情卡、反制卡和选中单位将领卡中。v0.43 应新增一个只读的敌情交战闭环读板，并在地图内展示一条紧凑 HUD，使战斗界面更接近战棋游戏的即时态势条。

## 目标

1. 在 `GameViewModel.swift` 新增敌情交战闭环只读派生：
   - 建议新增 `EnemyEngagementLoopSignalKind`、`EnemyEngagementLoopSignal`、`EnemyEngagementLoopReadout`。
   - 新增 `primaryEnemyEngagementLoopReadout` computed property。
   - 只读取既有 ViewModel 派生数据：敌军意图路线、首要战线压力、首要敌将威胁、首要反制建议、首要反制指令预览、当前选中回应军团的 `selectedCommanderChainReadout`、战场态势交汇读板。
   - 输出字段建议包含 `title`、`statusLabel`、`intentLabel`、`pressureLabel`、`enemyCommanderLabel`、`countermeasureLabel`、`responseLabel`、`nextStepLabel`、`riskLabel`、`compactLabel`、`signals`、同源 ID、`accessibilityLabel`。
   - signal 只保存 kind、标题、详情、位置和 sourceID；不要保存 SwiftUI `Color`。
   - 提供 references 方法或等价同源校验入口，供 `RenderBattlePreview` 检查。
2. 在 `BattleView.swift` 新增地图内紧凑 HUD：
   - 在地图区域顶部或地图内部上方展示 `primaryEnemyEngagementLoopReadout`。
   - HUD 只用一条紧凑横向读板，显示敌路/压力/敌将/反制/回应/下一步的短标签。
   - 不新增按钮，不拦截地图点击，不改变现有地图叠层。
   - SwiftUI 只消费 ViewModel 派生字段，不重新计算敌军意图、压力、威胁、反制或将领链。
3. 在 `Tools/RenderBattlePreview/main.swift` 更新云端断言：
   - 新增错误 case，建议 `missingEnemyEngagementLoopReadout`。
   - 断言读板引用首要战线压力、敌将威胁、反制建议、反制指令预览和战场态势交汇。
   - 聚焦首要反制后，断言读板能引用选中回应军团的 `selectedCommanderChainReadout`，且回应单位与反制指令同源。
   - 断言 signals、标题、状态、敌路、压力、敌将、反制、回应、下一步、风险和无障碍文案非空。
   - 断言读取读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.43`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.42 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、AI 权重、敌军意图评分、敌将威胁评分、反制评分、目标线评分或将领技能规则。
- 不新增命令队列、自动反制、自动目标线执行、一键连招、多回合搜索或真实作战计划执行器。
- 不改变移动、攻击、姿态、技能、结束回合、城市、外交、存档或 AI 真实执行逻辑。
- 不新增地图叠层规则；本轮只新增一条地图内 HUD 解释既有叠层和读板。
- 不重做 BattleView 总布局，不引入第三方框架，不引入新素材。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- 敌军路线已由 `enemyIntentMapOverlays` 派生，并通过地图叠层展示。
- 战线压力、敌将威胁、反制建议和反制指令预览已经分别有 summary/preview 和 Render 断言。
- `focusCountermeasure(_:)` 只改变 ViewModel 选择态、聚焦 ID 和 banner，不写 `GameState`；它可用于把反制回应军团选中后读取 `selectedCommanderChainReadout`。
- `BattlefieldConvergenceSummary` 已证明目标线/反制/阶段/将令/机动/热区/控区的同源关系，v0.43 闭环读板应引用它，不把它改成更胖的万能对象。
- 地图 HUD 应落在 `WarMapView` 附近，作为地图内提示，不应再挤压侧栏卡片。

## 验收标准

- `primaryEnemyEngagementLoopReadout` 能在确定性战斗预览场景中生成。
- 读板能引用敌军路线、战线压力、敌将威胁、反制建议、反制指令预览和战场交汇读板。
- 聚焦首要反制后，读板能引用回应军团的 `selectedCommanderChainReadout`，且回应单位与反制指令同源。
- 标题、状态、敌路、压力、敌将、反制、回应、下一步、风险、compact、signals 和无障碍文案非空。
- 地图内 HUD 在宽屏、横屏和竖屏预览中可读，不明显遮挡地图顶部信息或拦截地图交互。
- 读取闭环读板不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算敌军意图、压力、威胁、反制或将领链。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：地图顶部已有战役状态 chip，HUD 容易遮挡；应放在地图内部低位或用紧凑布局，并禁止拦截触控。
- 风险：敌军意图排序可能随 AI 权重变化；Render 断言应优先核对同源和非空，不硬编码具体敌军路线文案。
- 风险：聚焦反制会改变选中单位；断言应明确先检查未聚焦读板，再聚焦后检查回应将领链。
- 禁止把闭环 HUD 变成自动反制、自动技能或自动目标线执行系统。
- 禁止在 SwiftUI 中重新计算敌军路线、战线压力、敌将威胁、反制建议或将领链。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
