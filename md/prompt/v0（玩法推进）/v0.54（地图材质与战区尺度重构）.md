# v0.54 地图材质与战区尺度重构

## 版本号与分配依据

版本号为 v0.54。`origin/main` 最新正式验收版本为 v0.53，已完成地图主导壳层、侦察叠层过滤、单层情报坞和原创将领徽章；本轮沿玩法推进序列递增，集中解决 v0.53 六图仍存在的战区偏小、地图空场、地貌同质和军团锚点抽象问题。

## 背景与公开参考结论

Agent X 于 2026-07-24 重新核对《大征服者：罗马》Google Play 官方商店公开截图。只提炼信息架构和视觉层级，不复制素材：

- 战略地图持续铺满可用画面，地形、国界、道路、水系和城市构成连续底图，UI 只占顶部、四角和底边。
- 平原、林地、山脉、水域和城镇通过不同纹理、明暗与轮廓识别，而不是重复使用同一图标。
- 城市名称、阵营徽记、军旗、兵种剪影、将领头像和兵力点共同形成地图锚点；有将领军团的层级明显高于普通地块。
- 大比例尺和战术比例尺都保持低饱和地貌底色，把红、蓝、金等高纯度颜色留给阵营、目标和命令。

本轮禁止下载、提交、描摹或生成原游戏的地图、人物、图标、纹理、商标与界面素材；实现只使用原创 SwiftUI 形状、Canvas、SF Symbols 和项目既有数据。

## 本轮目标

1. 把 `WarMapView` 的全屏背景重构为具有陆地分区、水系、等高线、道路脉络和纸面颗粒感的原创战略底图，消除六边棋盘之外的无意义空场。
2. 重做平原、森林、丘陵、水域、道路和城市六类 tile 材质，使它们在小尺寸下仍能靠纹理和轮廓区分，弱化重复的大型地形 glyph。
3. 调整 `HexMetrics` 的安全边距、tile 高宽比和横/竖屏尺度，让短横屏和宽屏战区更充分占用地图工作区，同时保持全图、路线、单位和 HUD 不裁切。
4. 升级城市与军团地图锚点：城市使用城墙/建筑剪影、阵营顶线和名称牌；军团使用军旗轮廓、兵种剪影、兵力条、战功与指挥官徽章，保持所有旧状态可读。
5. 扩展 RenderBattlePreview：保留全部既有语义与像素门禁，新增六类地貌材质策略、战区横向占位和地貌颜色/纹理分布检查；由最新 `main` artifact 输出六图供 Agent C 逐图复判。

## 非目标

- 不修改核心规则、存档、AI 评分/意图/行动顺序、战斗、移动、技能、姿态、资源、城市经营、外交、任务、胜负或回合结算。
- 不增加地图缩放、拖拽、摄像机、战争迷雾、新地形类型、3D、动画系统、外部图像或第三方框架。
- 不删除侦察视角、路线/目标叠层、情报坞、边缘工具、抽屉、命令坞或现有无障碍语义。
- 不全面拆分 `BattleView.swift`，不修改受保护的 Xcode 工程文件。

## 当前架构依据

- `BattleView` 以顶部资源带、唯一 `WarMapView`、右侧五个工具、按需抽屉和底部选择命令坞组成壳层。
- `WarMapView` 使用 `HexMetrics` 统一计算 12x8 棋盘坐标，并通过 `MapOverlayPresentation` 过滤侦察叠层。
- `MapBackdropView` 目前只是三色渐变和规则斜线，棋盘之外仍像 UI 背景，不像连续战区。
- `TerrainTextureView` 已为六类地形分派小组件，但每类只含少量胶囊、圆点或三角形，`TerrainGlyphView` 仍承担过多辨识责任。
- `CityBadgeView` 与 `UnitTokenView` 已展示阵营、城防、兵种、将领、姿态、经验、冷却、行动状态和生命，本轮只改展示层级，不删状态。
- `Tools/RenderBattlePreview/main.swift` 已输出三尺寸单位/城市共六图，并检查地图主导壳层、情报坞、工具轨和命令坞。

## 设计与实现要求

### 1. 原创战略底图

- `MapBackdropView` 应包含多层低对比地貌：不规则陆地色块、水域带、等高线/海流线、道路或边界脉络与轻颗粒。
- 底图必须铺满 `WarMapView`，横竖屏均无明显空白；不能抢过 tile、路线、单位、目标和 HUD。
- 使用稳定、确定性的 Canvas 路径，不依赖随机数、时间或生命周期状态，保证云端六图可复现。
- 颜色至少覆盖青绿水域、橄榄陆地、灰褐高地和中性暗底，避免单一蓝灰或纯绿色主题。

### 2. 六类地貌材质

- 平原：田垄/草痕；森林：树冠群与树干节奏；丘陵：多层山脊和阴影；水域：多层波纹/流向；道路：有底边、主路和虚线高光；城市：街区、城墙或建筑节奏。
- 每类至少有两层视觉信号，并提供稳定的展示元数据或签名供 RenderBattlePreview 做只读断言。
- 空 tile 的大型 `TerrainGlyphView` 应降权或仅保留小型地标，不能继续用同一尺寸 SF Symbol 覆盖材质。
- tile 默认边界应比选中/攻击/技能/阵营边界更弱，棋盘整体应更接近连续地貌。

### 3. 战区尺度

- `HexMetrics` 缩小重复预留的 top/bottom inset，优先使用地图工作区高度；横屏允许小幅压扁 tile，但要保持可辨识的六边形。
- 390x844、932x430、1024x768 都必须保持棋盘最外圈、城市、军团、路线箭头和情报坞不裁切。
- 地图尺寸只由容器和棋盘尺寸决定，不得随选择对象、动态文案或抽屉状态跳动。
- 禁止使用 `UIScreen.main.bounds`、任意硬编码摄像机偏移或第二套坐标系统。

### 4. 城市与军团锚点

- 城市 compact badge 使用代码绘制的城墙/建筑轮廓和更清楚的名称牌，继续显示阵营和城防。
- 军团 token 使用军旗或盾牌轮廓而非普通圆角标签；兵种图形和短标签需有主次，生命条、经验、姿态、冷却和完成状态不得丢失。
- 指挥官徽章继续使用原创首字符/头盔符号，不使用人物肖像；有无将领不能只靠颜色区分。
- 小 tileScale 下保证文本和标记不溢出、不遮住相邻军团；无障碍标签继续描述完整状态。

### 5. SwiftUI 质量边界

- 新组件只负责绘制，不读取或写入业务状态；按钮继续调用现有 ViewModel 方法。
- 避免 `AnyView`、不稳定 identity、重复 filter/sort、随机纹理和无 value 动画。
- 使用现代 SwiftUI API、明确尺寸和稳定 layout；交互区域仍至少 44x44。
- 不修改 `GameViewModel` 和 `GameState` 以满足纯视觉需求。

## 验收标准

- 战略底图铺满地图工作区，六边棋盘之外不再是规则斜线空背景。
- 六类地貌在不看大 glyph 时仍能通过纹理和轮廓区分，色彩至少覆盖水域、陆地、高地和道路/城市。
- 三种尺寸棋盘、路线、城市、军团、情报坞、工具轨和命令坞无裁切或重叠，短横屏战区占屏感较 v0.53 提升。
- 城市和军团不再像普通 UI 标签；将领、兵种、生命、战功、姿态、冷却和行动状态仍可读。
- 核心、AI、ViewModel 派生规则和存档无业务变化；受保护 Xcode 文件不进入本轮提交。
- 最新 `origin/main` 对应的 v0.54 run 成功，artifact manifest 与 commit/run/attempt 一致，所有云端步骤成功，JUnit failures 为 0，六图人工复判通过。

## 风险与禁止项

- 风险：纹理过密会与路线/目标争抢。所有底图和地貌纹理必须低对比，命令颜色始终在最上层。
- 风险：扩大 tile 会裁切 HUD 或外圈。必须通过统一 metrics 与像素门禁解决，不用多个局部 offset 补偿。
- 风险：军团锚点变大会遮挡相邻格。组件尺寸必须跟随 `tileScale`，保持命中坐标不变。
- 禁止复制原游戏资产或品牌元素，禁止随机绘制导致截图不稳定。
- 禁止修改核心/AI/存档，禁止降低旧门禁阈值制造通过，禁止伪造测试、截图、日志或 artifact。
- 禁止本地运行任何验证命令，禁止创建 PR/其他分支，禁止触碰受保护 Xcode 文件。
