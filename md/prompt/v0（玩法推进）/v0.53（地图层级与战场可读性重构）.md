# v0.53 地图层级与战场可读性重构

## 版本号与分配依据

版本号为 v0.53。`origin/main` 最新正式验收版本为 v0.52，已完成地图主导壳层、边缘工具、按需抽屉和选择命令坞；本轮沿玩法推进序列递增，继续解决 v0.52 云端六图暴露的地图内部层级问题。

## 背景与参考结论

Agent X 重新检索了《大征服者：罗马》的公开商店与战役攻略截图。只提炼信息架构，不复制素材：

- 地图是持续占满背景的第一视觉主体，顶部资源条和边缘工具很薄。
- 同一时刻只突出当前决策所需的路线、目标与危险区域，不让所有情报等权叠加。
- 战场图标体积小，但阵营、兵种、城市、将领和生命等决策信息有清晰层级。
- 侧边列表和底部入口贴边，避免大块说明面板长期遮挡战区。

v0.52 的 932x430、390x844、1024x768 云端截图已经证明壳层可用，但仍有四类问题：

1. 敌军路线、反制路线、目标线、军议路线、热区、控区与多个 tile 徽标同时高亮，地图中心信息噪声过高。
2. `selectedMapReconPerspective` 目前只改变 HUD 文案，没有真正控制地图叠层的视觉优先级。
3. 竖屏六边格按可用区域居中，战区落在屏幕中下部，上半部出现明显空场；横屏底部三条 HUD 也占据较多地图面积。
4. 单位将领只用角落星标表示，和普通军团的视觉层级差异不足。

本轮继续使用原创 SwiftUI 形状、SF Symbols 和现有颜色体系。禁止下载、提交、描摹或生成原游戏的地图、人物、图标、纹理、商标与界面素材。

## 本轮目标

1. 让现有四种地图侦察视角真正成为叠层视觉过滤器：敌路、反制、目标线、热区各自突出对应路线、tile 徽标和图例，其他叠层仅保留必要上下文或暂时收起。
2. 调整 `HexMetrics` 的比例适配与垂直偏置，减少竖屏上半部空场，让横屏/宽屏战区在不裁切 HUD、工具和命令坞的前提下更集中、更易扫读。
3. 将地图底部侦察 HUD、敌情闭环 HUD 和叠层图例收敛成稳定的单层情报坞；当前视角、状态、切换入口和必要上下文仍可见，其余完整信息继续在既有抽屉中可达。
4. 把将领角标升级为原创的微型指挥官徽章，使有将领军团无需依赖颜色即可与普通军团区分，并保持兵种、姿态、经验、冷却、生命和行动状态可读。
5. 同步 RenderBattlePreview 像素验收、CI 版本和架构/测试文档，通过 `main` push 后的最新 artifact 证明三种尺寸、城市/单位共六图均无裁切、异常空白或关键入口丢失。

- 不修改 AI 评分、意图预测、行动顺序、敌将威胁、反制建议、战线压力、热区、控区、目标线或报告生成逻辑。
- 不新增地图缩放/拖拽手势、迷雾规则、战争迷雾状态、动画系统、人物肖像资产、地图纹理、装备或新玩法。
- 不删除任何现有 panel、抽屉分类、命令入口、HUD 数据源或无障碍语义。
- 不做 `BattleView.swift` 的全面拆文件或无关 API 批量现代化；不得修改用户受保护的 Xcode 工程文件。

## 当前架构依据

- `BattleView` 负责薄顶部带、唯一 `WarMapView`、五个边缘工具、按需 `BattlefieldDrawerView` 与底部 `SelectionCommandDockView`。
- `WarMapView` 当前无条件组合敌路、军议、目标线、反制四类 route layer，并把热区、控区、目标线、反制、敌军目标等状态同时传入每个 `HexTileView`。
- `GameViewModel.selectedMapReconPerspective` 已有 `.enemyIntent`、`.countermeasure`、`.objective`、`.terrainPressure` 四种状态；`selectMapReconPerspective(_:)` 已是现成入口，本轮不再创建第二套视角状态。
- `mapReconPerspectiveHUDReadout`、`primaryEnemyEngagementLoopReadout` 与 `activeMapOverlayLegendItems` 已提供完整只读信息；SwiftUI 只负责选择显示层级，不重新计算报告。
- `HexMetrics` 当前固定居中地图，竖屏宽度先达到上限时会产生过大的上下空白。
- `UnitTokenView` 已显示兵种、姿态、经验、技能冷却、行动状态和生命；将领仅为右上角星标。
- `Tools/RenderBattlePreview/main.swift` 已输出六张 PNG 并检查地图主导壳层、边缘工具和城市/单位命令坞；本轮在保留全部旧语义断言的基础上增加地图聚焦与战区占位验收。

## 设计与实现要求

### 1. 侦察视角控制叠层优先级

在视图层建立轻量、纯展示的叠层呈现策略，可为独立 struct/enum，也可为清晰的只读 computed property，但不得写入 `GameState`：

- `.enemyIntent`：完整显示敌军路线、目的地和目标徽标；反制只保留必要上下文，目标线、热区、控区不与敌路争夺同等强调。
- `.countermeasure`：完整显示反制路线、落点和目标；敌军路线可低对比保留为来源上下文，目标线、热区与控区收起。
- `.objective`：完整显示战场目标路线、阶段和目标 tile；军议路线可作为当前行动上下文，敌路、反制、热区与控区降低或收起。
- `.terrainPressure`：完整显示热区、控区和态势交汇相关 tile；路线类叠层只保留最必要的低对比上下文。
- 玩家正在执行的可达、攻击目标、技能范围/目标、选中格和当前军议落点不得被视角过滤隐藏。
- 过滤只改变渲染可见性、透明度和 zIndex，不改变任何 ViewModel 集合、排序、选择态、预览或命令结果。
- `activeMapOverlayLegendItems` 的数据不得删改；地图上只显示当前视角相关的紧凑图例，完整图例仍应在战场/情报抽屉或无障碍描述中可达。若现有抽屉没有完整图例入口，可在当前地图情报坞中保留可横向滚动的全部图例，但非当前项须明显降权。

### 2. 地图比例与构图

- 调整 `HexMetrics`，按容器宽高比使用稳定的垂直偏置：竖屏在宽度成为限制时把战区上移到地图工作区约 30%...42% 的视觉中心，不能继续机械居中到下半屏。
- 横屏和宽屏优先利用地图高度，允许通过压缩 HUD 占位、缩小 top/bottom inset 或小幅提高 tile 上限扩大战区，但不得裁切最外圈六边格、城市、单位、路线箭头或攻击徽标。
- 保持 `tileWidth`、`tileHeight`、`tileScale` 和 `actionScale` 有明确上下限；动态文案、选中对象和抽屉开关不得导致地图尺寸跳动。
- 禁止 `UIScreen.main.bounds`。保留 `WarMapView` 为唯一地图，不建立第二套坐标系统。

### 3. 单层地图情报坞

- 替换当前底部最多三层纵向堆叠：`MapReconPerspectiveHUDView`、`EnemyEngagementLoopHUDView`、`MapOverlayLegendView` 应组合进一个职责明确的地图情报坞。
- 横屏/宽屏优先使用单行：左侧四种侦察视角切换，中部当前状态/信号，右侧紧凑上下文或图例；高度目标约 34...44pt。
- 竖屏允许换成两行，但不得恢复三条大面板，整体高度应稳定且不覆盖命令坞或边缘工具。
- 敌情闭环信息不能删除：在敌路/反制视角显示其状态、风险或下一步的紧凑表达，完整链路继续由敌情抽屉承载。
- 当前视角的图例应明显强调，非当前视角图例降权或通过横向滚动访问。状态区不能只靠颜色区分。
- 四种视角按钮保持至少 44pt 命中区域、SF Symbol、文本 accessibility label 和选中标记；不要加入使用说明文案。

### 4. 顶部带与窄屏标题

- 保留菜单、回合/战役状态、五类资源和结束回合。
- 390pt 竖屏使用 `ViewThatFits` 或明确的短标题，避免当前“战役 · 第 1 回合 · 战...”硬截断；banner 可使用更短同源摘要，但不得伪造状态。
- 不能通过删除资源种类解决宽度问题；紧凑资源仍须在 VoiceOver 中完整报读。

### 5. 原创指挥官徽章

- 有将领的军团在 `UnitTokenView` 使用原创代码绘制的圆形/盾形微徽章，可用姓名首字符或 SF Symbol 表示指挥官，配合金色边框、轮廓或形状差异；禁止人物肖像和外部资产。
- 徽章应在 `tileScale` 下保持可辨识，并避开姿态、冷却、经验、行动状态与生命条。
- 普通军团不显示空徽章；有无将领的差异不能只靠颜色。
- 地图 tile 的 accessibility label 继续完整描述将领身份，命令坞仍显示全名。

### 6. SwiftUI 质量边界

- 复用 `selectedMapReconPerspective`，不新增重复状态；若需要纯本地展开状态，必须为 `private @State`。
- 复杂布局提取为职责清楚的 View；本轮禁止改 `project.pbxproj`，因此新增小类型先留在 `BattleView.swift`。
- 按钮动作调用现有 ViewModel 方法；不在 `body`、按钮闭包、`onAppear` 或生命周期回调里加入业务规则。
- 避免 `AnyView`、重复 filter/sort 和不稳定 identity。需要隐藏叠层时优先用稳定的参数/透明度或明确条件，不复制 overlay 数据。
- 使用 `foregroundStyle`、`clipShape(.rect(cornerRadius:))`、带 value 的动画；本轮若无必要，不新增动画。
- 交互命中区域至少 44x44，图标按钮保留文字 accessibility label；尊重 Dynamic Type、VoiceOver、Differentiate Without Color 和 Reduce Motion。

## 风险与禁止项

- 风险：过滤叠层可能误隐藏当前命令目标。攻击、技能、选中、可达和当前命令预览优先级高于侦察过滤。
- 风险：上移地图可能与顶部状态带或工具栏重叠。必须通过 inset 与稳定区域解决，不用多个任意 offset 互相补偿。
- 风险：HUD 收敛后信息可能不可达。完整敌情、战场与图例仍需通过抽屉或可访问控件到达，不能直接删除。
- 风险：将领徽章过大遮住 token。徽章必须随 scale 收敛并避开现有角标。
- 禁止复制原游戏视觉资产、人物肖像、地图纹理、文案或品牌元素。
- 禁止修改核心规则、AI 结果、预览公式、存档结构或用 UI 自行计算业务数值。
- 禁止删除/跳过/弱化 RenderBattlePreview 旧断言，禁止伪造测试、截图、日志、manifest 或 artifact。
- 禁止本地运行任何验证命令，禁止创建 PR 或其他分支，禁止触碰受保护 Xcode 文件。
