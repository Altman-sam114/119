# v0.44（将领战机威胁桥接读板）

## 版本号

v0.44

## 版本分配依据

`update_log.md` 当前最新完成版本为 v0.43（敌情交战闭环 HUD），并已通过 `origin/main` 最新云端结果包复判。v0.43 已把敌军路线、战线压力、敌将威胁、反制指令、回应将领链和战场态势交汇压缩成地图内闭环 HUD。下一步应继续推进“大征服者罗马”体验中玩家对将领战术价值的直觉判断：选中有将领军团后，用一个短读板把本方将令机会、主动技能目标、敌方将领威胁、反制入口和目标线阶段串起来，告诉玩家这名将领当前是适合发动技能、参与合击、压制敌将还是救线。递增为 v0.44。

## 背景

当前项目已有：

- `SelectedCommanderBrief`：选中单位将领名、特性、被动贡献、技能状态、技能效果和战功摘要。
- `SelectedGeneralSkillTargetReadout`：主动技能目标数量、收益、地图标记和目标短列表。
- `SelectedCommanderChainReadout`：将领被动、技能目标、战功、将令、目标线阶段和处境入口的短指挥链。
- `CommanderSynergySummary`：本方将令协同、协同步骤、预计收益和阻塞原因。
- `EnemyCommanderThreatSummary`：敌方将领威胁、技能窗口、目标、影响和状态。
- `CountermeasureSummary` 与 `CountermeasureCommandPreview`：反制建议、回应单位、推荐姿态、落点、目标、下一步和阻塞原因。
- `BattleObjectiveStageCommandPreview`：目标线阶段命令入口、技能阶段 cue、推荐姿态和下一步。
- `EnemyEngagementLoopReadout`：敌情闭环 HUD 已能串联敌路、压力、敌将、反制和回应。

但选中本方将领时，玩家仍需要跨将领卡、将令卡、敌将卡、反制卡和目标线卡判断“这名将领现在最该做什么”。v0.44 应新增一个只读的“将领战机威胁桥接读板”，放在将领卡内部，直接显示本方将领机会与敌方威胁/反制入口的关系。

## 目标

1. 在 `GameViewModel.swift` 新增将领战机威胁桥接只读派生：
   - 建议新增 `CommanderOpportunityBridgeSignalKind`、`CommanderOpportunityBridgeSignal` 和 `SelectedCommanderOpportunityBridgeReadout`。
   - 新增 `selectedCommanderOpportunityBridgeReadout` computed property。
   - 只读取既有 ViewModel 派生数据：`selectedUnit`、`selectedCommanderBrief`、`selectedCommanderChainReadout`、`selectedGeneralSkillTargetReadout`、`selectedCommanderActionGuidance`、`selectedCommanderSynergySummary`、`primaryEnemyCommanderThreatSummary`、`primaryCountermeasureSummary`、`selectedCountermeasureCommandPreview` 或 `primaryCountermeasureCommandPreview`、`selectedBattleObjectiveStageCommandPreview`、`primaryEnemyEngagementLoopReadout`。
   - 输出字段建议包含 `unitID`、`title`、`statusLabel`、`opportunityLabel`、`skillWindowLabel`、`enemyThreatLabel`、`counterLabel`、`entryLabel`、`nextStepLabel`、`riskLabel`、`compactLabel`、`signals`、同源 ID 和 `accessibilityLabel`。
   - signal 只保存 kind、标题、详情、位置和 sourceID；不要保存 SwiftUI `Color`。
   - 提供 references 方法或等价同源校验入口，供 `RenderBattlePreview` 检查。
   - 无选中单位或选中单位无将领时返回 nil；不要为无将领单位制造伪读板。
2. 在 `BattleView.swift` 展示短读板：
   - 在 `CompactGeneralTraitView` 和 `GeneralTraitCardView` 内部展示，建议插入在 `CommanderChainReadoutView` 之后、技能详情/技能状态之前。
   - 紧凑版只显示一行：战机/敌将/入口或等价短标签。
   - 完整版可显示两行：机会与威胁、入口与下一步；不要展开成新大卡。
   - SwiftUI 只消费 ViewModel 派生字段，不重新计算将领机会、敌将威胁、反制入口或目标线阶段。
3. 在 `Tools/RenderBattlePreview/main.swift` 更新云端断言：
   - 新增错误 case，建议 `missingCommanderOpportunityBridgeReadout`。
   - 在现有 `missingCommanderChainReadout`、`missingEnemyEngagementLoopReadout` 附近加入断言。
   - 断言固定单位场景中选中 `rome-legion-1` 后，读板非空，并引用同一 `selectedCommanderBrief`、`selectedCommanderChainReadout`、`selectedGeneralSkillTargetReadout`、`selectedCommanderSynergySummary`、首要敌方将领威胁、反制建议/指令、目标线阶段和敌情闭环中存在的来源。
   - 若当前选中回应军团与首要反制单位同源，断言桥接读板能引用 `selectedCountermeasureCommandPreview`；否则允许引用首要反制指令预览，但必须说明入口来自反制或目标线。
   - 断言标题、状态、机会、技能窗口、敌将、反制、入口、下一步、风险、compact、signals 和无障碍文案非空。
   - 断言读取桥接读板不改变单位、城市、资源、回合或当前势力。
4. 更新 `.github/workflows/ci-results.yml` 的 `CI_VERSION` 为 `v0.44`。
5. 更新 README、flow、flowchart、test、prompt README、AGENTS、update_log；保留 v0.43 云端通过证据。

## 非目标

- 不修改 `GameState` 核心规则、将领技能规则、AI 权重、敌方将领威胁评分、反制评分、目标线评分或将令协同评分。
- 不新增命令队列、自动技能、自动反制、一键连招、自动目标线执行、多回合搜索或真实作战计划执行器。
- 不改变移动、攻击、姿态、技能、结束回合、城市、外交、存档或 AI 真实执行逻辑。
- 不新增新的地图叠层规则；本轮只解释既有技能目标、敌将威胁、反制、目标线和敌情闭环来源。
- 不重做 BattleView 总布局，不引入第三方框架，不引入新素材。
- 不运行任何本地测试、build、typecheck、RenderBattlePreview、`Tools/verify_project.mjs`、`git diff --check`、YAML/JSON/Plist 解析或脚本语法检查。

## 当前架构依据

- `GameViewModel` 已是 UI 派生数据的边界，最近 v0.38-v0.43 均通过新增只读 readout 将既有 summary 串联到 SwiftUI。
- `selectedCommanderChainReadout` 已证明将领 brief、技能目标、战功、将令、目标线阶段和处境入口的同源关系，v0.44 不应替代它，而应在它之上增加“战机/威胁/入口”桥接。
- `primaryEnemyCommanderThreatSummary` 和 `primaryEnemyEngagementLoopReadout` 已提供敌将威胁和敌情闭环；本轮只读取，不改变敌方预测或真实 AI 行动。
- `CountermeasureCommandPreview` 与 `BattleObjectiveStageCommandPreview` 已提供既有命令入口 cue；本轮只把入口文案暴露到将领卡，不能改变按钮 action 或 disabled 规则。
- `CompactGeneralTraitView` 和 `GeneralTraitCardView` 是最小 UI 落点，避免把战场面板或地图 HUD 继续加重。

## 验收标准

- `selectedCommanderOpportunityBridgeReadout` 能在确定性战斗预览单位场景中为 `rome-legion-1` 生成。
- 读板能引用同一选中将领 brief/指挥链、技能目标读板、本方将令协同、首要敌将威胁、反制建议或指令、目标线阶段和敌情闭环中存在的来源。
- 标题、状态、机会、技能窗口、敌将、反制、入口、下一步、风险、compact、signals 和无障碍文案非空。
- 完整/紧凑将领卡都能读到桥接短摘要，不在短横屏或竖屏明显溢出。
- 读取桥接读板不改变 `GameState` 的单位、城市、资源、回合或当前势力。
- SwiftUI 只展示 ViewModel 派生结果，不重新计算将领机会、敌将威胁、反制建议、目标线阶段或敌情闭环。
- 云端 GitHub Actions 最新 run 通过，Agent C 下载并核对最新 artifact 后给出通过结论。

## 风险和禁止项

- 风险：将领卡已有指挥链、技能状态、战功和目标收益，桥接读板若过长会重复拥挤；实现必须压缩成短读板。
- 风险：敌将威胁和反制建议的首要对象可能随 fixture 或评分变化；Render 断言应核对同源和非空，不硬编码具体文案或坐标。
- 风险：`selectedCountermeasureCommandPreview` 依赖选中回应军团；若当前选中将领不是首要反制回应单位，可退回首要反制预览，但必须在读板中保留清晰入口。
- 禁止把桥接读板变成自动技能、自动反制、自动目标线执行或命令队列。
- 禁止在 SwiftUI 中重新计算技能目标、敌将威胁、反制评分、将令协同或目标线阶段。
- 禁止用旧 run、旧 artifact、本地输出或未验收结果推进 Agent X 循环。
